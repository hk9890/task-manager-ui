## Description
Verifier found that the new search mode implementation is not yet correct in the active runtime path. Search controller wiring exists and tests pass, but the real app has two runtime failures: (1) search init invokes empty `bd search --json` and surfaces `search query is required`, and (2) shell-level hotkeys intercept search text entry so characters like `b` switch modes instead of editing the search/filter input.

## Instructions
1. Prevent empty-query search init from calling unsupported `bd search` behavior. Use a supported empty-state or gateway-backed fallback that preserves the standalone architecture.
2. Ensure active search-mode key handling gets first chance to consume text-entry keys before shell-level mode-switch hotkeys fire.
3. Preserve gateway-only data access; do not shell out from the UI layer.
4. Add or update tests that cover the runtime regressions:
   - search startup does not error on empty query
   - typing search text does not trigger shell mode-switch hotkeys
5. Re-run `go build ./cmd/bwb`, `go vet ./...`, and `go test ./...`.

## Files to Modify
- `internal/app/model.go`
- `internal/mode/search/model.go`
- `internal/mode/search/model_test.go`
- `internal/app/model_test.go`
- related search/ui files if needed
