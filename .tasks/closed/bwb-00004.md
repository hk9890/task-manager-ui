## Description
Real runtime review against `pearles.png` and `our-tool.png` shows the current Beads Workbench dashboard still fails a core product requirement: the board lanes do not use the full terminal height like Perles. Instead of full-height, equal-height vertical lanes anchored to the bottom of the board area, the current UI renders short floating boxes near the top with large unused vertical space below. This makes the screen look unfinished and unprofessional.

## Evidence
- User-provided screenshots `pearles.png` and `our-tool.png`
- Real PTY runtime capture from current app shows 4 columns at the top and large unused vertical space below
- User explicitly requires a Perles-like full-height dashboard and does not accept the current look

## Instructions
1. Rework the board renderer and shell layout so the 4 dashboard lanes use the available board height, extending visually to the bottom of the board area.
2. Ensure the visible columns share the same rendered height in the main dashboard screen.
3. Remove the floating-card look; the dashboard should read as one coherent full-height board grid.
4. Keep the current 4-column grouping, compact rows, and no-large-sidebar layout unless a small adjustment is strictly required.
5. Add/update golden tests at representative heights and widths so lane height and bottom alignment are test-covered.
6. Re-check the real PTY/runtime against the user's Perles-style expectation.

## Files to Modify
- `internal/ui/board/board.go`
- `internal/ui/board/board_test.go`
- `internal/app/model.go`
- `internal/app/model_test.go`
- related board/app golden files

## Acceptance Criteria
- [ ] On representative terminal sizes, visible dashboard columns render with equal visual height
- [ ] The dashboard lanes extend to the bottom of the available board area instead of floating near the top
- [ ] The board reads as a coherent full-screen dashboard grid closer to Perles
- [ ] Updated golden/runtime tests verify lane height and bottom alignment behavior
- [ ] Real PTY/runtime review confirms the dashboard no longer has the floating-box look
