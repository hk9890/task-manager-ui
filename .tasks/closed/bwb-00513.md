## Description

Today `bwb --version` only shows a useful version string for tagged releases (goreleaser injects `main.version`). Local builds (`mise run build`, `go build ./cmd/bwb`) show the default. We also lose Commit and Date entirely. Adopt dtctl's pattern: a dedicated `version` package with `Version`, `Commit`, `Date` vars injected via `-ldflags` everywhere ‚Äî local mise build, CI build, and goreleaser.

## Files to Modify

- `cmd/bwb/main.go` ‚Äî currently defines `var version = "dev"` at line 21; used at line 92 (`--version` print) AND line 114 (`BuildVersion: version` passed to logger). Both call sites must switch to read from the new package.
- New file: `internal/version/version.go` ‚Äî package with `Version`, `Commit`, `Date` vars (default `"dev"`/`"unknown"`)
- `.mise.toml` ‚Äî `build` task: add ldflags injecting `git describe`, `git rev-parse --short HEAD`, and UTC date
- `.goreleaser.yaml` ‚Äî update `ldflags:` line to set all three vars (currently only `main.version`)
- Any test that asserts on `--version` output (the output format will change ‚Äî see step 5 below)

## Instructions

1. Create `internal/version/version.go`:
   ```go
   package version

   var (
       Version = "dev"
       Commit  = "unknown"
       Date    = "unknown"
   )
   ```
2. Refactor `cmd/bwb/main.go`:
   - Remove the local `var version = "dev"` at line 21.
   - Replace the `--version` print at line 92 with a read from `internal/version.Version` (or render Version+Commit+Date ‚Äî see step 5).
   - Replace `BuildVersion: version` at line 114 with `BuildVersion: version.Version`.
   - Import the new `internal/version` package (alias as `bwbversion` if it clashes with anything).
3. Update `.mise.toml` `build` task ‚Äî model on dtctl's makefile (lines 3-10):
   ```toml
   [tasks.build]
   description = "Build the bwb binary"
   run = """
   VERSION=$(git describe --tags --always --dirty 2>/dev/null || echo dev)
   COMMIT=$(git rev-parse --short HEAD 2>/dev/null || echo unknown)
   DATE=$(date -u +%Y-%m-%dT%H:%M:%SZ)
   go build -ldflags "-s -w \
     -X github.com/hk9890/beads-workbench/internal/version.Version=$VERSION \
     -X github.com/hk9890/beads-workbench/internal/version.Commit=$COMMIT \
     -X github.com/hk9890/beads-workbench/internal/version.Date=$DATE" \
     ./cmd/bwb
   """
   ```
4. Update `.goreleaser.yaml` `ldflags:` block. Replace the existing single-line ldflags with (model on `../dtctl/.goreleaser.yaml:24-28`):
   ```yaml
   ldflags:
     - -s -w
     - -X github.com/hk9890/beads-workbench/internal/version.Version={{ .Version }}
     - -X github.com/hk9890/beads-workbench/internal/version.Commit={{ .ShortCommit }}
     - -X github.com/hk9890/beads-workbench/internal/version.Date={{ .Date }}
   ```
5. Update the `--version` output to show all three values. Current format: `bwb 0.3.1`. New format: `bwb 0.3.1 (commit abc1234, built 2026-05-16T14:23:00Z)`. **This is a public output format change** ‚Äî search for tests that match on `--version` output (e.g., `grep -rn '"bwb "' cmd/bwb/*_test.go internal/`) and update them.
6. Build locally: `mise run build && ./bwb --version`. Should show all three values populated (not `dev`/`unknown`).
7. (Optional) Verify goreleaser pickup: `goreleaser release --snapshot --clean` ‚Äî requires goreleaser installed locally. If not installed, defer to ug82.9 verification.
