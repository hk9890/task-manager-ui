## Description
Make board, search, and detail consume the available shell body height through one shared layout contract instead of each surface calculating height differently.

## Instructions
1. Inspect shell body sizing in `internal/app/model.go` and identify the current height/width calculations used by board, search, and detail.
2. Introduce a shared workspace sizing contract for the main application body so all primary screens consume the same available height budget.
3. Remove the current inconsistency where dedicated detail is wrapped differently from board/search and ends up using height differently.
4. Preserve header/footer/help/toast/modal behavior while making the main workspace full-height.
5. Board and search keep their mode-specific layouts, but they must satisfy the same shell height/width contract as detail.
6. Add or update tests that lock down the shared body-height contract and prevent regressions in shell layout sizing.

## Acceptance Criteria
- [ ] Board, search, and detail derive body size from one shared shell contract
- [ ] Full available body height is used consistently across the main screens
- [ ] Shell body height is measured explicitly enough in tests to catch regressions in board/detail/search sizing
- [ ] Dedicated detail no longer has a special boxed body wrapper that wastes height relative to board/search
- [ ] Existing overlays/header/footer behavior remains intact
- [ ] Tests pass: go test ./internal/app ./internal/ui/...
