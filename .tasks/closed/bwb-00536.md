## Description
Implement the redesigned issue detail layout described in the parent epic. Rework the main detail renderer so the screen uses a boxed Dependencies pane on the left, a dominant boxed center Content pane, and a grouped metadata/actions rail on the right separated by a vertical divider.

## Instructions
1. Add explicit height handling to the detail renderer state/path so vertical allocation is implementable and testable.
2. Refactor `internal/ui/details/details.go` layout logic so the primary detail view renders:
   - left: a `Dependencies` section box that expands to use the available column height
   - center: one `Content` pane using most of the center height, with issue details/body first and a smaller comments subsection rendered near the bottom of that pane
   - right: grouped metadata plus quick actions with no nested boxes, separated from the center column by a visible vertical rule.
3. Reuse the dashboard section border treatment for the left and center boxes by using the same `styles.FormSection` rounded-border component rather than inventing a new box style.
4. Render dependency issue rows with the same compact visual language as dashboard issue rows. Reuse shared issue-row rendering where practical (`internal/ui/shared/issuerow`) so type/priority/status/id styling stays aligned with the dashboard.
5. Ensure the same structured issue relationship rendering approach is usable for other issue-detail side-panel / structure-data surfaces instead of creating a one-off dependency-only renderer.
6. Replace the current plain-text comment-body rendering with markdown-aware rendering using a reusable shared markdown render component, and use that same component for both the main issue details/body text and comment text.
7. Keep the top summary compact and readable; avoid duplicate standalone ID lines competing with the title.
8. Make the layout actively consume available vertical space instead of producing a sparse upper-half-only composition.
9. Preserve the accepted interaction model: 3 focus panes (`Dependencies`, `Content`, `Metadata`), center-pane scrolling covers both issue details and comments, dependency-row direct interaction is out of scope unless essentially free.

## Acceptance Criteria
- [ ] Detail renderer accepts explicit height input.
- [ ] Dependencies render in a bordered section that visually matches dashboard section borders.
- [ ] Dependency references use shared compact issue-row styling rather than raw `id · title` strings.
- [ ] The same relationship-rendering component/path can support dependency/structure panels elsewhere in detail mode.
- [ ] The center Content pane occupies most of the center-column height and includes comments as a smaller lower subsection.
- [ ] Comments and issue details both use the same reusable markdown-render component/path.
- [ ] Right-side metadata is rendered as grouped text with a vertical separator and includes Quick actions.
- [ ] Top-of-detail summary no longer repeats the issue ID as a separate low-value line.
- [ ] The final layout looks dense and intentional rather than vertically under-filled.
- [ ] The implemented interactions match the explicit 3-pane focus/overflow rules from the epic.

## Files to Modify
- `internal/ui/details/details.go`
- `internal/ui/details/metadata.go`
- `internal/ui/shared/issuerow/row.go` (only if shared rendering helpers need a small extension)
- `internal/ui/shared/markdown/*` (or equivalent shared markdown component surface)
- `internal/app/*` and/or detail-mode state wiring if height/focus plumbing requires it
- any directly-related detail renderer support files/tests required by the redesign
