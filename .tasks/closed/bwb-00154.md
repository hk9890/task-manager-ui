## Description
Discovered while trying to commit all current changes on `main`. The repo-managed pre-commit hook formats staged Go files and then re-stages them, but the restage step fails for files under `cmd/bwb/`. During `git commit`, the hook prints:

- `pre-commit: formatting staged Go files`
- `The following paths are ignored by one of your .gitignore files: cmd/bwb`

This aborts the commit before it can complete.

## Steps to Reproduce
1. Stage a change under `cmd/bwb/main.go`.
2. Run `git commit -m "test"`.
3. Observe the pre-commit hook fail with the ignored-path message for `cmd/bwb`.

## Why
The repository has a `.gitignore` pattern matching `bwb`, which also matches the tracked `cmd/bwb` directory path. The hook's re-stage logic appears to feed `git add` a path shape that trips that ignore rule, so tracked Go files under `cmd/bwb/` cannot be re-staged by the hook.

## Acceptance Criteria
- [ ] The repo-managed pre-commit hook can format and re-stage tracked Go files under `cmd/bwb/`
- [ ] `git commit` no longer fails with the ignored-path message for `cmd/bwb`
- [ ] The fix does not require skipping hooks

## Files to Modify
- `scripts/git-hooks/pre-commit`
- `.gitignore` (only if required)
- related docs/tests if needed
