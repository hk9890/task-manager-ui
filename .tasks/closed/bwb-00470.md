## Description
In `internal/mode/board/model.go`, when the Not Ready column load fails, `notReadyErr` is the first non-nil of `readyExplainErr` and `storedBlockedErr` (around lines 481–484). A `storedBlocked` failure disappears silently if `readyExplain` also failed. This is an artifact of F1 — without the 5th parallel call there'd be only one error source.

**F1 was declined** ([[f568]] closed won't-do, 2026-05-23): the downstream merge in `internal/mode/board/model.go` is permanent. This task therefore proceeds unconditionally — replace the first-non-nil pattern with `errors.Join(readyExplainErr, storedBlockedErr)` so both sources surface when both fail.

Complexity review F5.

## Files to Modify
- `internal/mode/board/model.go` — replace the `notReadyErr` first-non-nil pattern (around lines 481–484) with `errors.Join(readyExplainErr, storedBlockedErr)`. `errors.Join` returns nil when all inputs are nil and returns a single-wrapping error when exactly one is non-nil, so single-source-fail behavior is preserved at the call site (the returned error still wraps the original).
- New or extended test in `internal/mode/board/` (e.g. `model_errors_test.go` or extend an existing test file in this package) — inject failing fakes for both `ReadyExplain` and the storedBlocked `Query`, assert both substrings appear in the surfaced error.
