## Description

Verify the epic outcome (`beads-workbench-0x36`) before closing. Manual end-to-end verification against the real built binary is REQUIRED — type-check + unit tests are necessary but NOT sufficient for runtime UI changes.

You MUST test all three touched surfaces (Board, Search, Detail) against a running app before this task can close. Follow `docs/RUNTIME_UI_VERIFICATION.md` section 2 (built binary + embedded fixture) and the agent-visible PTY recipes in section 2 A–C.

PTY proofs are positive-presence (assert what SHOULD be on screen), not negative-existence (assert what should NOT) — negative proofs are racy on the fast embedded fixture.

If any criterion fails: file a bug (`bd create --type=bug ...`). Do NOT close this task until the bug is closed and re-verified. Do NOT silently fix defects inside this acceptance task.

## Verification Procedure

1. `go test ./...` and `go test -count=10 ./internal/app/...` (the count=10 catches ticker-induced races from `0x36.6`)
2. `go vet ./...`
3. Build: `go build -o /tmp/bwb ./cmd/bwb`
4. Set up the embedded fixture and run the manual checklist (`docs/RUNTIME_UI_VERIFICATION.md` section 3).
5. Run PTY captures per the criteria below using `scripts/capture_bwb_screen.py`; attach each capture's output as a comment on this task via `bd comments add`.
6. If any criterion fails: file a bug, leave this task open until the bug is closed and re-verified.
