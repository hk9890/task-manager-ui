## Description
In dedicated detail mode, the dependency/browser pane currently behaves like an opener: Up/Down only changes the highlighted row and Enter is required to load that issue. The user wants it to behave like search results instead: while browsing the dependency/browser pane, changing the highlighted row should immediately refresh the content and metadata panes for that highlighted issue.

## Instructions
1. Preserve dedicated detail mode layout and left/right pane focus behavior.
2. When the dependency/browser pane is focused, Up/Down should change the highlighted issue and immediately request/load/render that issue in the content and metadata panes without Enter.
3. Keep focus in the dependency/browser pane while browsing so repeated Up/Down continues traversing issues.
4. Keep traversal usable across dependency and parent-group browser items when the loaded issue context changes.
5. Make loading/stale state obvious while the newly highlighted issue is still loading.
6. Enter in the dependency/browser pane should become a no-op because selection already auto-previews the highlighted issue.
7. Update focused detail-mode tests and any help/golden coverage affected by the new browsing semantics.
8. Verify focused detail packages and the full repository test suite.
