In definition.go:41-46, Query uses a tagged-union pattern where Type selects which of ListIssues/ReadyIssues/BlockedIssues is populated. But there is zero enforcement — a caller can set Type=QueryTypeReadyIssues while populating ListIssues, and the consumer won't know until runtime. The built-in sections do this correctly today, but the pattern is fragile for future providers. Suggested action: at minimum, add a Validate() method on Query that checks exactly one payload matches Type. Or consider separate concrete query types behind an interface. The current approach works for 4 built-in sections but will not scale safely. File: internal/dashboard/definition.go:41-46
