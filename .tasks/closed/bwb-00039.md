## Description
Implement full logging for all BWB execution paths, with no special handling that disables logging for non-interactive commands. Non-interactive startup paths such as `--check-config` and `--print-config` should participate in the same logging/debug contract as interactive startup.

## Expected Behavior
When `bwb` is invoked with `--debug`, startup diagnostics are emitted consistently and the persistent JSON Lines log is created when logging is available, regardless of whether the command is interactive or non-interactive.

## Actual Behavior
`cmd/bwb/main.go` currently skips logger construction for non-interactive paths like `--check-config` and `--print-config`, so those commands do not participate in the normal monitoring contract.

## Steps to Reproduce
1. Build the binary: `go build -o /tmp/bwb ./cmd/bwb`
2. Use any valid project directory.
3. Run: `XDG_STATE_HOME=/tmp/bwb-state /tmp/bwb --cwd <repo> --debug --check-config 2> /tmp/bwb-debug-check.log`
4. Observe that current non-interactive commands do not produce the same debug/logging evidence as interactive startup.

## Impact
Monitoring behavior differs by execution path, which makes the logging contract harder to reason about and weakens non-interactive troubleshooting.

## Instructions
1. Remove the special-case skip for logger construction on non-interactive commands.
2. Ensure non-interactive commands (`--check-config`, `--print-config`, and similar startup-only paths) use the same logging initialization and debug mirroring contract as interactive startup.
3. Preserve stdout/stderr behavior for command results while adding the intended diagnostics/log records.
4. Update tests to cover the new behavior explicitly.
5. Update `docs/MONITORING.md` and any other relevant docs to describe the final contract.

## Files to Modify
- cmd/bwb/main.go
- internal/logging/*
- cmd/bwb/*_test.go and/or related startup logging tests
- docs/MONITORING.md

## Acceptance Criteria
- Non-interactive commands initialize logging the same way as interactive startup.
- `--debug --check-config` and `--debug --print-config` emit startup diagnostics and create persistent log records when logging is available.
- Automated tests cover the non-interactive logging behavior.
- Documentation matches the implemented all-path logging contract.
