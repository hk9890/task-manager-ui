## Description
`internal/testing/datasets/datasets_test.go` (build tag `//go:build integration`) contains two tests that claim to verify the env-gated skip path but make no falsifiable assertion. Their own comments admit the limitation, and the comment is also factually wrong about why introspection is hard. They must be made falsifiable.

**Note:** The file's existing comment at `:69–78` claims "We can't easily introspect t.Skipped() on a finished sub-test." This is incorrect — `testing.T.Skipped()` is callable on a captured `*testing.T` even after the sub-test goroutine aborted via `runtime.Goexit` from `t.Skip`. Use the canonical pattern below.

## Files
- `internal/testing/datasets/datasets_test.go`
  - `:48–79` `TestThisRepoSkipsWhenEnvNotSet` (gates on `datasets.EnvParityThisRepo`, calls `datasets.ThisRepo`)
  - `:83–98` `TestExternalSkipsWhenEnvNotSet` (gates on `datasets.EnvParityExternalPath`, calls `datasets.External`)
- `internal/testing/datasets/` (helper file housing `ThisRepo`/`External`) — read-only here; only test code changes.

## Decision: Option 1 only (do not delete)
Option 2 (delete tests, rely on transitive coverage) is **not viable**: `ThisRepo` has no other callers in this file; `External`'s other caller (`TestExternalMtimesUnchanged` around `:185`) itself self-skips on the same env at `:175–178` before reaching `External`, so it provides zero transitive coverage. Deleting would re-introduce the regression class the epic was created to catch. Use Option 1 for both tests.

## Canonical pattern (use this, not the variants in earlier drafts)

```go
func TestThisRepoSkipsWhenEnvNotSet(t *testing.T) {
    if os.Getenv(datasets.EnvParityThisRepo) != "" {
        t.Skipf("%s is set; this test verifies the skip path when it is unset", datasets.EnvParityThisRepo)
    }

    var subT *testing.T
    t.Run("gate-off", func(s *testing.T) {
        subT = s
        datasets.ThisRepo(s) // expected to call s.Skip() and abort this goroutine
        // Unreachable when skip works correctly.
    })

    if !subT.Skipped() {
        t.Fatalf("datasets.ThisRepo did not skip when %s was unset", datasets.EnvParityThisRepo)
    }
}
```

`testing.T.Skipped()` is documented to be safe to call on a sub-test handle after `t.Run` returns. The captured `*testing.T` remains valid across the `runtime.Goexit` that `t.Skip` performs.

Mirror the same shape for `TestExternalSkipsWhenEnvNotSet` against `datasets.External` and `datasets.EnvParityExternalPath`.

Also: while editing, **delete the incorrect comment at `:69–78`** so the next reader doesn't inherit the misconception.
