## Description

Ship the animated braille spinner ticker and its header indicator. Split from the original Foundation ticket because the ticker has known interaction risk with the existing flaky tick scheduler (`beads-workbench-0ny3`); shipping it separately keeps that risk off the critical path for `0x36.2/.3/.4`.

After this task:
- Header renders a braille spinner glyph (frame from `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏`) between "Beads Workbench" and the tab bar whenever `loadingStates()` is non-empty.
- Header is width-invariant across idle/loading — idle renders a 2-cell whitespace placeholder so the tab bar does not shift horizontally.
- New `withSpinnerTickScheduler` test seam exists, parallel to the existing `withRefreshTickScheduler`, so the test suite can disable the spinner ticker.

Does NOT block `0x36.2/.3/.4` (Board/Search/Detail can land their non-blocking-refresh changes without the spinner present — they will simply have no visible refresh signal until this task lands). Blocks `0x36.5` acceptance and `0x36.7` integration test.

## Files to Modify

- `internal/ui/loading/loading.go` — add the braille frame sequence as a pinned constant. Add `NextFrame(prev int) int` and `Glyph(frame int) string` (split API per reviewer — model owns frame state; renderer is pure). Add `tickMsg` and `SpinnerTickCmd(d time.Duration) tea.Cmd`.
- `internal/app/model.go` — hold a spinner frame index on the Model. Wire a `withSpinnerTickScheduler` test seam parallel to the existing `withRefreshTickScheduler` (see `internal/app/model_test.go:3181-3186` for the pattern). Start the ticker from `Init()` via the seam (production = real tick, tests = no-op). Advance frame on `tickMsg`. In `renderHeader()` (lines 776-815), render a fixed 2-cell cell (`Glyph(frame) + " "`) between the title and the first tab when `len(loadingStates()) > 0`; when idle, emit two literal spaces so `lipgloss.Width(headerLeft)` is invariant.
- `internal/app/model_test.go` (or sibling) — tests per the criteria below, including width-invariance assertion and 10x stability run.

## Instructions

1. Pin the spinner glyph sequence as a package-level constant in `internal/ui/loading/loading.go`: `var SpinnerFrames = []rune{'⠋','⠙','⠹','⠸','⠼','⠴','⠦','⠧','⠇','⠏'}`. Expose `NextFrame(prev int) int` (returns `(prev+1) % len(SpinnerFrames)`) and `Glyph(frame int) string` (returns `string(SpinnerFrames[frame % len(SpinnerFrames)])`).
2. Add `tickMsg struct{}` and `SpinnerTickCmd(d time.Duration) tea.Cmd` returning `tea.Tick(d, func(time.Time) tea.Msg { return tickMsg{} })`. Default cadence: 100ms.
3. Add a package-global seam variable in `internal/app` parallel to the existing `scheduleRefreshTickCmd`. Name it `scheduleSpinnerTickCmd`. Default value points at `loading.SpinnerTickCmd`. Tests use `withSpinnerTickScheduler(t, noopTickCmd)` to swap it out — follow the exact pattern in `internal/app/model_test.go:3181-3186`.
4. On `app.Model`, add `spinnerFrame int`. In `Init()`, fire one `scheduleSpinnerTickCmd(100*time.Millisecond)`. On `tickMsg`, advance `spinnerFrame = loading.NextFrame(spinnerFrame)` and re-fire the tick.
5. In `renderHeader()`, between title and first tab: when `len(m.loadingStates()) > 0`, render `loading.Glyph(m.spinnerFrame) + " "` (2 cells). When idle, render `"  "` (2 spaces). The total width of the left segment MUST be the same in both cases. Use a lipgloss style that matches header text styling (e.g. muted/dim).
6. Add a unit test asserting `lipgloss.Width(rendered_header_left(loading=true)) == lipgloss.Width(rendered_header_left(loading=false))`. Add a test asserting one of the 10 pinned spinner glyphs appears in the rendered header when any scope is loading, and that none of them appear when idle.
7. Run `go test -count=10 ./internal/app/...` locally — must be green to confirm the new ticker does not amplify any existing flakes from `beads-workbench-0ny3`.
