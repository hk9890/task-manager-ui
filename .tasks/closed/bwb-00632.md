## Description
Make release and snapshot builds report meaningful version metadata through `bwb --version` instead of always falling back to `dev`.

## Instructions
1. Update build/release configuration so `cmd/bwb` receives version metadata at link time for release and snapshot builds.
2. Keep local developer builds working without ldflags by falling back to `dev` when metadata is absent.
3. Verify the configured build path produces a binary whose `--version` output is not `dev` for release/snapshot-style builds.
4. Keep this task narrowly focused on build metadata wiring and verification; runtime `--version` flag behavior itself belongs to `beads-workbench-s2d1.1`.

## Files to Modify
- `.goreleaser.yaml`
- `cmd/bwb/*.go` (only if needed for version variable plumbing)
- `docs/RELEASING.md` (if build metadata expectations need updating)

## Acceptance Criteria
- [ ] Release/snapshot build wiring passes version metadata into the `bwb` binary
- [ ] Local builds without ldflags still report the documented fallback value
- [ ] A local snapshot-style verification (for example `goreleaser build --snapshot --clean` or equivalent repo-supported check) confirms the built binary reports non-`dev` version output
- [ ] The task does not widen scope into unrelated CLI flag work
