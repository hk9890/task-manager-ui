## Description

The consistency review (2026-05-22) surfaced naming, documentation, and
component-shape inconsistencies. This epic eliminates or explicitly documents
each finding so the codebase has one predictable convention per concern.

**Hard constraint: NO FUNCTIONAL CHANGE.** Every task under this epic is
cleanup/consistency only — file renames, comment additions, documentation, and
signature shape changes that preserve behavior. No runtime behavior, output, or
public-contract semantics may change. `mise run quality` must pass unchanged.

## Findings in scope

1. `internal/gateway/beads/writes.go` uses a bare name while the read side uses
   a `read_` prefix (`read_gateway.go`, `read_payloads.go`).
2. `internal/mode/details/model_teatest.go` is test-only scaffolding compiled
   into the production binary (no `_test.go` suffix, no build tag).
3. `doc.go` exists in 13 of ~31 packages with no predictable rule.
4. `mode/details/Model` diverges from its `mode/board` / `mode/search` siblings
   with no documentation of the intentional difference.
5. Three subprocess-spawn sites with no rationale comments.
6. `mode/board.NewModel` uses a variadic for a 0-or-1 argument.

## Success Criteria

- Each finding is resolved or explicitly documented as a deliberate exception.
- No functional change: build, tests, and `mise run quality` pass; runtime
  behavior is identical.
- Acceptance-review task confirms all of the above.
