## Description
When no search result is selected (no-match query, or before selecting a result), the
Content preview pane renders the dashboard-styled meta row for an EMPTY placeholder
summary, producing the meaningless line "? P0 (NO (none)" above "No selected result.",
plus the thin rule. The Metadata pane likewise shows Type:(unknown) Status:(none) Priority:(unknown).

## Repro
1. Enter Search; run a no-match query (or observe before selecting any result).
2. Look at the Content + Metadata preview panes.
Observed: meta row "? P0 (NO (none)" + "No selected result." + a thin rule.

## Root cause (verified)
internal/ui/details/details.go: contentHeaderMetaRow (added in kxon) renders type/priority/
status/id tokens even for the empty placeholder built in internal/ui/search/search.go
(~lines 168-177): {ID:"(none)", Status:"(none)", Priority:-1, Type:""}. The fallbacks then
produce CompactIssueType("")="?", CompactPriority(-1)="P0", CompactIssueState("(none)")="(NO", id="(none)".

## Instructions
1. Pin the placeholder predicate. Preferred: have internal/ui/search/search.go pass an explicit placeholder/empty flag into the detail State (cleaner than sniffing sentinel strings). Acceptable fallback: treat as placeholder when summary.ID == "(none)" AND summary.Type == "" (the exact sentinel set in search.go ~168-177). Choose the explicit-flag approach if feasible.
2. In renderContentPaneLines / contentHeaderMetaRow (internal/ui/details/details.go), for the placeholder case ONLY, suppress the meta row AND the thin rule, rendering just the placeholder title line (e.g. "No selected result.").
3. Optionally suppress the Metadata "Core" noise (Type:(unknown)/Status:(none)/Priority:(unknown)) for the placeholder.
4. Leave the normal header (meta row + title + rule) for real issues unchanged.
