## Description
The /test-review verdict on branch perf/bd-latency-cap-and-cache identified two concrete gaps where the suite gives false confidence. This epic closes both before they regress.

## Findings (from /test-review on 2026-05-23)

### Gap A — Mutation error paths are unverified (Q4 + Q7)
`internal/app/model.go` wraps every write-side gateway call in an error branch that emits a user-visible toast:
- `:1732` UpdateIssue (full edit) — "update issue failed: %w"
- `:1739` CloseIssue — "close issue failed: %w"
- `:1748` AddComment — "add comment failed: %w"
- `:1764` UpdateIssue (status-only) — "update status failed: %w"
- `:1783` UpdateIssue (priority-only) — "update priority failed: %w"

`FakeBeadsGateway.SetError(method, err)` is wired for all 16 gateway methods, but the suite injects errors only for `MethodSearchIssues` and `MethodHealthCheck`. None of the five toast strings above are asserted anywhere in the test corpus. A regression that swallows a mutation error, mislabels the toast, or drops the error branch entirely produces zero test failures.

### Gap B — Two skip tests are structurally incapable of failing (Q5 + Q8)
`internal/testing/datasets/datasets_test.go`:
- `:48–79` `TestThisRepoSkipsWhenEnvNotSet`
- `:83–98` `TestExternalSkipsWhenEnvNotSet`

Both declare a `skipped` bool that is never written, and the file's own comment says: "This test passing without panic confirms the skip path is safe." That is not a falsifiable assertion — these tests pass regardless of whether the helpers under test actually skip.

## Goals
- Five new assertions catch regressions on each of the five mutation error branches.
- The two unfalsifiable skip tests are either replaced with falsifiable assertions or removed.
- `mise run test` continues to pass; no other tests regress.
