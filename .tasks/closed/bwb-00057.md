Discovered from user review after epic beads-workbench-90w was closed. Expected: the Dependencies pane should use a somewhat larger share of width so dependency rows and structure data are comfortably readable beside Content, but not as large as one-third. User decision: target roughly 25% of available width (subject to sensible min/max bounds) instead of the current narrow near-fixed 22..28 column pane. Actual: detail layout currently uses a narrow left pane derived from `splitThreePaneWidths` with `left = clamp(available/5, 22..28)`, which makes Dependencies feel too cramped. Repro: open the real app detail view at 120x34 or 160x34 and compare Dependencies width vs Content width; dependency titles truncate aggressively. Impact: relationship browsing/readability is weaker than intended and panel balance feels off.

## Acceptance Criteria
- [ ] Dependencies width uses an updated formula targeting roughly 25% of the available pane width rather than the current near-fixed narrow rail.
- [ ] The pane remains bounded with sensible min/max values so narrow terminals still render acceptably.
- [ ] Real-app review at 120x34 and 160x34 shows materially better dependency readability without starving Content.
