## Outcome

Unit test asserts that mutating one cached issue does not bust unrelated cached issues. The implementation already does this correctly via `memory.Forget(id)`, but the test was missing.

## Files (modify)
- `internal/repository/caching/caching_test.go` — add `TestUpdateIssue_DoesNotBustUnrelatedIssue`

## Test design
```go
func TestUpdateIssue_DoesNotBustUnrelatedIssue(t *testing.T) {
    backing := newCountingFakeRepo() // increments per-method call counts
    backing.seed("X", ...)  // seed both X and Y in backing
    backing.seed("Y", ...)
    cache := caching.New(backing)
    
    // Prime the cache with both X and Y
    _, _ = cache.Issue(ctx, "X")  // backing.Issue called once
    _, _ = cache.Issue(ctx, "Y")  // backing.Issue called once
    assert backing.IssueCalls == 2
    
    // Update X
    err := cache.UpdateIssue(ctx, "X", input)
    require.NoError(t, err)
    assert backing.UpdateIssueCalls == 1
    
    // Read X again — should hit backing (was busted)
    _, _ = cache.Issue(ctx, "X")
    assert backing.IssueCalls == 3
    
    // Read Y — should still hit cache (NOT busted)
    _, _ = cache.Issue(ctx, "Y")
    assert backing.IssueCalls == 3  // unchanged
}
```

Similarly add `TestCloseIssue_DoesNotBustUnrelatedIssue` and `TestAddComment_DoesNotBustUnrelatedIssue`.
