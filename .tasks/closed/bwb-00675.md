Acceptance verification found that the shipped left related-issues rail does not surface actual related links from the real bd CLI. In a seeded fixture repo, running 'bd link bwf-2 bwf-3 --type related' followed by 'bd show bwf-2 --json' produced no top-level 'related' array; instead the related link appeared inside 'dependencies' with dependency_type='related'. The current gateway show parser only reads primary['related'], so IssueDetail.Related stays empty and the detail rail/inline related section cannot render real related links.
