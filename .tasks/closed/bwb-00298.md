In load.go:276-292, syncEditorLauncher creates a full deep copy of every launcher definition (cloning Args, Env slices) just to set the editor launcher's Command field. This runs on every config load. The merge() function already copies definitions. syncEditorLauncher could mutate in place after merge has created the working copy, avoiding redundant allocations. Alternatively, the editor-launcher sync logic could be folded into merge() itself since they always run together (line 214 immediately after editor override). Suggested action: move editor-command sync into merge() and operate on the already-copied slice. File: internal/config/load.go:276-292, 214
