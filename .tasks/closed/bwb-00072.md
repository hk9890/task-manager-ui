## Description
Refresh the app shell chrome so the board becomes the focal point instead of a stack of persistent help lines.

## Goals
- Replace the tall multi-line header with a single-line app bar.
- Treat Board and Search as the only top-level tabs; Detail is a contextual view, not a tab.
- Move mode-specific help into the footer.
- Improve board issue scanability with compact color cues for type, priority, and status inspired by Perles.

## Decisions
- Approved header direction: app name + tabs + right-side context/status on a single line.
- Header tabs are only `Board` and `Search`.
- When detail is open, keep the browse tabs and use the right-side context area to show selected-issue/detail state instead of a Detail tab.
- Footer owns concise, mode-specific key hints.
- Board issue coloring should adapt small donor-style badge ideas from `/home/hans/dev/github/perles` without importing donor-only runtime/orchestration code.

## Success Criteria
- [ ] Shell header renders as a single line with only Board and Search tabs.
- [ ] Footer renders concise mode-specific help instead of persistent multi-line header help.
- [ ] Detail mode no longer appears as a top-level tab and still preserves clear selected-issue context.
- [ ] Board issue rows gain readable color hierarchy for type/priority/status metadata while preserving compact truncation behavior.
- [ ] Updated UI tests/goldens cover header/footer and board row rendering.
- [ ] `go test ./...`, `go build ./cmd/bwb`, and `go vet ./...` pass.
