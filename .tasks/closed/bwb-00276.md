service_test.go:11-31 defines a local `recordingRunner` that is functionally identical to `fakes.FakeProcessRunner` (same fields, same recording behavior, same defensive copy). The shared fakes package exists precisely for this purpose and is already documented in TESTING.md as the required test double.

This violates the project's 'shared deterministic seams live in internal/testing/fakes' convention. It also means any bug fix to the fake's recording behavior would not apply to these tests.

Suggested fix: replace all uses of the local `recordingRunner` in service_test.go with `fakes.FakeProcessRunner`, and change the test to `package launcher_test` (external test package, consistent with editor/service_test.go which already does this).
