## Description
The launcher framework exists under `internal/launcher/`, but the active app runtime path does not invoke it. The shell's launcher entrypoint still routes through `services.Editor.EditIssue()` instead of `services.Launcher.Launch()`, so launcher definitions are unreachable from issue context in the running app. This also causes the launcher-focused app tests to hang because they expect the fake launcher seam to be called but the runtime tries to open a real editor instead.

## Acceptance Criteria
- [ ] Pressing the launcher entrypoint in the shell invokes `launcher.Service` with the selected issue context
- [ ] Launcher failures surface clear UI feedback from `launcher.Service`
- [ ] `TestModelLaunchEditorActionUsesLauncherService` passes without timeout
- [ ] `TestModelLaunchEditorActionShowsErrorToastWhenLauncherFails` passes without timeout
- [ ] Launcher-focused app tests do not require a real interactive editor process
