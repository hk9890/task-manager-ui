## Description
The dashboard rewrite (epic beads-workbench-au54, specifically beads-workbench-lgln) makes the board model call the gateway directly, which removes the runtime purpose of `internal/dashboard/builtin_provider.go` and the `Provider` / `Definition` / `Section` / `Query` types in `internal/dashboard/definition.go`. But `docs/CODING.md:114,128,154` and `project-plan/ARCHITECTURE.md:57-60,228-255` explicitly describe this abstraction as the forward-looking entrypoint for a future file-backed dashboard provider.

## Decision (LOCKED)
**Option (a) — Keep `BuiltInProvider` as a metadata-only catalog.** Strip the `Query` payload from `Section`; keep IDs + titles as static data. Board reads section metadata for column titles and applies its own gateway calls. Future file-backed providers can still ship IDs + titles (and optionally per-section query expressions) without re-introducing the runtime coupling.

Rationale: minimal-diff path that preserves the documented v1 architecture direction without committing to anything concrete; aligns the docs with reality at low cost.

## Instructions
1. In `internal/dashboard/definition.go`:
   - Remove the `Query` field from `Section` (and the `QueryType`/`Query` types and `BlockedIssuesQuery`/`ReadyIssuesQuery` reaches if they're only used here).
   - Keep `Provider` interface, `Definition`, `Section{ID, Title}`, `ValidateDefinitions` — but validation now only checks IDs/titles + non-empty sections (not query payloads).
   - Update `ValidateQuery` references: either remove the function if no longer needed, or rescope.
2. In `internal/dashboard/builtin_provider.go`:
   - Collapse the four `*Section()` helpers to a static slice of `Section{ID, Title}` (no query info). Keep the same IDs and titles currently used: `not_ready`/"Not Ready", `ready`/"Ready", `in_progress`/"In Progress", `done`/"Done".
   - `Dashboards()` still returns one definition.
3. Update `internal/dashboard/builtin_provider_test.go` and `definition_test.go`:
   - Drop assertions that check query types/payloads.
   - Keep assertions on IDs, titles, and validation behavior (empty title rejected, empty sections rejected, etc.).
4. Update the docs in the same PR:
   - `docs/CODING.md` — lines 114, 128, 154 describe the provider; rewrite to reflect: provider returns IDs + titles only; board owns gateway query routing; future file-backed providers may ship section IDs + titles.
   - `project-plan/ARCHITECTURE.md` — lines 57-60 (architectural bounds), 106-108 (gateway interface — add `Query` and `ReadyExplain`), 228-255 (dashboard section — describe new data flow). Mention that the dashboard board model owns the query fan-out (3 parallel reads) and that the composer (`internal/dashboard/composer.go`) buckets results.
5. Do NOT touch `internal/mode/board/model.go` — that's lgln's job. This task only delivers the metadata-only definition and doc updates.
