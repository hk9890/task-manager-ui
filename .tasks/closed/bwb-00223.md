## Description
Two test functions carry doc comments claiming they are skipped pending a fix
("Remove the t.Skip below when…"), but neither contains a `t.Skip` — the
underlying bugs (znri.5, znri.7) were fixed and both tests now run as live
regression guards. The comments are misinformation: a reader could "clean up"
what looks like a dead skipped test and silently delete live coverage.

Surfaced by the project-review-test adversarial review (Finding 1).

## Affected files
- `internal/app/help_modal_render_test.go:21-22` (TestHelpModal_RenderedFrameHasClosingBottomBorder)
- `internal/app/fatal_error_quit_test.go:75-77` (TestFatalErrorScreen_CtrlQQuits)

## Instructions
1. In each file, remove only the stale "SKIPPED: … Remove the t.Skip below when…"
   sentences from the doc comment. Keep the rest of the comment, which correctly
   documents what the test pins.
2. Optionally reword the surviving comment so it reads as a live regression guard
   (e.g. "Regression guard for znri.5 …") rather than a pending-fix note.
3. Do NOT change any test logic — there is no t.Skip to remove; the bodies already run.

## Acceptance Criteria
- [ ] No "Remove the t.Skip" or "SKIPPED" wording remains in either file
- [ ] Both tests still run and pass (`go test ./internal/app/...`)
- [ ] Surviving comments accurately describe the test as a live regression guard
- [ ] No test logic changed
