## Description
Add the runtime-only CLI flags that change how BWB boots the TUI: repo targeting, lightweight debug diagnostics, and auto-refresh suppression.

## Instructions
1. Add `--cwd <path>` and use it as the effective project root / gateway working directory for startup instead of assuming the caller's current directory.
2. `--cwd` must not call `os.Chdir`; it sets the resolved project root and the beads runner working directory only.
3. Resolve a relative `--config <path>` against the process start cwd, not the `--cwd` target.
4. A non-existent or non-directory `--cwd` exits non-zero before TUI startup.
5. Ensure the resolved target path also feeds launcher/project-root context that already depends on startup working directory semantics; launcher definitions with no explicit `WorkDir` inherit the resolved `--cwd` project root.
6. Add `-d/--debug` as a lightweight stderr diagnostics mode; every debug line must be prefixed with `[bwb-debug]` and cover: resolved config path, resolved cwd, auto-refresh enabled/disabled state, and one line per `bd` invocation with argv plus exit code.
7. Add `--no-auto-refresh` so the periodic refresh tick is not scheduled and the focus-regain branch suppresses only the auto-refresh side effect; any non-refresh focus handling and manual refresh keybindings remain unchanged.
8. Thread these runtime options through the smallest necessary startup/app/gateway seams and keep default behavior unchanged when flags are absent.
9. Add or update tests that prove `--cwd`, `--debug`, and `--no-auto-refresh` affect only the intended paths.

## Files to Modify
- `cmd/bwb/main.go`
- `cmd/bwb/*.go` (new CLI/bootstrap helpers and tests as needed)

## Acceptance Criteria
- [ ] `--cwd <path>` changes the effective repo target without requiring the caller to `cd` first
- [ ] `--cwd` does not call `os.Chdir`; it sets project root and runner working directory in-memory only
- [ ] Relative `--config <path>` resolution remains anchored to the process start cwd, not the `--cwd` target
- [ ] A missing or non-directory `--cwd` exits non-zero before TUI startup
- [ ] Launcher/project-root behavior uses the same resolved target path, including default launcher workdir behavior
- [ ] `--debug` emits only the documented `[bwb-debug]` diagnostics and only when enabled
- [ ] `--no-auto-refresh` disables the periodic tick and focus-regain auto-refresh side effect while leaving default behavior unchanged without the flag
- [ ] Manual refresh behavior remains available when `--no-auto-refresh` is set
- [ ] Tests cover the runtime flag plumbing and regressions
