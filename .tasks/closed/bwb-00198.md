styles/colors.go:159 has normalizeIssueToken() which is identical to renderhelpers.NormalizeToken() (helpers.go:121). Same logic, same body, two copies. The styles package function is unexported but does the exact same thing. This creates a maintenance risk: if token normalization rules change, both must be updated. Suggested action: have styles/colors.go call renderhelpers.NormalizeToken() instead of maintaining its own copy, or move the canonical normalizer into styles since that's the lower-level package (renderhelpers already imports styles).
