## Description
Track delivery of the full Beads Workbench v1 standalone product described in `project-plan/`. This product-level epic covers the backend foundation, the Bubble Tea app shell, the browsing/search/detail experience, the editor/launcher workflows, and the cross-cutting testing/documentation work needed for the first complete standalone release.

## Goals
- Complete the standalone repository and official-beads gateway foundation
- Build a minimal Bubble Tea app shell with shared services and UI primitives
- Deliver built-in dashboards, search, and issue detail browsing without BQL
- Deliver create/update/close/comment flows, external editor handoff, and external launchers
- Establish automated TUI, integration, and end-to-end test harnesses that agents can run without manual UI verification
- Keep project docs (`CODING.md`, `TESTING.md`, `AGENTS.md`) aligned with the implemented product
- Validate the integrated product in an embedded-mode beads repository

## Success Criteria
- [ ] The gateway foundation epic is complete and accepted
- [ ] The app shell epic is complete and accepted
- [ ] The browsing experience epic is complete and accepted
- [ ] The editor and launcher epic is complete and accepted
- [ ] Cross-cutting automated TUI/e2e test harness work is complete
- [ ] Built-in dashboards cover ready, in-progress, blocked, and recent work
- [ ] Search supports text and structured filters without BQL
- [ ] Issue details show metadata, content, comments, and dependency context from official surfaces
- [ ] Users can create, update, close, and comment on issues from the product
- [ ] External editor handoff works for rich issue editing
- [ ] External launchers work with issue-context interpolation
- [ ] Backend, UI, integration, and end-to-end tests pass
- [ ] Project docs reflect the implemented product and testing strategy
- [ ] No direct SQL, BQL, or orchestration subsystem is required in the active product path
