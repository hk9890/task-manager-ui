## Description
After fixing startup selection synchronization, real PTY/runtime checks still show the initial selected issue can jump between columns as dashboard sections load asynchronously. The footer/detail selection is now synchronized once visible content appears, but the active startup focus can briefly land on a later-loading non-empty column (for example Blocked or In Progress) before settling on the earliest intended board column.

## Evidence
- Real wide-terminal PTY startup capture after fixing beads-workbench-3dx
- Observed startup selection first landing on `beads-workbench-yze.4.5` in Blocked, then later moving to Ready content once that section finished loading

## Instructions
1. Make startup board focus deterministic during async section loading.
2. Prefer stable dashboard-order startup focus rather than transient jumps based on load arrival timing.
3. Add automated coverage for startup focus stability.
4. Re-check startup runtime in a PTY after the fix.
