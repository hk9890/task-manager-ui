## Files (new)

- `internal/repository/memory/repository_test.go` -- unit tests covering filter / sort / dep resolution / mutation semantics

## Storage
```go
type Repository struct {
    mu       sync.RWMutex
    issues   map[string]*storedIssue
    catalogs Catalogs  // optionally seeded; defaults to bd-standard values
    clock    func() time.Time
    idgen    func() string
}
```

`storedIssue` holds all fields needed to satisfy the contract: ID, Title, Status, Priority, Type, Assignee, Labels, Description, Notes, DependsOn []string, Comments []Comment, Created/Updated/Closed time.

## Semantics to implement correctly

- `Dashboard()` -- compute ReadyExplain (ready = open AND all DependsOn closed; blocked = open AND any DependsOn not closed), InProgress, Closed (sorted by Closed DESC, limited), ClosedTotal, Blocked (status=blocked)
- `Issue(id)` -- include resolved dep summaries + comments
- `Search(q)` -- case-insensitive substring across Title + Description + Notes; SearchResultPage with Completeness=complete
- `CreateIssue` -- generate ID, store, default status=open, set Created/Updated
- `UpdateIssue` -- merge fields, bump Updated
- `CloseIssue` -- set status=closed, Closed timestamp
- `AddComment` -- append with author + timestamp
- Catalogs returned as seeded values; default to bd-standard if not seeded
