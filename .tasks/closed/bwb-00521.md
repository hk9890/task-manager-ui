## Description
Rewrite the board mode's data layer to use the new gateway primitives + composer, replacing the current per-section + per-count fan-out (`internal/mode/board/model.go:174-179, 199-202, 599-624`). This is the user-visible perf payoff for epic beads-workbench-au54: startup goes from ~3 s serial to ~700 ms parallel.

## Depends on
- beads-workbench-sl8c (Gateway Query method)
- beads-workbench-nbmx (Gateway ReadyExplain method)
- beads-workbench-tj4m (Dashboard composer)
- The two newly filed dependency tickets: gateway test seam (for subprocess assertions) and BuiltInProvider fate decision (resolves what happens to `internal/dashboard/builtin_provider.go` and the docs that reference it).

## Loading UX decision (locked during review)
**All-or-nothing**: the board shows a single "⏳ Loading dashboard…" indicator until ALL 3 reads land. Today's "Loading (n / m sections)" progressive indicator is removed. Worst case latency to first paint = the slowest of the 3 reads (~720 ms for `bd query closed`). Streaming/per-column rendering was considered and rejected for this milestone to keep the rewrite scoped; revisit only if the all-or-nothing UX shows measurable user pain.

## Instructions
1. Replace the dashboards-then-sections-then-counts pipeline with a single fan-out on board `Init`:
   - Cmd A: `Gateway.ReadyExplain({Limit: 0})` — uncapped (active set expected <100).
   - Cmd B: `Gateway.Query("status=in_progress", {Limit: 0})` — uncapped.
   - Cmd C: `Gateway.Query("status=closed", {IncludeClosed: true, SortBy: SortFieldClosedAt, SortOrder: SortDirectionDescending, Limit: closedLimit()})`.
2. `closedLimit()` definition: `max(50, sectionItemCapacity())`. Pre-`WindowSizeMsg` default of `sectionItemCapacity() = 20` means `closedLimit() = 50`. Keep this in one helper so the floor is consistent and documented.
3. Each cmd returns its own typed message (`readyExplainLoadedMsg`, `inProgressLoadedMsg`, `closedLoadedMsg`). The model accumulates results and only invokes `dashboard.Compose(...)` once all 3 have arrived (or one has errored — surface a single load error and abandon composition like today). Render the loading indicator until composition runs.
4. **uiboard.Column field migration (explicit)**: today `internal/ui/board/board.go` `Column` struct has `TotalCount int` + `CountLoaded bool` (rendered at `board.go:77-79`). The new world has `dashboard.ColumnData{Total int, TotalIsExact bool}`. Pick ONE migration:
   - (a) Map at the model→renderer boundary in `View()`: `Column{TotalCount: data.Total, CountLoaded: true}`, and drop `TotalIsExact` from the renderer entirely (the "N+" indicator becomes a model concern that appends "+" to the rendered total string before passing).
   - (b) Change `uiboard.Column` to carry `Total int; TotalIsExact bool` and render `fmt.Sprintf("%d%s", total, plus)` where `plus = ""` when exact else "+".
   - Recommend (b): the renderer's job is to render, not to format business signals — but the task can pick (a) if it shortens the diff. Document the choice in the PR description.
5. Drop all of: `sectionLoadedMsg`, `countIssuesLoadedMsg`, `countReadyLoadedMsg`, `countBlockedLoadedMsg` types and their handlers. Drop `sectionCounts`, `readyCount`, `blockedCount`, `countsLoaded`, `readyCountLoaded`, `blockedCountLoaded`, `pendingLoads`, `sectionQueries`, `dispatchCountCmds`, `invalidateCounts`. Drop `sectionItemCapacity` only if no longer referenced (Done capacity now lives in `closedLimit()`).
6. BuiltInProvider / dashboard.Definition: do NOT touch in this task. The separate provider-fate ticket owns that decision. This task wires `board.Model` to call the gateway directly; the section titles ("Not Ready", "Ready", "In Progress", "Done") can be defined as local constants in the board package or imported from wherever the provider-fate ticket lands. Coordinate with whichever ticket closes first.
7. Composer logger handoff: `dashboard.Compose` is pure and returns `Warnings []CardinalityWarning`. In `lgln`, after composition the board model passes the warnings to the runtime logger (`logging.Component("dashboard")`). One slog warn line per warning entry: `component=dashboard, group=<g>, count=<n>, threshold=<t>`.
8. **Unify manual reload + auto-refresh** (currently divergent at `board/model.go:265-273` and `model.go:357-368`). Move anchor capture to a single helper `func (m *Model) startReload(mode refreshMode) tea.Cmd` used by both. Selection anchor must be captured BEFORE clearing state, in both paths.
9. Update `internal/mode/board/model_test.go` to drive the new gateway interface via the in-memory fake. Add at least one scenario covering: all-empty load, all-three-groups populated, ReadyExplain error path, Query in_progress error path.
