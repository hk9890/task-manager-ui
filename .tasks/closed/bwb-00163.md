## Summary

After task beads-workbench-3lhw added a permanent 2-char selection gutter to the metadata pane, the search preview pane renders the metadata with the new gutter (e.g. `  Core`, `  Type    : task`), but the golden files for the search package still expect the old format without the gutter.

## Failing tests

```
go test ./internal/ui/search/...
--- FAIL: TestRenderGoldens/results_with_preview_w120
--- FAIL: TestRenderGoldens/results_narrow_w80
--- FAIL: TestRenderGoldens/default_all_results_w120
```

## Actual vs Expected

Actual output (correct new behavior):
- `  Core` (2-char idle gutter before group title)
- `  Type    : task` (2-char idle gutter before field)
- `  -----...` (2-char idle gutter before divider)

Golden files still expect:
- `Core` (no gutter)
- `Type    : task` (no gutter)
- `-------...` (divider at col 0)

## Repro

```bash
go test ./internal/ui/search/... -run TestRenderGoldens -v
```

## Fix

Update the three failing golden files in `internal/ui/search/testdata/` to match the new 2-char gutter format:
- `search_results_preview_w120.golden`
- `search_results_narrow_w80.golden`
- `search_default_all_results_w120.golden`

Or regenerate them with `UPDATE_GOLDEN=1 go test ./internal/ui/search/...`.

## Acceptance Criteria
- [ ] `go test ./internal/ui/search/...` passes with no failures
- [ ] The updated goldens show the 2-char gutter (`  `) on all metadata lines in the search preview panel
