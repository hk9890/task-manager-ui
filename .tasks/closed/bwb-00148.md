## Description

The Done section in the dashboard currently sorts by `updated_at`. An issue that receives a comment or update after being closed incorrectly floats to the top. The correct field is `closed_at`, which `bd list --sort closed` already supports.

**Rationale:** `closed_at` records when the work was completed; `updated_at` fires on any subsequent edit (comments, description tweaks). For a "recently completed" view, `closed_at` is the semantically correct field.

**Usage constraint:** `SortFieldClosedAt` should only be used in queries that filter to `status=closed`. Issues without a `closed_at` value (open/in-progress) will sort to an implementation-defined end of the result; this is not tested behavior and should not be relied upon.

## Instructions

1. In `internal/domain/queries.go` add:
   ```go
   // SortFieldClosedAt sorts by issue close date. Use only with status=closed filters;
   // behavior for non-closed issues is undefined.
   SortFieldClosedAt SortField = "closed_at"
   ```

2. In `internal/gateway/beads/read_gateway.go`, add a case to `mapListSortField()`:
   ```go
   case domain.SortFieldClosedAt:
       return "closed"
   ```

3. In `internal/dashboard/builtin_provider.go`, update `doneSection()`:
   ```go
   SortBy: domain.SortFieldClosedAt,
   ```
