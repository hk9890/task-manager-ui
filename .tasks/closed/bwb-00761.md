## Description
Implement the wide-screen detail renderer for the approved left related-issues rail.

## Instructions
1. Add a dedicated 3-column detail layout that activates only on wide terminals at about 140 columns or wider.
2. Keep the left rail compact at roughly 20 percent of available width, clamped to 22-28 columns, and keep the existing metadata rail at 34 columns.
3. Render the left rail as always-expanded grouped lists in this order: Blocked by, Blocks, Related. Render only non-empty groups.
4. Reuse the shared rail row renderer rather than duplicating row formatting logic inside the detail view.
5. Focus-pane styling in the renderer must be input-driven only. Default visual focus to the content pane until the navigation task wires active pane movement.
6. When the left rail is active in wide layout, suppress the duplicate inline Related Work section from the main content column. Continue showing inline related-work sections in fallback layouts below the wide breakpoint.
7. Preserve current two-column and single-column behavior below the wide-screen breakpoint.
8. Keep v1 scope explicit: no tree presentation, no nesting, no children group.
9. Add or update golden tests for wide-screen left-rail rendering and below-breakpoint fallback behavior.
