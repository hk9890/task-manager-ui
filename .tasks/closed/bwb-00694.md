## Description

The scale fixture is built for the express purpose of exercising edge cases that the 3-issue minimal fixture cannot. Once it lands (via [[faif.4]]), the full test suite must run against it — and any newly-surfaced bugs must be filed per the established discipline.

## Instructions

1. Run `mise run test:integration` with the scale fixture as default for parity and contract tests. Triage every failure:
   - **Real bug surfaced by scale data**: file via discipline (`bd create --type=bug --title="Discovered by scale fixture: ..."`), link back to this task, include verification requirement.
   - **Fake fidelity gap surfaced**: fix the fake per existing fake-update discipline (`internal/testing/fakes/doc.go`); add drift log entry.
   - **Test that incorrectly assumed 3-issue scale**: fix the test (probably needs minimal-fixture-only annotation, or rewrite assertions to be cap-aware).
   - **bd quirk only visible at scale**: update interface.go contract; add a new "bd quirks observed at scale" appendix if needed.
2. Run `mise run smoke -- --dir <scale-fixture-path>` if the smoke binary supports it (per [[faif.4]] step 6). Triage failures the same way.
3. Run the full `RunReadContract` (and `RunWriteContract` from 9x70 if landed) against the scale fixture. The fixture-agnostic invariants from 8qw9 should pass cleanly; if any don't, the invariant logic itself may have implicit small-N assumptions to fix.
4. Document material findings:
   - Bugs filed: list each with ID and one-line description.
   - Fake updates: append to drift log.
   - Test changes: note which tests now opt-in to minimal-only.
5. Goal: by task close, every existing test passes against the scale fixture (or is explicitly annotated as minimal-only with rationale).

## Acceptance Criteria

- [ ] `mise run test:integration` passes with scale fixture as the parity-test default
- [ ] `mise run smoke` against scale fixture passes
- [ ] `RunReadContract` (+ `RunWriteContract` if available) passes against scale fixture for both fake AND real implementations
- [ ] Every newly-discovered bug filed per discipline and linked back to this task
- [ ] Fake updates (if any) reflected in drift log
- [ ] Tests opting for minimal-fixture-only have an inline comment explaining why
