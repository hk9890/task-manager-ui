writes.go: UpdateIssue (line 72), CloseIssue (line 119), and AddComment (line 140) all accept issueID string but don't validate it's non-empty. ShowIssue (read_gateway.go line 142) correctly validates with ErrorCodeValidationFailed. The write methods will pass an empty string to bd CLI and rely on the CLI to reject it, which produces a less informative error (ErrorCodeCommandFailed instead of ErrorCodeValidationFailed). Suggested action: add the same TrimSpace+validation check from ShowIssue to UpdateIssue, CloseIssue, and AddComment.
