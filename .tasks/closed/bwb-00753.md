## Outcome

The CachingRepository (from kxci.2) gains a background goroutine that polls `beads.VCStatusHash` on a configurable interval (default 60s). When the hash changes, the Dashboard cache is invalidated (next read refetches from backing).

## Files (modified)
- `internal/repository/caching/caching.go` — add `Start(ctx)` and `Stop()` methods + tick goroutine
- `internal/repository/caching/options.go` — `WithRefreshInterval(d time.Duration)`, `WithVCStatusFunc(func(ctx) (string, error))` (test seam)
- `internal/repository/caching/caching_test.go` — tick tests

## Lifecycle
```go
// Start launches the background refresh goroutine. Idempotent.
// The goroutine exits when ctx is cancelled OR Stop() is called.
func (c *CachingRepository) Start(ctx context.Context)
// Stop signals the background goroutine to exit. Blocks until it does.
func (c *CachingRepository) Stop()
```

## Tick body
```
1. lastHash := current cached hash (under mu lock)
2. h, err := c.vcStatusFunc(ctx)
3. if err != nil: log warning, keep lastHash, return
4. if h != lastHash: lock mu, mark Dashboard dirty, drop ALL cached Issue(id) entries (we don't know which changed), update lastHash, unlock
```

NOTE: We invalidate ALL per-ID Issue cache entries on hash change because we can't tell which specific issues changed without a more expensive `bd query updated>X` poll (option b, deferred). This is a conservative correctness trade.

## Test seam
- `WithVCStatusFunc` lets tests inject a deterministic hash provider
- Tests use a fake clock + manual tick trigger (NOT real timers) where possible. If you need a timer, use a fast interval (~10ms) with explicit `time.Sleep` and accept some flake risk; or expose a `triggerRefresh()` test-only method behind a build tag.

## Test coverage
- Tick with unchanged hash: no invalidation
- Tick with changed hash: Dashboard refetched on next read; per-ID Issue cache cleared
- vcStatusFunc error: tick logs warning, no state change, lastHash preserved
- Start/Stop lifecycle: Start launches goroutine, Stop blocks until exit, second Start is no-op or re-launches cleanly
- Context cancellation: ctx.Done in Start → goroutine exits

## Wiring (deferred to kxci.5)
- kxci.5 will call `Start(ctx)` from `cmd/bwb/main.go` startup and `Stop()` in shutdown.

## Acceptance criteria
- [ ] Start/Stop work; race-clean

- [ ] Hash unchanged → no invalidation (verified by counting backing calls)
- [ ] vcStatusFunc errors don't crash or corrupt cache state
