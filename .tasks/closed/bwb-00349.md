## Description

Hands-on audit of real `bd` CLI behavior, with findings recorded directly in `internal/gateway/beads/interface.go` as per-method contract sections plus a "bd quirks observed" appendix. The output of this task IS the canonical gateway contract — no separate report.

## Instructions

1. Set up a temp DB for controlled experiments:
   ```bash
   tmp=$(mktemp -d) && cd "$tmp" && bd init && BD_NON_INTERACTIVE=1 bd config set ...  # (as needed)
   ```
   Also use the embedded fixture (`internal/testing/e2e/embeddedfixture/seed.json` recipe) and this repo's `.beads/` as reference datasets. NEVER touch `/home/hans/dev/github/ai-config-manager/.beads/` (or any other external repo's DB) without `--readonly`.

2. For each method on `BeadsGateway` (interface.go:11), exercise the real `bd <verb>` with `--json` across these dimensions:
   - **Inputs**: no filter, single-status filter, multi-status, sort flags, limit/offset, invalid input.
   - **Outputs**: empty result, single result, many results, results with null/missing fields.
   - **Edge cases**: closed issues filtered by default? Sort applied to all returned items or only the page? Limit cap behavior? Are returned fields complete or sparse?

3. As you discover behavior, write findings directly into `interface.go` as doc comments on each method. Format:
   ```go
   // ListIssues returns issues matching the query.
   //
   // Preconditions:
   //   - ...
   // Postconditions:
   //   - Returned IssueSummary has non-empty ID and a status from StatusCatalog
   //   - Sort: applied when SortBy is set; ...
   //   - Closed: excluded by default unless query.Statuses contains "closed" or IncludeClosed=true
   // Error semantics:
   //   - ErrorCodeCommandFailed when ...
   //   - ErrorCodeDecodeFailed when ...
   // bd quirks observed:
   //   - bd <flag> behavior at <case> diverges from <expectation> — see audit comment ABC
   ```

4. At the top of `interface.go` add an "Observed bd quirks (audit YYYY-MM-DD)" appendix listing systemic findings that don't belong to one method (e.g., "bd treats workstate=ready as a separate verb, not a flag" — that kind of thing).

5. For each "bd quirks observed" item, decide:
   - **Accept**: contract documents the quirk as part of the spec.
   - **Carve out**: contract says "bwb compensates by ..." (e.g., `validatingGateway` will normalize, or the gateway already filters).
   - **File upstream**: open a bug against bd for the divergence. Note the link in the doc comment.

6. Read the existing `RunReadContract` (`internal/gateway/beads/contract/contract.go`) BEFORE writing. The current assertions encode some implicit contract claims; reconcile them with your audit. If RunReadContract asserts something bd doesn't honor, that's a contract bug to surface in your report (and fix in a later task).

## When the audit reveals a real assumption-vs-reality mismatch

Apply the "Discovered-during-audit bug discipline" from the parent epic [[8qw9]]. Concretely, whenever you discover that bwb's consumer code, the gateway's decoder/builder, or the existing fake assumes a behavior bd does NOT actually honor:

1. **File a separate bug** via `bd create --type=bug --title="Gateway contract violation: <consumer> assumes <X>, real bd does <Y>"`. Do not bury the finding in the audit notes — surface it as a tracked defect.
2. **Bug body MUST include**: "Verify in app (`mise run bwb` against a representative DB) that this assumption-vs-reality gap actually causes user-visible incorrect behavior. If it does not (the wrong behavior never reaches the user), close as 'not a bug' with rationale. If it does, fix at the appropriate layer."
3. **Link the bug back to this task** via `bd dep add <bug-id> beads-workbench-8qw9.1` so the discovery context is preserved.
4. **Do NOT fix the bug as part of this audit task** — keep the audit focused on documentation. Bug fixes happen in their own ticket, where they're tested with the (eventually-updated) fake gateway.

This task's goal is to PRODUCE the contract spec and SURFACE every divergence as a tracked bug. Fixing is downstream work.
