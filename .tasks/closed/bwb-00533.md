## Description
Make board and search safe for automatic reload so freshness checks do not yank the user away from current context.

## Instructions
1. Introduce a non-destructive board auto-refresh path separate from the existing manual `r` full reload behavior.
2. Update `internal/mode/board/model.go` so board auto-refresh can preserve the focused column and previously selected issue when that issue still exists after reload.
3. Define predictable fallback behavior when the prior board selection no longer exists.
4. Update `internal/mode/search/model.go` and shell orchestration so search auto-refresh reruns the current query only when safe and does not interrupt active typing.
5. Preserve existing selection/query state as much as possible across auto-refresh, and do not force mode switches or clear current detail state as a side effect of board/search refresh.

## Acceptance Criteria
- [ ] Board auto-refresh restores prior selection and focus when the issue still exists
- [ ] Board uses a deterministic fallback when the prior selection disappears
- [ ] Board auto-refresh is non-destructive relative to the existing manual full reload behavior
- [ ] Search auto-refresh does not fire while the user is actively typing in the query field
- [ ] Search refresh preserves the current query and current result selection when possible
- [ ] Board/search refresh does not force active-mode changes or clear current detail state as a side effect

- `internal/mode/board/model_test.go`
