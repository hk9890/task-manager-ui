On the seeded embedded fixture, verify_bwb_state_flow mutation presets can fail before detail opens because they send ENTER immediately after startup. Repro: create a fixture repo, then run the mutation-cancel preset without extra pre-steps. Actual: capture times out waiting for Detail because the preset fires before the initial board selection/readiness text is present. Expected: built-in mutation presets should wait for the initial board selection/readiness state themselves instead of requiring caller-supplied pre-steps.

## Acceptance Criteria
- [ ] Mutation-save and mutation-cancel presets explicitly wait for an initial board readiness signal before sending ENTER.
- [ ] The readiness signal is concrete and deterministic (for example `wait-for-text:Selected:` or equivalent current board-selection text), not a blind sleep.
- [ ] Seeded-fixture runs no longer require extra caller-supplied pre-steps to reach detail reliably.
