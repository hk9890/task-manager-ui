## Description

Comparison tests need to point at multiple beads databases without duplicating setup. Build an adapter package that yields a `BeadsGateway` and a parallel `bd` CLI invoker bound to the same DB directory, with the external DBs strictly read-only.

## Instructions

1. Create `internal/testing/datasets/datasets.go` exporting:
   - `type Dataset struct { Name, Path string; ReadOnly bool }`
   - `Fixture(t *testing.T) Dataset` — wraps `embeddedfixture.SharedFixtureRepoPath`; `ReadOnly: false` (writable temp dir).
   - `ThisRepo(t *testing.T) Dataset` — uses repo root; honors `BWB_PARITY_THIS_REPO=1`, skip otherwise; `ReadOnly: true`.
   - `AIConfigManager(t *testing.T) Dataset` — uses `BWB_PARITY_AICM_PATH` env var (default `/home/hans/dev/github/ai-config-manager`); honors `BWB_PARITY_AICM=1`, skip otherwise; `ReadOnly: true`.
2. Create `func NewGateway(t *testing.T, ds Dataset) beads.BeadsGateway` that builds a `CommandRunner` with `WorkDir: ds.Path`. When `ds.ReadOnly`, prepend `--readonly` to every bd argv (verify by reading `internal/gateway/beads/runner.go` for the argv hook point; if no hook exists, add one as a separate small refactor before this task).
3. Create `func BdCount(t *testing.T, ds Dataset, args ...string) ([]byte, error)` — runs `bd --readonly count <args> --json` via `exec.Command` from `ds.Path` and returns stdout. Mirror for `BdList`, `BdReady`, `BdBlocked`, `BdSearch`.
4. Skip cleanly when `bd` is not on PATH or the dataset is opt-in but its env flag is unset.

## Acceptance Criteria

- [ ] `datasets.Fixture` returns a usable Dataset; `datasets.NewGateway` returns a gateway whose `ListIssues` succeeds against the fixture
- [ ] `datasets.ThisRepo` and `datasets.AIConfigManager` skip with `t.Skip` when env gate is off
- [ ] When `ds.ReadOnly`, any attempted write through the returned gateway or bd helper fails fast (verify with one negative test)
- [ ] `go test ./internal/testing/datasets/...` passes
- [ ] No file under `/home/hans/dev/github/ai-config-manager/.beads/` is modified by these tests (verify by snapshotting `ls -la .beads/` mtimes around a test run)
