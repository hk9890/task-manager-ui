## Description

bwb has a `pre-commit` hook in `scripts/git-hooks/pre-commit` that auto-runs `gofmt` on staged files, but **it only protects contributors who ran `mise run hooks:install`**. New contributors (or anyone using a GUI Git client that skips hooks) can land unformatted code. Add a CI check that mirrors what the hook does, plus `goimports` (since the hook only runs `gofmt`).

## Files to Modify

- `.mise.toml` — add a `fmt:check` task (read-only goimports verification)
- `.github/workflows/ci.yml` — add a step that runs `fmt:check`
- `scripts/git-hooks/pre-commit` — extend to run `goimports` on staged files (preserve staged-file scoping)

## Instructions

1. Add to `.mise.toml`:
   ```toml
   [tasks."fmt:check"]
   description = "Check formatting (goimports + go mod tidy)"
   run = """
   go install golang.org/x/tools/cmd/goimports@latest
   OUT=$(goimports -local github.com/hk9890/beads-workbench -l .)
   if [ -n "$OUT" ]; then
     echo "Files not formatted:"; echo "$OUT"
     echo "Run: goimports -local github.com/hk9890/beads-workbench -w ."
     exit 1
   fi
   go mod tidy
   if ! git diff --exit-code go.mod go.sum; then
     echo "go.mod/go.sum not tidy. Run 'go mod tidy' and commit."
     exit 1
   fi
   """
   ```
2. In `.github/workflows/ci.yml`, add a step before "Lint":
   ```yaml
   - name: Format check
     run: mise run fmt:check
   ```
3. Update `scripts/git-hooks/pre-commit` to run `goimports` (in addition to or instead of `gofmt`) for consistency between local and CI. **Important:** preserve the existing staged-file scoping — operate only on files listed by `git diff --cached --name-only -z --diff-filter=ACMR -- '*.go'`, NOT the whole tree. The current hook uses xargs over a staged-files list (`scripts/git-hooks/pre-commit:20-26`); the new step should follow the same pattern. Note also: `goimports` must be installed for the hook to work; either install it at hook-install time or vendor it via `go run golang.org/x/tools/cmd/goimports@latest` (slower per commit, no install required).

## Acceptance Criteria

- [ ] `mise run fmt:check` exists and passes on a clean tree
- [ ] CI fails on a PR that introduces unformatted code (verify with a deliberate one-line breakage in a draft PR, then revert)
- [ ] `pre-commit` hook also runs `goimports` on staged files (not the whole tree)
- [ ] `go mod tidy` drift caught by CI
