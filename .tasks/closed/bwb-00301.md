In keybindings.go:12-17, context identifiers (ShellContext, BoardContext, etc.) are untyped string constants. They're used as map keys in KeyBindings struct (lines 71-77), in ResolveKeyBindings, Match(), Keys(), Primary(), and throughout load.go. Using bare strings means typos like Match('shel', ...) compile and silently return false. Suggested action: define type BindingContext string and use it for the context constants and as the parameter type for Match/Keys/Primary/DisplayPrimary/etc. This catches typos at compile time without changing runtime behavior. File: internal/config/keybindings.go:12-17
