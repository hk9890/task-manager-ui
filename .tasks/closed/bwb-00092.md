## Description
Rework the Content pane header in the Details view so it reads like a dashboard row
and clearly separates header from body.

New header layout (top to bottom):
1. Compact metadata row: type, priority, status (each in its board/dashboard color via
   styles.IssueTypeStyle/IssuePriorityStyle/IssueStatusStyle, using renderhelpers.Compact*Styled)
   followed by the full issue ID (muted). e.g. `T  P1  CLS  beads-workbench-vtvb`.
2. Title line (plain text).
3. A thin horizontal rule (─ full width, muted) to visually divide the header from the body.
4. Existing `Description` label + body, comments, etc. (unchanged).

This replaces the current header (title first, then `id · status · priority`, then a
blank line). Net header line count is unchanged (4 lines), so skeleton height math stays.

## Instructions
1. In `internal/ui/details/details.go`, in `renderContentPaneLines`, replace the first
   header lines with: meta row, title, thin rule, `Description`.
2. Add a `contentHeaderMetaRow` helper using `renderhelpers.CompactIssueTypeStyled`,
   `CompactPriorityStyled`, `CompactIssueStateStyled`, and a muted full ID
   (`styles.IssueIDMutedStyle`), with a width-bounded plain fallback for narrow widths.
3. Add a `contentHeaderRule` helper rendering `strings.Repeat("─", width)` in a muted color.
4. Add the `renderhelpers` import.
5. Update/refresh affected golden/testdata + tests in `internal/ui/details/`.
