## Description

PLACEHOLDER EPIC — captures scope so we don't lose context. Not actively planned yet.

Two test-side `exec.Command` swaps exist with overlapping purpose:
- `routingExecutor` (private, `internal/gateway/beads/read_gateway_test.go`) — maps argv → canned response; used in ~70 decoder tests.
- `RecordingExecutor` (public, `internal/testing/fakes/recording_executor.go`) — does the same AND records calls; used in argv-cardinality tests.

These should be ONE shared executor. RecordingExecutor is the more capable; routingExecutor should be migrated and deleted.

Additionally, every bd call bwb makes (the OUTGOING contract) should have an explicit argv-shape assertion test. Today only the board init's 4 calls are pinned (`TestBoardInitRealGatewaySubprocessArgvCardinality`). Search, detail, write operations should have equivalent assertions so accidental migrations (e.g. someone swaps `bd list` for `bd query` and forgets a flag) trip a test immediately.

## Probable scope (when activated)

- Migrate all routingExecutor callers to RecordingExecutor; delete routingExecutor.
- Inventory every distinct bd argv shape bwb emits at runtime.
- Add per-call argv-cardinality tests where missing (e.g., search loads, detail show, write operations).
- Document the argv contract pattern in `internal/gateway/beads/doc.go`.

## Out of Scope

Read-side semantic contract (covered by epic [[8qw9]]).
Write-side semantic contract (covered by separate placeholder epic — see [[harden-gateway-write-contract]]).

## Activation Trigger

Plan and activate after [[8qw9]] closes. Before activation, audit whether bd's argv has changed (recent bd releases may have renamed flags) so the test pinning is current rather than retroactive.
