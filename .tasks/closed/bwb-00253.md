## Description
Acceptance review of epic beads-workbench-au54 measured ~1.5-2 s median cold startup on this repo (persistent log at `/home/hans/.local/state/bwb/bwb.log`, 6 cold sessions). The original <1.2 s AC was not met empirically.

Root cause: Dolt embedded-mode imposes ~470–2500 ms per-process startup cost on each `bd` invocation. With 4 parallel calls, wall time = max(call durations). Empirical comparison shows parallel and sequential are within ~10% of each other (Dolt is the bottleneck, not concurrency).

The bwb-side architecture is correct — 7 → 4 calls, RWMutex split, `bd ping` health check, parallel fan-out all confirmed working.

## Potential investigation paths
- **bd server mode** if/when upstream beads adds one (single long-lived process, no per-call Dolt init).
- **bd ping warmup** — pre-warm Dolt before the dashboard reads fire.
- **Dolt connection pooling** — likely requires upstream bd changes.
- **Reduce parallel count when Dolt contention dominates** — empirically test serial vs parallel modes.
