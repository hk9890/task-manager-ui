Discovered when cross-OS matrix (ug82.8) ran on the first push (commit 9439d6c). All test/lint mise tasks use `go run ...@$(cat .gotestsum-version)` for version pinning. PowerShell on Windows runners cannot expand `$(cat ...)` and fails with 'invalid version: unknown revision $(cat'. Affected tasks: test, test:integration, test:verbose, test:integration:verbose, test:coverage, lint, fmt:check. Fix: migrate from shell expansion to mise [tools] entries (e.g. 'go:gotest.tools/gotestsum = v1.13.0') so binaries are installed by mise and called directly. Removes the .gotestsum-version / .goimports-version / .golangci-version files (replaced by [tools] pins).
