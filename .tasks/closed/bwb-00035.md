## Description
Two related startup UX problems:

1. Initial board load feels slow with no clear progress feedback. The board already shows a loading state, but users get no indication of how many sections are loading or how far along things are.
2. If `bd` is not installed or not in PATH, the app silently fails inside the board view with a generic error string rather than routing to a dedicated, actionable error screen.

## Success Criteria
- [ ] Child implementation tasks complete
- [ ] Acceptance review task complete
- [ ] Board shows progress-aware loading feedback during startup
- [ ] App detects missing/broken `bd` at startup and shows a dedicated error screen before loading the board
