## Description
Redesign board row rendering so important issue signals are glanceable at high density. The target is a compact triage row style closer to Perles, with substantially more visible work per screen than the previous two-line-row layout.

## Instructions
1. Redesign row rendering in `internal/ui/board/` for materially better scanability and density.
2. Include compact representations for type, priority, state, ID, and title in the rendered row format.
3. Update the board row data shape and mapping as needed so `internal/mode/board/model.go` supplies the required row fields to the UI renderer.
4. Improve issues-per-screen over the old two-line layout while preserving distinguishability on realistic IDs/titles.
5. Target a compact single-row format where practical; if a row expands beyond one line in normal board rendering, it must be justified by readability and still meet density goals.
6. Add/update golden tests at 80, 120, and 180 columns with realistic repo data.
7. Use the dashboard UX test harness from `beads-workbench-1ri`, including runtime/render assertions rather than only model-state assertions.
8. Perform a PTY/runtime verification pass and report what improved in the actual board rendering.

## Files to Modify
- `internal/ui/board/board.go`
- `internal/ui/board/board_test.go`
- `internal/ui/board/testdata/*`
- `internal/mode/board/model.go`
- `internal/mode/board/model_test.go`
- `internal/mode/board/testdata/*` if mode goldens change

## Acceptance Criteria
- [ ] Normal board rows render in a compact single-line form at representative widths unless a justified exception is documented in tests
- [ ] The new layout shows more visible issues per viewport than the previous two-line-row design at the same representative widths
- [ ] Type/priority/state/ID/title signals are compact and glanceable in the rendered row format
- [ ] Long realistic IDs/titles remain distinguishable without destroying density
- [ ] Golden tests cover dense board rendering at 80, 120, and 180 columns
- [ ] Runtime/render-level tests verify the new row presentation, not only model state
- [ ] PTY/runtime review shows improved scanability on realistic repo data
