## Description
The UX review found that compact detail preview currently renders blocked-by information twice when an issue has upstream dependencies. The preview first shows a named "Blocked by" line and then immediately shows the dependency count summary line. This makes the compact preview feel repetitive and less polished.

## Evidence
- Reviewer report from full UX review
- `internal/ui/details/details.go` currently renders both a named blocked-by line and the summary count line in compact mode
- Visible in compact preview/golden output

## Instructions
1. Remove the duplicate blocked-by rendering from compact detail preview.
2. Keep compact preview concise and scan-friendly.
3. Update the relevant detail/search snapshots or tests.
4. Run focused detail/search/app tests and confirm they pass.

## Files to Inspect
- `internal/ui/details/details.go`
- `internal/ui/details/testdata/compact.golden`
- related search/detail tests

## Acceptance Criteria
- [ ] Compact detail preview no longer repeats blocked-by information
