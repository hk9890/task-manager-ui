## Description

Convert the 8 `TestModelEmbeddedFixture*` tests in `internal/app/model_integration_test.go` to unit tests using `fakes.FakeBeadsGateway`, then delete the integration versions. The gateway-contract tier (already built by the preceding tasks) ensures the fake's behavior matches real bd, so this is a safe move that does not lose coverage.

## Why

These 8 tests collectively cost ~88s of integration time. They're asserting **app behavior** (navigation, modal state, view rendering) ‚Äî not bd-contract behavior. With the contract suite enforcing fake/real parity, the app behavior can be unit-tested for free.

## Instructions

For each integration test, do an audit-then-port pass:

1. **Audit** what each test actually asserts:
   - `TestModelEmbeddedFixtureBoardToDetailSmokeWorkflow` ‚Äî board lands on Not Ready lane, enter opens detail with correct content
   - `TestModelEmbeddedFixtureDetailEditHotkeyUsesEditorService` ‚Äî `e` key in detail invokes editor for selected issue (already uses fake editor!)
   - `TestModelEmbeddedFixtureMutationModalsOpenWithoutCatalogDecodeToast` ‚Äî `c`/`u`/`a` open modals without a "decode failed" toast
   - `TestModelEmbeddedFixtureFullBoardCaptureGolden` ‚Äî board view matches golden at width 120
   - `TestModelEmbeddedFixtureStartupLoadsBoardWithoutGatewaySectionErrors` ‚Äî startup view has no error panels
   - (Bucket B tests handled separately by ko71/i8pt ‚Äî DELETE the integration versions in this task, do NOT port)
2. **Check coverage**: for each assertion above, search `internal/app/model_test.go` and `internal/app/scenario_workflow_test.go` for an equivalent unit test using `FakeBeadsGateway`. Many already exist.
3. **Port any missing assertions** as new unit tests:
   - Use `FakeBeadsGateway` pre-seeded to match the same data shape (3 issues with bwf-* IDs, similar status mix) so any golden files transfer cleanly.
   - For the golden file (`testdata/model_embedded_board_w120.golden`): re-generate it from the fake-driven unit test if any rendering differs; otherwise the existing file is reused as-is.
4. **Delete** `internal/app/model_integration_test.go` once the audit confirms coverage parity. Keep `runBDInRepo` only if it's still used by the link/dep scenario tests (see i8pt ‚Äî likely moved to embeddedfixture helper).
5. Document any assertion that genuinely cannot be expressed against the fake (likely: none, but if found, escalate before deleting the corresponding integration test).
