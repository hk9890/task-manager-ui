## Description
Improve the small ad-hoc UI verification toolkit based on the feedback from recent verifier usage.

## Why
The toolkit is already useful, especially `scripts/capture_bwb_screen.py`, but recent verifier sessions showed a few focused improvements would make it faster and more reliable to use in practice without turning it into a framework.

## What should be done
### 1. Field-specific state assertions in the wrapper
Improve `scripts/verify_bwb_state_flow.py` so representative flows can verify explicit fields instead of only shallow whole-object changed/unchanged checks.

Target implementation direction:
- add an explicit small `expect_fields` / equivalent mapping per representative flow
- support simple expectations such as:
  - field changed
  - field unchanged
  - optional exact expected value when practical

Keep this small and flow-specific; do not build a generic assertion language.

### 2. One stronger ephemeral/readiness helper in the PTY script
Improve `scripts/capture_bwb_screen.py` by adding a small wait primitive for transient UI messages.

Chosen scope:
- add `wait-for-text-once:<TEXT>[:timeout-ms]`
- semantics: pass once the text has appeared at least once during the wait window, even if it later disappears

This directly helps toast/ephemeral feedback checks without framework creep.

### 3. Better failure/debug evidence
Improve failure output so when a step fails it is easier to understand why.

Chosen scope:
- increase observed failure excerpt usefulness
- automatically include the latest checkpoint (if any) in failure output
- include a small recent raw-output tail on failure when available

### 4. Keep the wrapper thin
Do not turn `verify_bwb_state_flow.py` into a broad framework. It should remain a small convenience layer around:
- running the PTY script
- taking before/after `bd show --json` snapshots
- checking a few explicit fields

## Constraints
- Keep the toolkit small and ad-hoc oriented.
- No branching DSL or generalized framework behavior.
- Preserve the current usefulness of `capture_bwb_screen.py` as the main tool.

## Acceptance Criteria
- [ ] Wrapper supports explicit field-targeted state verification for representative mutation checks.
- [ ] PTY helper supports `wait-for-text-once` for transient/ephemeral-message verification.
- [ ] Failure output includes more useful debugging evidence than before (improved excerpt plus latest checkpoint/raw-tail context where available).
- [ ] A representative mutation-save flow passes with field assertions.
- [ ] One intentional failing probe demonstrates the improved failure output.
- [ ] The implementation stays small and script/tooling oriented.
- [ ] Real-app verification confirms the improved scripts are still practical for ad-hoc UI testing.
