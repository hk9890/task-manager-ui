TemplateInterpolator (service.go:88-106) and DefinitionResolver (service.go:56-86) are exported types, but neither is used outside the launcher package. DefinitionResolver is constructed and consumed entirely within NewService/launcherService. TemplateInterpolator is instantiated only in NewService.

Exporting these types suggests they are part of a public API contract when they are actually implementation details. This increases the API surface consumers must understand and constrains future refactoring.

Suggested fix: unexport both to `templateInterpolator` and `definitionResolver`. Keep `NewDefinitionResolver` exported only if there is an external consumer (there isn't currently). The `Definition` and `InterpolationContext` types are fine to stay exported since they are used in config wiring.
