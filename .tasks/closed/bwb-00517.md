## Description

The epic's success criteria reference relative reductions ("<15s", ">80% drop") but no baseline measurement exists. Without a baseline, the acceptance review (i1th.1) and the build-tag ticket (i1th.2) have nothing concrete to compare against.

This ticket captures the baseline numbers BEFORE any other epic work runs. Pre-resolves the empirical questions that would otherwise leak into multiple tickets.

Reviewer finding F29. Blocks i1th.1 (acceptance review) and i1th.2 (build tag).

## Files to Modify

- Create `docs/test-baseline-2026-05-16.md` (or update an existing doc) capturing the numbers — short, just data.

## Instructions

1. **Clean cache:**
   ```bash
   go clean -testcache
   ```

2. **Capture total test count:**
   ```bash
   go test -json ./... 2>/dev/null \
     | jq -r 'select(.Action=="pass" or .Action=="fail") | select(.Test != null and (.Test | contains("/") | not)) | .Package + "::" + .Test' \
     | sort -u | wc -l
   ```
   Record the count.

3. **Capture total wall time** (3 cold runs, take the median):
   ```bash
   for i in 1 2 3; do go clean -testcache; time go test ./... -timeout=180s >/dev/null 2>&1; done
   ```
   Record the 3 wall times and the median.

4. **Capture per-test breakdown for the 9 integration suspects:**
   ```bash
   go clean -testcache
   go test -json ./... -timeout=180s 2>/dev/null \
     | jq -r 'select(.Action=="pass" or .Action=="fail") | select(.Elapsed != null and .Elapsed > 1) | "\(.Elapsed)s  \(.Package)::\(.Test)"' \
     | sort -rn | head -20
   ```
   Record the list.

5. **Write `docs/test-baseline-2026-05-16.md`** with: total count, median wall time, the top-N slowest tests table, hardware fingerprint (`uname -srm`, Go version from `go version`).

6. Commit the baseline doc as part of this ticket.

## Acceptance Criteria

- [ ] `docs/test-baseline-2026-05-16.md` exists and contains: total test count, 3 wall-time measurements + median, top-20 slowest tests, hardware/Go fingerprint
- [ ] The integration ticket (i1th.2) can reference a concrete pre-split count for its coverage-parity AC
- [ ] The acceptance review (i1th.1) can compute "80% drop" against a concrete baseline number
