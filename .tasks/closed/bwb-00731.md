## Description
Create one reusable command execution layer for all future beads gateway operations. This keeps subprocess logic, environment propagation, JSON decoding, and error normalization out of feature code and makes the CLI-backed gateway testable.

## Instructions
1. Implement a shared `bd` command runner under `internal/gateway/beads/` or a closely related package.
2. Support working-directory selection and environment propagation for beads commands.
3. Add helpers for executing JSON-returning commands and decoding structured output.
4. Normalize subprocess failures, stderr content, exit-code failures, and decode failures into the gateway error model.
5. Add test seams so command execution can be exercised without fragile shell-dependent end-to-end tests.
6. Keep this layer generic enough for all planned read and write gateway methods.

## Files to Modify
- `internal/gateway/beads/` (create runner and decoding helpers)
- `internal/domain/` (update if shared result/error models need support)
- `*_test.go` under gateway packages (create tests)

## Acceptance Criteria
- [ ] One shared command execution layer exists for `bd` interactions
- [ ] Working directory and environment propagation are supported
- [ ] JSON command execution and decoding helpers exist
- [ ] Process, stderr, exit-code, and decode failures map into normalized gateway errors
- [ ] Tests can exercise command behavior without fragile shell-only coupling
