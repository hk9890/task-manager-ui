## Description
Track the focused usability improvements made to the ad-hoc UI verification toolkit after real verifier feedback.

## Scope
This polish work keeps the toolkit small and script-oriented while improving day-to-day usability:
- field-specific state assertions in the wrapper
- one stronger transient-message wait primitive in the PTY helper
- better failure/debug evidence on capture failures
- explicit guardrail to keep the wrapper thin and non-framework-like

## Outcome
The work should make the helper scripts faster to trust in ad-hoc UI investigation without expanding them into a generalized test framework.

## Acceptance Criteria
- [ ] Wrapper supports field-targeted verification for representative mutation flows.
- [ ] PTY helper supports a transient-message wait primitive.
- [ ] Failure output/debug evidence is more useful than before.
- [ ] Real-app verification confirms the scripts remain practical and small.
