## Description
Capture the resolved issue-details panel contract and the remaining implementation guidance before execution.

## Resolved Contract
- The special left browser panel exists only in issue details view.
- Board and search do not adopt the same explicit browser/content/metadata panel structure.
- Board and search still must share the same shell sizing/full-height contract so the app uses space consistently.
- In issue details, the left browser should show a stable parent-group context: the parent issue plus sibling child issues.
- Selecting a left-browser item updates the center/right detail workspace.
- Browsing inside the same parent group should not mutate the left-browser contents for every selection.
- If the current issue has no parent issue, omit or hide the special left browser panel for now.
- Do not rely on inline key hints.
- Ignore ordering concerns for now.

## Remaining Implementation Guidance
- The parent-group browser context needs explicit backend/domain support because the current issue detail model does not expose parent and sibling grouping.
- The right metadata panel still needs one final interaction decision for Enter-driven status editing.

## Acceptance Criteria
- [ ] Issue-details panel contract is explicit enough for implementation
- [ ] Board/search sizing expectations are explicit without forcing them into the same panel structure
- [ ] Left-browser data source is explicitly chosen as parent-group context
- [ ] No-parent fallback behavior is explicit
