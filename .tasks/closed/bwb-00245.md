## Summary

`FakeBeadsGateway.AddComment` appends `IssueComment{Body: input.Body}` with no Author field. Real bd 1.0.4 stores `author = git user.name`. UI components that display comment authors see empty strings from fake-backed tests.

## Discovered during

`beads-workbench-ibif` Write audit (parent epic: `beads-workbench-yspw`).

## Fix

In the fake's AddComment method, set `Author` to a sentinel value, e.g. `"fake-user"` or `"git-user.name"`, and document in fakes/doc.go.

## Acceptance criteria

- [ ] Fake AddComment sets Author to a non-empty sentinel.
- [ ] Add contract sub-test asserting `comment.Author != ""` after AddComment + ShowIssue.
- [ ] Update fakes/doc.go drift log.
