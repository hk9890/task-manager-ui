## Description

The current `Makefile` has an `install-hooks` target (referenced from `docs/CHANGE-WORKFLOW.md`) that wires `scripts/git-hooks/` into the repo via `git config core.hooksPath`. When the Makefile is removed in beads-workbench-i1th.5, this functionality must NOT silently disappear.

Reviewer finding F28.

## Files to Modify

- `.mise.toml` (add a new task)
- `docs/CHANGE-WORKFLOW.md` (update the install-hooks reference — coordinate with beads-workbench-i1th.6)

## Instructions

1. Inspect the existing Makefile `install-hooks` target to capture the exact command:
   ```bash
   grep -A5 'install-hooks:' Makefile
   ```
   (Likely `git config core.hooksPath scripts/git-hooks` plus `chmod +x scripts/git-hooks/*`.)

2. Add a mise task in `.mise.toml`:
   ```toml
   [tasks."hooks:install"]
   description = "Install repo git hooks (one-time setup)"
   run = '''
   git config core.hooksPath scripts/git-hooks
   chmod +x scripts/git-hooks/*
   '''
   ```

3. Update `docs/CHANGE-WORKFLOW.md` references from `make install-hooks` → `mise run hooks:install` (coordinate with beads-workbench-i1th.6 to avoid double-edit).

## Acceptance Criteria

- [ ] `mise run hooks:install` exists and sets `core.hooksPath` to `scripts/git-hooks`
- [ ] Verified: after running the task, `git config core.hooksPath` returns `scripts/git-hooks`
- [ ] `docs/CHANGE-WORKFLOW.md` references the mise task, not the Makefile target
- [ ] No remaining grep hits for `make install-hooks` in repo
