## Description
The `e` (edit_issue) flow launches the interactive editor inside a `tea.Cmd` goroutine while Bubble Tea owns the terminal (alt-screen + raw mode). The editor and TUI fight over the TTY, making the editor unusable. The fix routes the launch through `tea.Exec` (Bubble Tea's `p.exec()` runs inline from the Update loop, bracketed by ReleaseTerminal/RestoreTerminal — exactly the TTY handover needed). `tea.Exec` can only be processed when returned from `Update`, so the monolithic `editor.Service.EditIssue` must be split so `tea.Exec` can sit mid-flow.

## Instructions
1. In `internal/launcher/editor/service.go`, replace `EditIssue(ctx, issueID)` with two methods:
   - `PrepareDocument(ctx, issueID) (Prepared, error)` — runs `ShowIssue` + `RenderIssueEditDocument` + writes the temp file; returns the temp path and the original issue.
   - `ApplyEdits(ctx, issueID string, issue domain.IssueDetail, path string) (Result, error)` — reads the temp file, `ParseIssueEditDocument`, `BuildIssueUpdateInput`, calls `UpdateIssue` when changed, removes the temp file.
2. Temp-file cleanup: the normal-exit `defer os.Remove` must still fire on the success, no-change, and editor-error paths. Do NOT add startup-sweep logic — leftover docs from a crashed/killed editor are already swept by `cleanStaleTempFiles` (services.go ~line 76).
3. Delete the `Opener` interface and `ProcessOpener.Open` — `editor.Service` no longer launches a subprocess. Move command parsing (`splitEditorCommand`/`shellSplit`) into a helper that builds the editor `*exec.Cmd` from `cfg.Editor.Command`; the model passes that into `tea.Exec`.
4. In `internal/app/model.go`, replace the single `editIssueCmd` with the 3-phase flow:
   - prepare cmd → message carrying `Prepared`
   - on that message, return `tea.Exec(editorCmd, callback)` where `editorCmd` is a `tea.ExecCommand`
   - on the exec callback, run an apply cmd → existing `editIssueResultMsg` handler (model.go:456) is reused unchanged.
5. Add an `ExecCommandFactory func(*exec.Cmd) tea.ExecCommand` field to `Services` (production default = `tea.ExecProcess` semantics; tests assign a fake). Wire the default in BOTH constructors in `services.go`: `NewServices` (line 39) and `NewServicesWithLauncher` (line 115).
6. Migrate every caller/test of the old `EditIssue` API to `PrepareDocument`/`ApplyEdits` and the new message shape (`m.Update(KeyMsg("e"))` now yields a `tea.execMsg`, not a cmd returning `editIssueResultMsg`). Keep `go build ./...` green.
7. Update `internal/launcher/editor/doc.go` to document the `PrepareDocument → tea.Exec → ApplyEdits` contract; add a sentence to `docs/CONFIGURATION.md` "Shell editor/launcher UX behavior" noting the editor launch now suspends/restores the TUI via `tea.Exec`.
