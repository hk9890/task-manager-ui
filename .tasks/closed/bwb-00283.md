read_gateway.go: SearchIssues (lines 219-256), searchIssuesFromList (lines 280-313), and ListIssues (lines 44-89) all independently build nearly identical CLI arg arrays for status/type/assignee/label/priority/limit filters. The searchIssuesFromList method is almost a copy-paste of SearchIssues with 'list' instead of 'search'. This violates DRY and means filter changes need updating in 3+ places. Suggested action: extract a shared buildFilterArgs(query) helper that appends the common filter flags, then have each method compose the subcommand name + shared filter args + method-specific args.
