## Description

The gateway runner silently allows two routes that violate the documented "one project per gateway instance" invariant (CODING.md rule #3):

1. **Per-request WorkDir override** — `CommandRequest.WorkDir` (`internal/gateway/beads/runner.go:27, 174-180`) lets any caller pass an arbitrary working directory that overrides the gateway's bound `defaultWorkDir`.
2. **Full-environment inheritance** — when `RunnerConfig.Env == nil` the runner falls back to `os.Environ()` (`runner.go:73`). A stray `BD_DB_PATH`, `XDG_CONFIG_HOME`, or other `BD_*` env var in the user's shell silently redirects every `bd` invocation away from the bound project.

Either path can silently target a different beads database than the gateway was constructed for. Major findings G1+G3 from the 2026-05-16 audit, combined here because they share the same root cause (the runner not enforcing its binding).

## Files to Modify

- `internal/gateway/beads/runner.go`
- `internal/gateway/beads/cli_gateway.go` (constructor — set explicit Env)
- new test: `internal/gateway/beads/runner_binding_test.go`

## Instructions

1. Remove `CommandRequest.WorkDir` OR make it private and assert it is unused outside the runner. If retained for legitimate one-off cases, hard-fail if it points at any directory other than the gateway's bound `defaultWorkDir`.
2. Replace the `Env == nil` → `os.Environ()` fallback with an explicit allowlist. Suggested minimum allowlist: `PATH`, `HOME`, `USER`, `LANG`, `LC_*`, `TERM`. All `BD_*` env vars MUST be stripped unless the caller explicitly opted in.
3. Gateway constructor (`NewCLIGateway`) should set `RunnerConfig.Env` to the filtered allowlist by default. Callers should not need to know.
4. Add `runner_binding_test.go` covering:
   - Test: caller passes `CommandRequest.WorkDir` pointing at `/tmp/x` → request is rejected (or override ignored, with executor receiving the gateway's bound dir).
   - Test: parent env contains `BD_DB_PATH=/etc/passwd` → executor receives an env with that var stripped.
   - Test: parent env contains `PATH=...` and `HOME=...` → executor receives those.

## Acceptance Criteria
