## Description

Cluster of small bootstrap/config correctness hygiene items from the 2026-05-16 audit. None are user-visible alone but they form foot-guns.

1. **BS3 — `mergeContextBindings` mutates `base` in place** (`internal/config/load.go:185-195`). Currently safe only because `MergeKeyBindings` deep-clones first. A future caller passing `DefaultKeyBindings()` directly would corrupt the package default. Rename to `mergeContextBindingsInPlace` or build a new map.
2. **BS5 — `resolveAndValidateCWD` only checks IsDir, not access** (`cmd/bwb/main.go:90-100`). A `--cwd` to an unreadable directory passes startup and the gateway fails later with a confusing error.
3. **BS-minor — `generateSessionID` returns literal "00000000" on rand.Read failure** (`internal/logging/logging.go:199-205`). Multiple runs in degraded entropy state collide. Append a nanosecond timestamp fallback.

## Files to Modify

- `internal/config/load.go`
- `cmd/bwb/main.go`
- `internal/logging/logging.go`
- extend existing tests in each file

## Instructions

1. **Mutation:** rename `mergeContextBindings` to `mergeContextBindingsInPlace` and document, OR refactor to build a new map. Either is acceptable.
2. **CWD access:** add an `os.Open` (or os.Stat with executable-bit check) probe to `resolveAndValidateCWD`. Test: pass a dir with `--x` permissions, assert startup fails with code 1 and a clear message.
3. **Session ID:** in `generateSessionID`, on `rand.Read` error append `time.Now().UnixNano()` (formatted hex) to keep uniqueness. Test by stubbing the rand reader.

## Acceptance Criteria

- [ ] Each of the three items addressed
- [ ] Tests cover the new behaviors
