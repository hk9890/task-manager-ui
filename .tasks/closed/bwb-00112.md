When BWB starts in a directory with no beads database, app.Model.Init runs healthCheckCmd (bd ping) and board.Init() concurrently via tea.Batch. board.Init() fires ~4 more bd queries that all fail with 'no beads database found' — 5 failing subprocesses per launch. Monitoring analysis observed this across 3 sessions (bluebox, setup-workstation, dtctl).

Fix: gate board.Init() on a successful startupHealthCheckMsg so the missing-DB case is detected with a single bd ping; only fire board section loads after the health check passes. Keep happy-path latency acceptable (ping is sub-500ms). Update affected app tests.
