In keybindings.go:369-388, canonicalKeyName checks for literal space (key == ' ') on line 371 before checking trimmed empty on line 374. If key is multiple spaces ('  '), it falls through to trimmed empty, returns '' — but then isValidKeyName('') returns false. A single space becomes 'space' and is valid. This edge case is unlikely in practice since YAML parsing would normalize whitespace, but the function's contract is confusing: it special-cases one whitespace character but not others. The trimmed variable is set from TrimSpace(key) but the space check uses the untrimmed key — mixing representations in the same function. Suggested action: document that the function expects pre-trimmed input or normalize consistently. Minor, but the mixed logic is a maintenance trap. File: internal/config/keybindings.go:369-388
