In `internal/app/model.go:523`, `selectionEqual` uses `reflect.DeepEqual(a.Issue, b.Issue)` to compare IssueSummary values. This:

1. Imports the reflect package unnecessarily — `IssueSummary` is a struct with only simple types (strings, int, []string). A targeted comparison on `ID` field (or ID + other key fields) is sufficient and much faster.
2. reflect.DeepEqual is fragile to struct evolution — if someone adds a pointer field, time.Time, or interface to IssueSummary, DeepEqual behavior changes silently.
3. The actual use case is detecting whether the selected issue changed, which only needs ID comparison. If the same issue's details changed (e.g. status update), the detail load path already handles reload.

Replace with `a.Issue.ID == b.Issue.ID` (or compare the handful of relevant fields if you need more than ID).
