Discovered from user review after epic beads-workbench-90w was closed. Expected: the Metadata pane should support actionable field navigation with up/down and Enter for a constrained set of editable fields in detail mode. Actual: current behavior only meaningfully supports status and does not expose a true field-by-field navigation/editing model. This is follow-up product work rather than a regression bug.

## Scope Constraint
Limit this follow-up to a small v1 metadata interaction surface:
- editable fields: `Status` and `Priority` only
- non-editable informational rows remain read-only

## Interaction Model
- Up/down moves selection between editable metadata fields only.
- Enter on `Status` opens the existing status dialog.
- Enter on `Priority` cycles through priorities in place (`P0 -> P1 -> P2 -> P3 -> P4 -> P0`).

## Acceptance Criteria
- [ ] Metadata field navigation is explicitly limited to Status and Priority for this follow-up.
- [ ] Up/down navigates between Status and Priority only.
- [ ] Enter on Status triggers the existing status edit flow.
- [ ] Enter on Priority cycles priority through P0..P4 and wraps back to P0.
- [ ] Non-editable informational rows remain readable but are not treated as actionable fields.
