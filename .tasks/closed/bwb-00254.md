## Description

Search mode uses a hardcoded `defaultSearchLimit = 40` (`internal/mode/search/model.go:16`) regardless of terminal height, the same class of problem fixed for the dashboard in beads-workbench-xtoo. The search result list may show fewer results than fit on screen, or request more than are visible.

This is a follow-up to the dashboard accuracy epic (beads-workbench-xtoo) and should be addressed separately after that epic closes.

## What to investigate
- How does the search result list determine its visible height?
- Is it the same `FormSection` / `height - 3` pattern, or a different renderer?
- Can `defaultSearchLimit` be replaced with a dynamic height-based cap using the same `sectionItemCapacity` approach introduced in beads-workbench-j1m5?

## Acceptance Criteria
- [ ] Search result list requests as many results as fit in the visible area (no hardcoded 40)
- [ ] Resize does not re-issue search queries (same policy as board)
- [ ] `go test ./...` passes
