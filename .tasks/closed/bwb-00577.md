## Description

`internal/mode/details/model_test.go` (~682 lines) is entirely unit-level — all assertions run directly on `HandleKey` outputs. There is no `teatest` coverage exercising real Bubble Tea runtime behavior for the details mode: focus switching across content/metadata/dependencies panes, scroll persistence within a focus, and reset on selection change.

Identified during 2026-05-16 audit. Independent of beads-workbench-db0z.7 (which fixes the scroll-reset bug); this ticket adds the broader runtime coverage that would catch similar regressions.

   - Open detail on issue A
   - Scroll content pane to offset > 0
