## Summary

Discovered while implementing beads-workbench-24is (per-process log filenames). The persistent log now writes to `~/.local/state/bwb/bwb-<session_id>.log`, one file per BWB session. lumberjack's MaxAge (30 days) only prunes rotated backups of the *current* process's own file — it never touches other sessions' files. Sessions that terminate without rotating leave their `bwb-<session_id>.log` behind indefinitely.

## Impact

Disk usage on machines with many short BWB sessions accumulates O(sessions) small log files in the state dir. No functional defect — purely housekeeping.

## Acceptance

- [ ] On startup (or sink construction), prune `bwb-*.log` files in the state directory older than the rotation MaxAge (30 days).
- [ ] Do not delete the current session's own files.
- [ ] docs/MONITORING.md reflects the cleanup behavior if user-visible.
