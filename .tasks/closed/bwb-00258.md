internal/testing/ui/scenarios.go defines hardcoded key sequences (BoardToSearchKeys returns ctrl+space, OpenDetailKeys returns enter, etc.) that must match the keybinding configuration in internal/config. The project supports user-configurable keybindings (documented in CODING.md). These scenario helpers hardcode the defaults but have no connection to the actual default keybinding definitions.

If someone changes the default keybinding for open_detail from Enter to Space, these helpers silently become wrong and scenario tests pass for the wrong reason (they send keys that no longer match the bound action).

Suggested action: either (a) derive these from the default config keybinding definitions programmatically, or (b) add a comment at each helper documenting which config action it corresponds to, so a keybinding change has a clear audit trail to update.
