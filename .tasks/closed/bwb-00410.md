## Description
Create one shared compact issue-row renderer so board and search use the same metadata hierarchy, selection styling, truncation rules, and updated architecture documentation.

## Instructions
1. Introduce a small shared UI component under `internal/ui/shared/` for rendering one compact issue row with these inputs: id, title, type, status, priority, selected state, and target width.
2. Decide whether the shared renderer accepts `domain.IssueSummary` directly or defines its own input struct. If a new struct is introduced, document why `domain.IssueSummary` is insufficient. Eliminate `board.Row` if it becomes redundant.
3. Move row-level responsibilities into that component:
   - selected-row prefix rendering
   - type/priority/status/id/title assembly
   - width-aware title truncation
   - styled vs plain width-safe rendering behavior
4. Keep token-level concerns in existing helpers where appropriate (normalization, compact token text, compact ID derivation), but avoid splitting final row assembly across board/search again.
5. Explicitly unify the existing id-width discrepancy: board currently uses `minInt(12, MaxInt(7, width/5))` while search has no upper cap. Adopt one shared formula unless a documented reason for divergence remains.
6. Update `internal/ui/board/board.go` to use the shared row component instead of assembling rows inline.
7. Update `internal/ui/search/search.go` to use the shared row component so search inherits the same compact metadata styling semantics as board.
8. Board and search must both use the shared row renderer for selection indicator, type/priority/status/ID token assembly, and width-aware truncation. Any intentional rendering difference must be documented as a renderer parameter, not a separate code path.
9. Fold in the documentation work originally scoped in `beads-workbench-ekp.1`: update `docs/CODING.md` (and `project-plan/ARCHITECTURE.md` only if truly needed) so row, optional list, panel, and detail responsibilities reflect the implemented extraction.
10. Add tests for the shared row component plus board/search integration tests that prove rendering consistency and deterministic truncation. Regenerate impacted search goldens and verify the diff because search snapshots will change when it adopts the shared renderer.
11. Resolve or supersede follow-up bug `beads-workbench-b9u` as part of this work.
