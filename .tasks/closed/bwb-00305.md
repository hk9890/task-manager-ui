In keybindings.go:399-474, allowedActionsForContext allocates a new map every time it is called. It is called from: (1) package-level var initialization (allowedMappingKeys, allowedSequenceMappingKeys), (2) sanitizeOverride, (3) buildContextBindings — so it runs multiple times per config load. The maps are static and never mutated. Currently this wastes allocations and makes the code longer than necessary. The same function body is also a 70+ line switch that repeats the exact constant slices already used in DefaultKeyBindings(). Suggested action: make allowedActionsForContext return from pre-built package-level map[string]map[string]struct{} instead of rebuilding every call, or derive from DefaultKeyBindings() to stay DRY.
