In service.go:93-98, TemplateInterpolator.Interpolate sorts placeholder keys before substitution, with the comment rationale presumably being deterministic output. But the placeholders are all distinct mustache-style tokens ({{issue.id}}, {{issue.title}}, etc.) that cannot overlap or conflict — no placeholder is a substring of another. The replacement order is therefore irrelevant to the output.

The sort adds a conceptual burden ('why is this sorted?') and an unnecessary allocation. Replace with a direct range over the map:
```go
func (TemplateInterpolator) Interpolate(input string, ctx InterpolationContext) string {
    result := input
    for key, val := range ctx.Placeholders() {
        result = strings.ReplaceAll(result, key, val)
    }
    return result
}
```
