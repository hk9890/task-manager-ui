## Description
Verifier found that the standalone board controller and renderer exist, but the actual app runtime still uses the older list-only board path inside `internal/app/model.go`. As a result, the closed board task is not truly integrated into the shipped app path.

## Instructions
1. Replace the root app's list-only board rendering/loading path with the real `internal/mode/board` controller.
2. Keep the app shell as the owner of shared selected-issue/detail state, but forward board controller messages so real board navigation updates shell selection and detail loading.
3. Ensure the app's board surface renders dashboard columns/sections from the built-in dashboard provider and supported gateway query types.
4. Remove or refactor duplicate list-only board code from `internal/app/model.go` as needed.
5. Add or update app-level integration tests proving the real board mode is active in the runtime path and board navigation updates shell detail state.
6. Run `go build ./cmd/bwb`, `go vet ./...`, and `go test ./...`.
