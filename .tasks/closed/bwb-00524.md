## Description
Introduce a small CLI bootstrap layer ahead of Bubble Tea startup so BWB can expose standard flags and config inspection/validation flows without requiring a TTY.

## Instructions
1. Refactor `cmd/bwb` startup so flag parsing and exit-code handling happen before config loading + TUI launch.
2. Add `-h/--help`, `-v/--version`, `-c/--config`, `--print-config`, and `--check-config` using the stdlib `flag` package plus explicit short/long aliases.
3. Make `--help`, `--version`, `--print-config`, and `--check-config` exit without attempting to open the TUI.
4. Extend config loading so an explicit `--config <path>` overrides the current XDG-resolved default; when `--config` is explicitly provided and the path does not exist or is not a regular file, exit non-zero with a clear error.
5. Implement `--print-config` as a fully-resolved config dump to stdout using the repo's config shape in YAML, with a leading `# source: <path>` comment; nothing is written to stderr on success.
6. Implement `--check-config` as a validate-only path: load the resolved config, emit warnings to stderr, print a short success message to stdout, and exit `0`; return `1` on load/validation errors.
7. Add version output with a sensible local fallback such as `dev`; this task defines the runtime `--version` behavior, while release-build metadata wiring is handled separately.
8. Add tests that exercise the non-interactive paths under redirected stdout/stderr and verify exit codes plus that Bubble Tea startup is never reached.

## Files to Modify
- `cmd/bwb/main.go`
- `cmd/bwb/*.go` (new CLI/bootstrap helpers and tests as needed)
- `internal/config/load.go`
- `internal/config/load_test.go`

## Acceptance Criteria
- [ ] `go run ./cmd/bwb --help` prints usage text, exits `0`, and does not fail on missing TTY
- [ ] `go run ./cmd/bwb --version` prints a version string and exits `0`
- [ ] `--config <path>` overrides the default config lookup path
- [ ] An explicitly provided `--config <path>` that is missing or not a regular file exits non-zero with a clear error
- [ ] `--print-config` prints the fully-resolved config as YAML to stdout with a leading `# source: <path>` comment and writes nothing to stderr on success
- [ ] `--check-config` validates config, reports warnings to stderr, prints success to stdout, and exits `0`; config load/validation failures exit `1`
- [ ] Unknown flags and missing required flag values exit `2`
- [ ] Tests cover redirected stdout/stderr non-interactive execution and assert that Bubble Tea startup is not reached
