## Description
Build a fast, reliable UI verification workflow for Beads Workbench so agents can review real app behavior without depending on user confirmation. The primary path should extend the existing in-process embedded-fixture Bubble Tea verification approach (real gateway, real model, real key sequences, real View output) because it is already fast and deterministic. Only add process-level visible-screen capture where a concrete gap remains that the in-process path cannot cover.

## Goals
- Close the specific verification gaps that recent runtime bugs exposed
- Verify layout, navigation, search typing/filtering, and detail behavior quickly
- Keep the default workflow fast, deterministic, and stable enough for repeated use during development
- Handle editor/launcher subprocess flows explicitly rather than leaving them undefined
- Document both automated and manual UI review workflows clearly

## Why Now
Recent runtime issues showed that current tests still leave gaps in how quickly and confidently agents can validate the UI. Local experiments also showed that naive real-terminal capture approaches are fragile, so this work must be deliberate and architecture-aware rather than a generic "capture the screen somehow" project.
