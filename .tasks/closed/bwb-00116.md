## Summary

The P1 ClearLabels bug (`beads-workbench-py38`) shipped because the only unit test asserted the wrong bd flag against a stub executor. Add an integration-level test (RunWriteContract sub-test or real_write_contract_integration_test.go entry) that exercises `UpdateIssue(ctx, id, {ClearLabels: true})` against real bd and confirms the labels are actually cleared in the resulting issue.

## Discovered during

`beads-workbench-ibif` Write audit (parent epic: `beads-workbench-yspw`).

## Rationale

Stub tests confirm argv shape but cannot validate that bd accepts the flag. Without an integration counterpart, the next wrong-flag bug would slip through identically.

## Implementation

Add a `RunWriteContract/UpdateIssue/ClearLabels` sub-test that:
1. CreateIssue with labels=`["a","b","c"]`.
2. UpdateIssue with `ClearLabels:true`.
3. ShowIssue and assert `Labels` is empty.

The test must run against both fake (where `ClearLabels` should also be modeled — separate gap) and real bd.

## Acceptance criteria

- [ ] Sub-test added to `RunWriteContract`.
- [ ] Test passes against real bd (after `beads-workbench-py38` is merged).
- [ ] Fake also implements label-clear semantics for the test to pass on both tiers.
