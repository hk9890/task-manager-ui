## Description

`internal/repository/filestorage/filestorage.go:206` uses `bufio.NewScanner(f)` with the default 64 KiB max token size. A single `SnapshotIssue` line aggregating long Description + Notes + Comments can exceed 64 KiB; `scanner.Scan` then returns false with `scanner.Err() == bufio.ErrTooLong` and Load fails wholesale. Hydrate logs a warning and cold-starts; the warm-hydrate feature is unusable for any project containing one oversized issue.

## Instructions

1. After `scanner := bufio.NewScanner(f)`, call `scanner.Buffer(make([]byte, 0, 64*1024), 16*1024*1024)` (or a higher cap if there's reason — 16 MiB matches the order of "a very long markdown design doc"). Document the choice with a short comment.
2. Keep the rest of the loop unchanged.

## Test

Add `TestLoad_LargeIssueLine` that:

2. Calls `Save(r, path)`.
3. Calls `Load(path)` and asserts no error.
4. Asserts the loaded repository's `Issue(...)` returns the long Description intact.
