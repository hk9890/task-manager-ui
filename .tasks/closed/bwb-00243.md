## Summary

`internal/gateway/beads/interface.go:97-104` documents the actor env var as `BD_ACTOR`. Real bd 1.0.4 reads `BEADS_ACTOR` (per `bd create --help`: `--actor default: $BEADS_ACTOR, git user.name, $USER`). Both vars are stripped by `filterEnvToAllowlist` so behavior is unchanged, but the doc is misleading.

## Discovered during

`beads-workbench-ibif` Write audit (parent epic: `beads-workbench-yspw`).

## Fix

Replace `BD_ACTOR` with `BEADS_ACTOR` in interface.go. Also audit `internal/gateway/beads/runner.go` for the allowlist comment and confirm BEADS_ACTOR is correctly excluded.
