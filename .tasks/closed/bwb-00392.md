## Description
Remove the weak generic mode controller boundary. The shell should own concrete board and search controllers plus the shell-owned detail model.

## Instructions
1. Update `internal/app/model.go` to store concrete board and search controllers directly instead of generic `mode.Controller` values.
2. Update `internal/mode/board/model.go` and `internal/mode/search/model.go` so their `Update` methods no longer return `mode.Controller`. Use explicit concrete return types or another package-local non-generic convention that keeps ownership concrete.
3. Refactor shell message forwarding in `internal/app/model.go`, including board/search forwarding helpers, to call concrete update methods and reassign concrete controller state without polymorphic controller storage.
4. Reduce `internal/mode/contracts.go` to the smallest shared IDs, messages, and selection types still needed after the refactor. Remove controller interface machinery if it is no longer needed.
5. Eliminate the current shell-side capability recovery paths that type-assert through `mode.Controller`: selection provider, shell key capture, loading reporter, and result counter.
6. Keep detail state shell-owned in `internal/mode/details` and do not introduce a new top-level mode registry or replacement generic controller abstraction.
7. Update board/search constructors, message flows, and tests to match the explicit ownership model.

## Files to Modify
- `internal/app/model.go` — update shell ownership, forwarding, and remove generic controller usage
- `internal/mode/contracts.go` — remove or shrink generic controller contract
- `internal/mode/board/model.go` — align concrete board ownership and update return convention
- `internal/mode/search/model.go` — align concrete search ownership and update return convention
- `internal/mode/details/model.go` — keep detail state contract explicit
- `internal/app/model_test.go` — update shell tests
- `internal/mode/board/model_test.go` — update board tests as needed
- `internal/mode/search/model_test.go` — update search tests as needed

## Acceptance Criteria
- [ ] `internal/app/model.go` no longer stores board and search as `mode.Controller`
- [ ] Board and search update wiring no longer depends on `mode.Controller` return values
- [ ] Shell code no longer uses type assertions for selection provider, shell key capture, loading reporter, or result counter
- [ ] Board, search, and detail behavior remains unchanged in existing tests or updated equivalent coverage
- [ ] No new top level mode abstraction is introduced
