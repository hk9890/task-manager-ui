Discovered during live review of the detail metadata pane after prior selection styling fixes. Expected: the metadata pane should use the app-wide blue `› ` selection pattern, but the entire pane should reserve that two-character selection gutter so **all** lines remain aligned. Actual: selected/editable rows still use a different indentation pattern than other metadata lines, so titles, fields, divider rows, and labels do not share one clean left edge.

## Required UX Rule
- Keep the application-wide selection marker style: `› ` (blue indicator + trailing space).
- Do **not** invent a metadata-specific selection pattern.
- Reserve the same 2-character gutter for every metadata line so all text starts in the same column.
- Selected row uses `› `.
- Non-selected rows use `  `.
- The selection marker glyph and its styling must be sourced from a single shared helper in `internal/ui/styles` (already reused by issue-row rendering); do **not** introduce a new package, struct, or renderer abstraction.

## Acceptance Criteria
- [ ] The metadata pane reserves a permanent 2-character selection gutter for all rendered lines.
- [ ] Group titles, editable rows, read-only rows, divider rows, and label-group bullet lines all align to the same content column.
- [ ] The selected metadata row uses the standard app-wide `› ` selection marker.
- [ ] Non-selected metadata lines use `  ` in that gutter so alignment stays stable.
- [ ] The 2-character gutter is reserved regardless of focus state; switching focus into/out of Metadata does not shift any line horizontally.
- [ ] The selection marker glyph and styling are sourced from a single shared helper in `internal/ui/styles`; no new package, struct, or renderer abstraction is introduced.
- [ ] Truncation behaves correctly with the styled marker and preserves display-width alignment.
- [ ] Very narrow widths degrade safely instead of producing broken output.
- [ ] Automated tests cover: (a) unfocused gutter reservation, (b) focused selected-row marker only on the selected row, (c) label-row alignment, and (d) divider alignment.
- [ ] Real-app review confirms the metadata pane has one clean left edge and no indentation mismatch.
