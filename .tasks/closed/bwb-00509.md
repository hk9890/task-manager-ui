## Description

Five fake-based unit tests in `internal/app/model_test.go` and `internal/app/scenario_workflow_test.go` pay 3–12s each because `runBatch` calls each tea.Cmd synchronously, including the 60s `tea.Tick` returned by `scheduleRefreshTickCmd()` inside `Model.Init()`. The block ends early (we see exactly ~3s per call) — likely because something else in the batch returns first via channel select — but the cost still dominates the unit tier.

Affected tests (~30s total wall):
- `TestModelReusableDetailToolScenarioCoversEditorAndLaunchersWithFakes` (12s)
- `TestModelBuiltInLauncherHotkeysUseLauncherService` (9s)
- `TestModelMutationResultMarksBrowseDirtyAndRefreshesOnlyActiveSurface` (3s)
- `TestModelEditIssueActionUsesEditorServiceAndUpdatesDetail` (3s)
- `TestModelEditHotkeyUsesEditorService` (3s)

## Why

These are *pure unit tests*. They should run in milliseconds, not seconds. The auto-refresh tick is irrelevant to what they assert.

## Instructions

1. There is already a test seam: `withRefreshTickScheduler(t, scheduler func() tea.Cmd)` in `internal/app/model_test.go` (around line 2772). It swaps the package-level `scheduleRefreshTickCmd` var for the duration of one test.
2. Audit the 5 affected tests: add `withRefreshTickScheduler(t, func() tea.Cmd { return nil })` at the top of each. This replaces the 60s tick with a no-op for the test's duration.
3. **Better**: investigate whether there's a more general place to install the no-op — e.g. inside `mustNewModel` for tests that don't explicitly want the tick. If yes, prefer that over per-test opt-out, but only if it doesn't break tests that DO care about the refresh tick (search for existing usages of `withRefreshTickScheduler` first to make sure those still get their custom scheduler).
4. Re-run `go test -run 'TestModelEditHotkey|TestModelEditIssueAction|TestModelMutationResultMarks|TestModelBuiltInLauncherHotkeys|TestModelReusableDetailTool' ./internal/app -count=1 -v` and confirm each test now completes in <100ms.
5. Verify no other unit test got faster *or slower* in a way that suggests we broke an intentional dependency on the tick.

## Files to Modify

- `internal/app/model_test.go` — add tick-disable calls (or refactor `mustNewModel`)
- `internal/app/scenario_workflow_test.go` — same
