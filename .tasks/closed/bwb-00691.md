## Description

Make skeleton rows pulse in sync with the existing header spinner tick. Add `SkeletonPhase int` plumbing from `m.spinnerFrame` (in `internal/app/model.go`) through the mode-level View signatures (board/search/details) into `issuerow.RenderCompactSkeleton`, and have the skeleton renderer pick one of 3 muted shades by phase so placeholders breathe once every ~1.2 seconds.

## Why

Static skeletons look dead. The header braille spinner animates, but it's far from the loading surface; the user can't tell loading from stuck. A slow synchronized pulse on the surface skeleton fixes this without per-cell shimmer complexity.

## Instructions

### 1. Centralize the phase function

In `internal/ui/loading/loading.go`, add:
```go
// SkeletonPhase returns the skeleton color-cycle index for a given spinner
// frame counter. Phase advances every 4 frames (~400 ms at the 100 ms spinner
// tick), giving a full 3-shade cycle every ~1.2 s.
func SkeletonPhase(frame int) int {
    return frame / 4
}
```
And the matching test (`internal/ui/loading/loading_test.go`).

### 2. Define the 3-shade palette

In `internal/ui/styles/colors.go` (or wherever `TextMutedColor` lives — check the file):

```go
// SkeletonShades cycles through 3 muted lipgloss.AdaptiveColor values to
// produce a ~1.2 s breathing pulse on cold-start placeholders. Hand-tuned
// for both dark and light themes; the dark-theme dynamic range is wider
// than light by design (TextMutedColor on light themes is close to the
// background).
var SkeletonShades = []lipgloss.AdaptiveColor{
    {Light: "...", Dark: "#454545"}, // dim
    {Light: "...", Dark: "#696969"}, // mid (matches TextMutedColor dark)
    {Light: "...", Dark: "#7F7F7F"}, // bright
}
```

Pick the light-theme hex values by hand (avoid "..." in the actual code). If the light-theme adaptive set comes out too flat to perceive as motion, document the tradeoff in a code comment — light theme may pulse less visibly than dark. Don't fall back to "use only one color on light theme" silently; ship as designed and let T4 verification flag if it's unusable.

### 3. Extend `RenderCompactSkeleton` to accept the phase

In `internal/ui/shared/issuerow/row.go`:
```go
type SkeletonOpts struct {
    Width  int
    Seed   int
    Phase  int    // styles.SkeletonShades index (modulo applied internally)
    Styled bool
}
```
Pick foreground color as `styles.SkeletonShades[((opts.Phase % N)+N) % N]` (N = `len(styles.SkeletonShades)`) instead of `TextMutedColor`.

### 4. Plumb phase through mode View signatures

State construction is in **mode models, not app/model**. Add `spinnerFrame int` (or `skeletonPhase int`, cleaner) as a parameter to:

- `internal/mode/board/model.go` — `View()` method (line ~281 where it constructs `board.State`). Add `skeletonPhase int` parameter; populate `board.State.SkeletonPhase`.
- `internal/mode/search/model.go` — `View()` method (line ~338); same.
- `internal/mode/details/model.go` — `View(...)` method (lines ~80, ~99); same.

Then in `internal/app/model.go`:
- Find every call site to `m.board.View()`, `m.search.View()`, `m.detail.View(...)` (line ~949-956).
- Pass `loading.SkeletonPhase(m.spinnerFrame)` as the new argument.

Add `SkeletonPhase int` to each State struct:
- `internal/ui/board/board.go:State` 
- `internal/ui/search/search.go:State`
- `internal/ui/details/details.go:State`

Plumb `SkeletonPhase` through to every `issuerow.RenderCompactSkeleton(...)` call added in T1/T2.

### 5. Tests

- `internal/ui/shared/issuerow/row_test.go`: at the same `Width`/`Seed`, `RenderCompactSkeleton({Phase: 0, ...})` and `RenderCompactSkeleton({Phase: 1, ...})` produce different styled output but identical plain text (strip ANSI for comparison). Three distinct outputs across Phase ∈ {0, 1, 2}.
- `internal/ui/loading/loading_test.go`: `SkeletonPhase(0) == SkeletonPhase(3)`; `SkeletonPhase(0) != SkeletonPhase(4)`; `SkeletonPhase(-1) == 0` (or document chosen behavior for negatives).
- `internal/app/model_loading_refresh_test.go`: after several `loading.TickMsg` dispatches, the rendered View string changes (skeleton phase advanced), while the ANSI-stripped plain text is identical when a skeleton is visible.
