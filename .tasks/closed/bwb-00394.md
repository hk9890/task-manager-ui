## Description
Create a shared markdown renderer package for Beads Workbench viewing surfaces. Use a terminal focused Go library rather than hand formatting markdown in each component. This task is strictly about read only viewing inside the app.

## Instructions
1. Add a dedicated markdown rendering package under internal/ui/shared or equivalent.
2. Prefer charmbracelet/glamour as the default integration choice for ANSI terminal markdown rendering.
3. Expose a small width aware API for read only rendering.
4. Include deterministic fallback behavior when rendering fails, content is empty, or input is effectively plain text.
5. Decide and document the renderer test strategy up front: either assert raw ANSI output intentionally or strip ANSI sequences and assert readable text plus structural markers.
6. Flag that downstream detail and search goldens may need refresh after integration.
7. Do not touch external editing or issue edit document flows.

## Files to Modify

- go.sum
- internal/ui/shared/markdown/renderer.go
- internal/ui/shared/markdown/renderer_test.go
