## Description
The /project-review-test review (2026-05-25) identified six concrete defects in the test suite. The unit suite is honest and fast (2.7s, 1060 tests) and parity tests against real bd carry weight, but the suite does not deliver the confidence the docs claim:

1. **bd-1.0.4 workaround branches in `internal/repository/beads/lean_writes.go` have zero targeted coverage.** Both the `leanIsCloseNotFound` re-close idempotency path (lines 107-149) and the `ClearLabels` `--remove-label` workaround (lines 81-95) are documented workarounds for upstream bugs (gastownhall/beads#4025) — exactly where regressions hide when bd ships 1.0.5.

2. **`docs/TESTING.md` references contract-harness symbols that do not exist** (`RunReadContract`, `TestRealRepositoryIssueLifecycleScenario`, `TestRealRepositoryLinksAndDepsScenario`). They live only inside an abandoned `.claude/worktrees/5q6t/` worktree. The actual harness is `TestRepositoryContract` in `internal/repository/contract/parity_integration_test.go` with a different shape.

3. **Two unit-tagged test files fork real subprocesses**, contradicting docs/TESTING.md's "<100ms, no fork" unit-suite contract: `internal/launcher/process_runner_test.go` (forks sleep, sh) and `cmd/bwb/architecture_guardrails_test.go` (forks `go list`).

4. **Loadgen integration tests (~131s wall) are dominated by always-true assertions** (`MaxMs < 0`, percentile `< 0`, `ActualEdges < 0`) on values that cannot be negative. The substantive risk — percentile distribution sanity, percentile↔Max relationship, report payload structure — is not asserted. Three E2E tests account for ~100s and do not earn their cost.

5. **`scripts/verify_kb8e_manual_test.go` embeds an absolute developer-specific bd binary path.** Build-tag-gated so it doesn't break CI, but no other developer can run it.

6. **`internal/app/edit_flow_teatest_test.go` follows three condition gates with hardcoded `time.Sleep(200ms)`** to let the BubbleTea runtime dequeue `editIssueResultMsg`. Latent CI-flake risk; the three sleeps are also the slowest individual unit tests in the suite.

## Outcome
A test suite that:
- protects the bd-1.0.4 workaround branches against silent regressions
- documents only contract-harness symbols that exist
- restores the "no subprocess forks in unit suite" contract
- replaces theatre assertions with substantive ones in the slowest integration package
- contains no developer-specific paths or unobservable sleeps
