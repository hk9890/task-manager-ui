## Description
Add the genuinely missing assertion helpers and fast invocation paths so agents can run UI verification quickly without rebuilding custom checks each time.

## Scope
Focus on capabilities beyond the existing helpers in `internal/testing/ui`:
- reusable assertions for visible startup/layout sanity and absence of obvious runtime error panels
- reusable assertions for scenario-driven navigation/search outcomes on rendered output or model state
- a fast default `go test -run` style path for common UI verification loops using the deterministic in-process strategy chosen in gyg.1
- avoid duplicating helpers that already exist unless they are being consolidated into clearer entrypoints

## Expected Outcome
This task should make the deterministic UI workflow quicker to invoke and more uniform to assert on, especially for startup sanity, error absence, search behavior, and workflow scenarios introduced by gyg.2.

## Files to Inspect
- `internal/testing/ui`
- scenario code from gyg.2
- `internal/app/`

- [ ] New helpers cover startup/layout sanity and absence of obvious runtime error states without bespoke per-test string loops
- [ ] New helpers can validate search typing/filtering and navigation outcomes on scenario-driven tests

- [ ] The fast path is suitable for repeated use during active implementation
