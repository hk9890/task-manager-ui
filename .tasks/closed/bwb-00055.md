Discovered from user review after epic beads-workbench-90w was closed. Expected: in the Dependencies pane, up/down keys should provide the same visible issue selection behavior as the dashboard, including shared row styling/color treatment for the selected dependency and non-selected dependency rows. Actual: detail-mode state tracks a related selection index, but dependency rendering hardcodes `renderReferenceRow(... Selected:false, Styled:false)`, so the pane does not show dashboard-like selection or ANSI/styled row rendering. Repro: focus Dependencies in detail mode and use up/down keys on a detail item with multiple dependency/structure rows. Impact: dependency browsing is confusing and the pane does not behave like a proper issue list.

## Acceptance Criteria
- [ ] Dependency rows render through the shared compact issue-row path with styled output enabled.
- [ ] The currently selected dependency row uses the same visible selected treatment as dashboard rows.
- [ ] Up/down in Dependencies moves the visible selected row predictably.
- [ ] Enter still opens the selected dependency issue.
- [ ] The fix applies to dependency/structure rows rendered in this pane without introducing a separate one-off row renderer.
