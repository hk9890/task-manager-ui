## Summary

`internal/gateway/beads/interface.go:247-249` says:

> An issue with stored status "open" and no blocking deps is ready. An issue with stored status "in_progress" and no blocking deps is also returned by bd ready (bd ready is not limited to status=open).

This is **wrong** for bd 1.0.4. `bd ready --help`:

> Show ready work (open issues with no active blockers). Excludes in_progress, blocked, deferred, and hooked issues.

Verified in audit workspace `/tmp/tmp.YxiJqI6tmt`: an issue with status `in_progress` and no dependency blockers does NOT appear in `bd ready --json`.

## Discovered during

`beads-workbench-oyb2` Read-lists audit (parent epic: `beads-workbench-yspw`).

## Impact

Doc-only. The fake at `internal/testing/fakes/fake_contract_test.go:84-89` incidentally matches real bd (only seeds open into ReadyIssuesResponse), so the contract test suite is unaffected. But the interface contract misleads anyone reading the gateway docs.

## Fix

Update `interface.go:243-250` to remove the false in_progress claim and clarify that bd ready returns ONLY open-status issues with no dependency blockers.

## Acceptance criteria

- [ ] interface.go ReadyIssues postcondition reflects real bd 1.0.4 behavior.
- [ ] Add a contract assertion (or test) that an in_progress no-blockers issue is NOT returned by ReadyIssues.
