## Description
The current search UX feels broken because query, results, content, and metadata are all treated as peer focus states, even though query and results behave like a vertical rail. Redesign search interaction so the query becomes the primary entry state, search execution is explicit, and Up/Down vertically connects query with results while keeping result live preview behavior.

## Interaction Spec
### Query mode
- Query is the primary focus state and should visually dominate when active.
- Typing edits the query text only; it does not automatically trigger a search request.
- Enter executes the search using the current query text.
- Down moves focus from query to results when results exist.
- Up in query is a no-op.

### Results mode
- Results remain live-preview driven: moving selection updates Content + Metadata without Enter.
- Down moves to the next result.
- Up moves to the previous result.
- If focus is on the first result and Up is pressed again, focus returns to query instead of staying trapped in results.

### Preview panes
- Left/Right continues to move horizontally between Results, Content, and Metadata.
- Content Left returns to Results; Content Right moves to Metadata; Metadata Left returns to Content.
- Query is reached vertically from Results, not as a left/right pane.

## Instructions
1. Update search mode keyboard handling to make search execution explicit on Enter while query is focused.
2. Stop triggering a search on every typed character, backspace, or clear-key in query mode.
3. Implement vertical navigation between query and results: Down from query enters results, Up from first result returns to query, otherwise Up/Down traverses results normally.
4. Preserve result-selection live preview behavior so Content + Metadata refresh automatically as results move.
5. Keep horizontal navigation between Results, Content, and Metadata coherent with the new vertical query/results model.
6. Update help text, focused mode tests, and any search goldens affected by the new query-first interaction.
7. Verify focused search/app packages and the full repository test suite.
