## Description

The built-in `shell-command` launcher template runs `sh -lc "printf … \"{{issue.title}}\" \"{{issue.assignee}}\" \"{{issue.labels}}\""`, interpolating untrusted issue fields directly into the body passed to `sh -lc`. `templateInterpolator.Interpolate` performs literal string replacement with no escaping.

Result: an issue with `Title: $(rm -rf ~)` (or backticks, or `";rm -rf /;"`) executes arbitrary code on the operator's machine when they press `l` on detail mode. Issue titles, labels, and assignee values are written by any contributor/bot — they are operator-untrusted input crossing a trust boundary into a login shell.

This is the single Blocker from the 2026-05-16 codebase audit.

## Files to Modify
- `internal/config/model.go:90-97` (default definition)
- `internal/launcher/service.go:92-106` (interpolation pipeline — review only)
- new test: `internal/launcher/service_security_test.go`

## Instructions
1. Change the default `shell-command` definition so issue fields are passed as positional args, not interpolated into the body. Suggested form:
   ```yaml
   Command: sh
   Args: ["-lc", "printf 'issue=%s\\ntitle=%s\\nassignee=%s\\nlabels=%s\\n' \"$0\" \"$1\" \"$2\" \"$3\"", "{{issue.id}}", "{{issue.title}}", "{{issue.assignee}}", "{{issue.labels}}"]
   ```
   Now `sh` receives the values as `$0..$3`; they are never re-parsed as code.
2. Audit any other built-in or documented examples that use `sh -lc` / `sh -c` with interpolation into the body. None should remain after this change.
3. Add `service_security_test.go` covering the FakeProcessRunner pathway:
   - `Title: "$(touch /tmp/pwned)"` → assert argv contains the literal `$(touch /tmp/pwned)` and the file is NOT created
   - `Title: "\";rm -rf /;\""` → same shape
   - Backticks, semicolons, newlines, `&&`, `||` in title/labels/assignee — all must reach the child as literal data
4. Document the rule in `docs/CODING.md` near the launcher contract: launcher templates MUST NOT interpolate issue fields into the body of `sh -c` / `sh -lc`. Use positional args.
