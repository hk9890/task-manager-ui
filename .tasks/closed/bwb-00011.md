## Description
The current wide-terminal browsing screen is structurally broken, not merely under-polished. The board/dashboard runtime renders all built-in sections side-by-side with no responsive column limiting, which produces unreadable narrow columns and hides useful selected-issue context. The resulting screen feels like a debug skeleton rather than a reviewable product surface.

## Evidence
- User screenshot review (`img.png`)
- Runtime PTY exploration showed severe truncation (`beads-workbench-...` rows), weak selected-issue visibility, and ineffective use of wide-terminal space
- Verifier confirmed that on a wide terminal the expected browse/detail split is not product-acceptable and selected issue context is reduced mostly to footer text

## Root Causes To Address
1. Board column count does not adapt to available width.
2. Board columns can render far too narrow to keep issue IDs/titles readable.
3. The browse/detail split does not sufficiently protect a visible detail preview during board browsing.
4. Board height/layout leaves excessive dead space while failing to surface useful issue context.

## Instructions
1. Make the board layout responsive to terminal width instead of rendering all sections side-by-side unconditionally.
2. Ensure visible board columns remain readable; if necessary, limit simultaneously visible sections and make others navigable.
3. Ensure the wide browse screen shows a clearly visible selected-issue detail/context preview during board browsing.
4. Reduce border/chrome overhead and dead space so more terminal area is devoted to issue content.
5. Update golden/render tests at representative wide-terminal sizes and re-check the real runtime in a PTY after the fix.

## Files to Modify
- `internal/app/model.go`
- `internal/ui/board/`
- `internal/ui/details/`
- related app/board/detail golden tests

## Acceptance Criteria
- [ ] At a representative wide terminal size, the board does not render all sections as unreadably narrow columns
- [ ] Visible board columns preserve enough width to distinguish issue IDs and titles without near-total truncation
- [ ] The selected issue detail/context is visibly present in the main browsing screen, not only in footer text
- [ ] Dead space/chrome is materially reduced compared with the current browsing screenshot/runtime
- [ ] Updated golden/render tests cover the improved wide-terminal presentation
- [ ] Real PTY/runtime re-check shows the browsing screen is reviewable as a product surface
