## Description
Stable `bd --json` output justifies typed command payloads. Keep the gateway boundary but make the CLI adapter simpler and safer.

## Instructions
1. Introduce typed payload structs for the `bd` read commands used by the app, including `list`, `ready`, `blocked`, `show`, `search`, `statuses`, `types`, and `label list-all` as needed.
2. Refactor `internal/gateway/beads/read_gateway.go` to decode into those structs instead of `map[string]any` and field by field extraction helpers.
3. Keep gateway level validation, pagination, and error normalization behavior intact.
4. Update tests and fixtures to cover the typed mapping paths.

## Files to Modify
- `internal/gateway/beads/read_gateway.go` — replace generic map decoding with typed payload mapping
- `internal/gateway/beads/*.go` — add typed payload definitions and helpers as needed
- `internal/gateway/beads/read_gateway_test.go` — update gateway read tests
- `internal/gateway/beads/testdata/*` — refresh or extend fixtures as needed
- `internal/domain/issues.go` — update read models only if typed mapping exposes needed fields already present in stable payloads

## Acceptance Criteria
- [ ] Primary gateway read paths no longer decode command output through `map[string]any`
- [ ] `BeadsGateway` interface remains unchanged
- [ ] Existing list, ready, blocked, show, search, and catalog behavior stays covered by tests
- [ ] Error and pagination behavior remains unchanged from the caller perspective
