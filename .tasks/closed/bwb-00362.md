## Description

Two related logging hygiene problems in `internal/logging/logging.go`:

1. **Interleaved stderr lines under concurrency** — `stderrHandler.WithAttrs` and `WithGroup` (`:230-254`) create new handler instances each carrying their own `sync.Mutex` but sharing the same `io.Writer`. When concurrent components (e.g., `startup` and `gateway` loggers from different goroutines) log, the mutexes do not coordinate, so output lines can interleave. Major finding BS1.

2. **Silent persistent-sink failures** — `buildPersistentSink` (`:127-148`) opens the log file with `os.OpenFile` solely to test writability, then constructs a `lumberjack.Logger` that opens its own descriptor later. If the directory is writable but the file becomes unwritable between probe and lumberjack's first write (RO mount, ENOSPC, quota), nothing surfaces a fallback warning — slog writes silently disappear. Major finding BS2.

## Files to Modify

- `internal/logging/logging.go`
- new test: `internal/logging/concurrency_test.go` (or extend existing logging tests)

## Instructions

1. Share a single `*sync.Mutex` across all derived handlers by storing it as a pointer. `WithAttrs`/`WithGroup` should COPY the pointer, not allocate a fresh `sync.Mutex`. Verify the root mutex is allocated once in `NewStderrHandler` (or equivalent).
2. Wrap lumberjack writes with a handler that, on `io.Writer` error, falls back to stderr-only and emits ONE warning record indicating the persistent sink failed. Hold a sentinel so the warning is emitted exactly once per session.
3. Add a concurrency test:
   - 10 goroutines × 100 records → no interleaved lines (parse each line as valid JSON / valid structured record).
4. Add a write-failure test (or document as untestable):
   - Use a fake `io.Writer` returning ENOSPC; assert one fallback warning is emitted and stderr-only continues.

## Acceptance Criteria

- [ ] Derived handlers share one mutex (verified by code inspection + concurrency test)
- [ ] Persistent-sink write failures surface a fallback warning at most once
- [ ] Concurrency test passes deterministically (no interleaved output)
- [ ] Full quality gate green
