modal/modal.go:219-222: pressing 'n' when not in an input field always emits CancelMsg regardless of whether config.Required is true. But line 204-207 for the Escape key respects Required and blocks cancel. This is inconsistent: Required should mean the user can't dismiss via any path, but 'n' bypasses that guard. Suggested action: add 'if !m.config.Required' check before the 'n' cancel path, matching the Escape behavior on line 205.
