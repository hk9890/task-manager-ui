Finding 3 from the ipq7 audit (PERFORMANCE, not correctness). In Hydrate the catalogsCache restore (caching.go ~398-402) is nested inside 'if !dirty && dashboardHydrated'. A v2 file with a valid persisted CatalogsCache but an empty/zero DashboardCache (degenerate, or saved before any Dashboard fetch) takes the else branch and never restores catalogs even when the hash matches and catalogs are provably fresh -> next Catalogs() re-fetches. Fix: hoist the catalogs restore out of the dashboard-gated branch; gate only on !dirty (hash match) && v2Header.CatalogsCache != nil.
