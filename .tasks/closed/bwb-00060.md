## Description
Simplify the UI core architecture before more dashboard provider work lands. Preserve the intentionally small product shape while removing accidental complexity in shell coordination and CLI decoding.

## Goals
- Keep `BeadsGateway` as the only issue IO boundary
- Flatten board, search, and detail ownership instead of carrying a weak generic top level mode abstraction
- Replace map based `bd --json` decoding with typed payload structs
- Preserve the dashboard provider seam but tighten its contract for the upcoming second provider

## Non Goals
- Add new top level modes
- Introduce plugin architecture or arbitrary dashboard query execution
- Change the launcher or editor product flows

## Success Criteria
- [ ] `internal/app` and `internal/mode` no longer rely on a weak generic controller abstraction plus recovery type assertions
- [ ] Selection and detail synchronization use one clear ownership model
- [ ] `internal/gateway/beads` decodes stable `bd --json` payloads through typed structs instead of generic maps
- [ ] `internal/dashboard` has an explicit validated contract that supports multiple providers without widening scope
- [ ] `docs/CODING.md` and `project-plan/ARCHITECTURE.md` match the resulting package boundaries
- [ ] Acceptance review confirms build, vet, and test guardrails still pass
