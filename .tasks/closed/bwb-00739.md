## Description

`.github/workflows/ci.yml` currently runs `go test ./...` as a single step, taking ~95s and combining unit + integration. After this epic, CI uses mise as the execution layer and runs unit + integration as separate steps so failures are attributable to category and can be parallelized later. Depends on i1th.4 (mise tasks exist with `[tools] go = <version>`).

## Files to Modify

- `.github/workflows/ci.yml`

## Instructions

1. **Toolchain source-of-truth (locked by i1th.4 decision):** mise installs Go from `.mise.toml [tools]`. **Remove `actions/setup-go@v5`** entirely — do not run both, they will install different Go versions and put both on PATH.

2. **Replace the workflow step list** with the structure below. Note: `checkout` MUST remain the first step (the example below is the complete replacement, not just the new bits):

```yaml
jobs:
  quality-gates:
    name: Validate scripts, lint, build, vet, and test
    runs-on: ubuntu-latest
    steps:
      - name: Check out repository
        uses: actions/checkout@v4

      - name: Install mise (handles Go toolchain via .mise.toml)
        uses: jdx/mise-action@v2

      - name: Scripts syntax
        run: mise run scripts

      - name: Lint
        run: mise run lint

      - name: Build
        run: mise run build

      - name: Vet
        run: mise run vet

      - name: Architecture guardrails
        run: mise run guardrails

      - name: Unit tests
        run: mise run test

      - name: Integration tests
        run: mise run test:integration
```

3. **Action pinning convention.** Use `jdx/mise-action@v2` — matches the repo's existing major-pin convention (`actions/checkout@v4`, `actions/setup-go@v5`). Do not pin to a SHA unless the user instructs otherwise.

4. **Verify on a PR branch.** Push a branch with these changes, observe the workflow run, confirm each step succeeds and that "Unit tests" wall time is dramatically shorter than the previous combined step.

## Acceptance Criteria

- [ ] `actions/setup-go@v5` is REMOVED from `.github/workflows/ci.yml` (mise handles Go)
- [ ] `actions/checkout@v4` remains as the first step
- [ ] `jdx/mise-action@v2` is the second step
- [ ] Subsequent steps invoke `mise run <task>` for: scripts, lint, build, vet, guardrails, test, test:integration
- [ ] "Unit tests" and "Integration tests" are SEPARATE workflow steps (each appears in the GitHub Actions UI as its own row)
- [ ] No bare `go test ./...` invocation in the workflow
- [ ] On a PR push, all steps pass; the Unit tests step completes in <30s on the GitHub-hosted runner
