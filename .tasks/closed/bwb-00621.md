## Description

Expand the CI test job to a matrix across ubuntu, macos, and windows. bwb is a TUI — terminal handling, file paths, and shell behavior differ enough across OSes that a single-OS suite misses real bugs.

**Note**: bwb only **ships** to linux + darwin (per `.goreleaser.yaml`). This task only adds **testing** on windows — it does NOT add windows release builds. Catching windows-specific bugs in CI helps Linux/macOS users too (path separators, line endings, environment quirks).

## Files to Modify

- `.github/workflows/ci.yml` — convert `quality-gates` job to a matrix over `os`

## Instructions

1. Replace the single `runs-on: ubuntu-latest` with a matrix:
   ```yaml
   quality-gates:
     name: Validate scripts, lint, build, vet, and test
     runs-on: ${{ matrix.os }}
     strategy:
       fail-fast: false
       matrix:
         os: [ubuntu-latest, macos-latest, windows-latest]
   ```
2. Some steps are platform-incompatible — guard them:
   - `mise run scripts` runs `bash -n` on a shell script; gate this step with `if: matrix.os != 'windows-latest'`
   - `mise run lint` and `mise run guardrails` should be ubuntu-only (slow, deterministic, no OS-specific value) — gate with `if: matrix.os == 'ubuntu-latest'`
   - Build / vet / unit tests / integration tests run everywhere
   - The coverage gate (from `.5`) is ubuntu-only — gate accordingly
3. Verify `jdx/mise-action@v2` supports windows; if not, swap to `actions/setup-go@v5` with `go-version-file: go.mod` for windows.
4. Expect first windows run to surface ≥1 path-separator or fixture issue. Triage: either fix in this PR if trivial, or file a follow-up and skip the failing test on windows with a TODO comment linking the follow-up issue.

## Acceptance Criteria

- [ ] CI matrix runs on ubuntu-latest, macos-latest, windows-latest
- [ ] `fail-fast: false` is set (one OS failing doesn't kill the others)
- [ ] Build + vet + unit tests + integration tests pass on all three OSes (or windows-specific failures are explicitly skipped with linked follow-up issues)
- [ ] Lint/guardrails/coverage remain ubuntu-only
