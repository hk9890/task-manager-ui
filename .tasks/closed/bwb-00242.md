## Summary

`interface.go:380-381` for ShowIssue: "result.Related is merged from related/relates-to dependency types in both Dependencies and Dependents arrays, plus the top-level Related array."

Real bd 1.0.4 NEVER emits a top-level `related` key in `bd show --json` output. All related relationships appear inside `dependencies`/`dependents` arrays with `dependency_type`. The `bdIssuePayload.Related` field is handled defensively (empty input → empty output, no crash). The gateway code is correct, but:
- The contract clause misleads about expected real-bd output.
- `TestGatewayShowIssueMergesTopLevelAndDependencyRelatedWithoutDuplicates` tests a fake scenario that bd cannot produce.

## Discovered during

`beads-workbench-aww9` Read-detail audit (parent epic: `beads-workbench-yspw`).

## Fix

Update the interface.go contract clause to note this field is defensive-only — preserved from a prior bd schema or hypothetical future shape. Add a doc note in the test stating it exercises the defensive code path.
