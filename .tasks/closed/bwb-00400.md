## Description
In `internal/ui/modal/modal.go`, pressing `n` always emits `CancelMsg`, even when `config.Required == true`. The `Escape` key handler correctly checks `!m.config.Required` (line ~205), but the `n` shortcut (line ~219-222) skips this check entirely. Users can dismiss Required modals (e.g. confirmations) by pressing `n`.

## Instructions
1. Open `internal/ui/modal/modal.go`
2. Find the `n` key handler (around line 219-222) in the `Update` method
3. Add the same `!m.config.Required` guard that the Escape key uses:
   ```go
   case "n":
       if !m.config.Required {
           return m, func() tea.Msg { return CancelMsg{} }
       }
   ```
4. Add a test in `internal/ui/modal/modal_test.go` that verifies pressing `n` on a Required modal does NOT produce a `CancelMsg`

## Files to Modify
- `internal/ui/modal/modal.go` (add Required guard to `n` handler)
- `internal/ui/modal/modal_test.go` (add test for `n` on Required modal)

- [ ] Pressing `n` on a non-Required modal still emits CancelMsg (existing behavior preserved)
