Discovered from live user review after metadata field navigation/editing shipped. Expected: metadata actionable-field traversal follows a sensible order, and Enter on an actionable field opens a dialog to change that field. User decision: actionable order must be `Status` first, then `Priority`; Enter on both `Status` and `Priority` should open a dialog; within that dialog the user changes the value and presses Enter to save, or Escape to close without saving. Actual: user reports the metadata selection order feels wrong under up/down traversal, and the current `Priority` behavior does not match the expected dialog-driven change flow.

## Acceptance Criteria
- [ ] Metadata actionable-field order is `Status` first, `Priority` second.
- [ ] Up/Down in Metadata traverses actionable fields in that order only.
- [ ] Enter on `Status` opens the status-change dialog.
- [ ] Enter on `Priority` opens a priority-change dialog (not instant cycle).
- [ ] In both dialogs, Enter saves the changed value.
- [ ] In both dialogs, Escape closes without saving.
- [ ] Real-app verification confirms the full UX flow: select field -> Enter opens dialog -> choose value -> Enter saves / Escape cancels.
