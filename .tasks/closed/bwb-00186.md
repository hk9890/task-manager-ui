details/details.go:153 calls strings.TrimSpace(line) on every line of description/notes/comment text. This destroys meaningful indentation in code blocks, YAML snippets, or nested lists that might appear in issue descriptions. For a tool that displays issue detail, preserving original formatting matters. Suggested action: use strings.TrimRight(line, ' \t') instead of TrimSpace to remove trailing whitespace without destroying leading indentation, or don't trim at all and let TruncateString handle width capping.
