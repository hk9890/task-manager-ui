## Description

Timeboxed (~30 min) investigation. Before we set a CI coverage gate, we need to know the current baseline so the gate is calibrated correctly (~5pts below current, so it ratchets up rather than blocks day-1).

**Scope decision:** measure BOTH unit-only and unit+integration coverage. For bwb, integration tests cover the gateway↔bd boundary (a meaningful chunk of behavior). ug82.5 will gate on **unit+integration combined** so the threshold reflects real coverage rather than under-counting.

## Instructions

1. Run unit-only baseline locally:
   ```
   go test -race -coverprofile=coverage-unit.out -covermode=atomic ./...
   go tool cover -func=coverage-unit.out | tail -1
   ```
2. Run combined unit+integration baseline:
   ```
   go test -race -tags=integration -coverprofile=coverage-all.out -covermode=atomic ./...
   go tool cover -func=coverage-all.out | tail -1
   ```
3. Capture both total coverage percentages.
4. Identify the 3-5 packages with the lowest coverage from the combined run:
   ```
   go tool cover -func=coverage-all.out | sort -k3 -n | head
   ```
5. **Update this task's notes** with:
   - Unit-only baseline %
   - Combined unit+integration baseline %
   - Recommended initial gate (combined baseline minus ~5pts, rounded down — this is what ug82.5 will enforce)
   - Top low-coverage packages so we know what's intentionally untested vs. a gap
6. Hand the recommended gate value (and the combined-scope decision) to `beads-workbench-ug82.5`.

## Acceptance Criteria

- [ ] Unit-only baseline % recorded in notes
- [ ] Unit+integration combined baseline % recorded in notes
- [ ] Recommended initial gate % (from combined) recorded in notes
- [ ] Top low-coverage packages listed in notes
