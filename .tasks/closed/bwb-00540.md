## Description

Audit task. Produce a complete inventory of every distinct argv shape bwb sends to the bd subprocess at runtime. Output: a doc/table that maps each `<consumer>` → `<bd verb + flags>`, with source-code references. This is the input for [[ppja.3]] (per-call argv-cardinality tests).

## Instructions

1. Use grep to find every `runner.Run(ctx, args...)` call site (or equivalent — read `internal/gateway/beads/runner.go` first to know the API surface).
2. For each call site, trace back to the gateway method that owns it, then to the consumer that triggers the gateway method. Build the chain: `<consumer in mode/board, mode/search, app, launcher, etc.>` → `<gateway method>` → `<exact argv>`.
3. Run `mise run bwb` (or its non-interactive equivalent — `mise run smoke --format json` exercises a lot of paths) with a debug logger and capture all argvs from the persistent log (the gateway logs argv per call). Cross-check the static analysis from step 2 against the dynamic observation.
4. Document the inventory in `internal/gateway/beads/doc.go` (or a new `internal/gateway/beads/ARGV_CONTRACT.md`) as a table:
   ```
   | Consumer                         | Method        | argv shape                                            |
   |----------------------------------|---------------|-------------------------------------------------------|
   | board.Init                       | ReadyExplain  | bd ready --explain --json                             |
   | board.Init                       | Query         | bd query status=in_progress --json                    |
   | board.Init                       | Query         | bd query status=closed -a --sort closed --limit 50 ...|
   | board.Init                       | CountIssues   | bd count --by-status --json --status closed           |
   | search.SearchIssues              | Query         | bd query status=open --json (...)                     |
   | ...                              | ...           | ...                                                   |
   ```
5. Flag any argv shape that lacks a corresponding test assertion (the existing `TestBoardInitRealGatewaySubprocessArgvCardinality` covers board.Init; many others are uncovered). This list becomes the work backlog for [[ppja.3]].
6. Apply the "Discovered-during-audit bug discipline" — if you find an argv that's inconsistent with what `interface.go` documents (or with what bd actually accepts), file a bug per the parent epic [[ppja]].

## Acceptance Criteria

- [ ] Inventory doc/table exists covering every distinct argv shape bwb emits
- [ ] Each entry references the consumer (file:line or function name)
- [ ] Static analysis cross-checked against dynamic capture from a real bwb session
- [ ] Argv shapes lacking a pinning test are explicitly listed as the [[ppja.3]] backlog
- [ ] Bugs discovered (if any) filed per discipline and linked back
- [ ] `mise run quality` passes (doc-only changes)
