service_test.go tests three error paths: opener failure, no-changes skip, and successful update. But it does not test what happens when the initial `gateway.ShowIssue` call fails (service.go:61-63). This is the first thing EditIssue does and a likely real-world failure (network issue, invalid issue ID, etc.).

The test infrastructure already supports this — `FakeBeadsGateway.SetError(MethodShowIssue, err)` exists and is unused in these tests.

Suggested fix: Add one test that sets a ShowIssue error and verifies EditIssue propagates it without writing temp files or calling the opener.
