## Description
`internal/mode/details/model.go` calls `uidetails.Render()` in both `HandleKey` (for scroll) and `ClampScroll` just to count rendered lines. This happens on every single scroll key press. The full render is expensive (string building, formatting) and should be cached.

## Instructions
1. Open `internal/mode/details/model.go`
2. Find `HandleKey` and `ClampScroll` methods where `uidetails.Render()` is called
3. Add a `cachedLineCount int` field to the Model struct
4. Compute line count once when the issue data changes (in `SetIssue` or equivalent), not on every key press
5. Use the cached value in `HandleKey` and `ClampScroll`
6. Invalidate the cache when issue data or viewport dimensions change
7. Ensure existing detail mode tests still pass

## Files to Modify
- `internal/mode/details/model.go` (add line count caching)
- `internal/mode/details/model_test.go` (verify caching doesn't break scroll behavior)

## Acceptance Criteria

- [ ] Line count is cached and invalidated when issue data or dimensions change
