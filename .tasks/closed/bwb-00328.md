## Description

`internal/repository/filestorage/filestorage.go` calls `os.CreateTemp("", ...)` which places the temp file in `$TMPDIR` (default `/tmp`). On default systemd-based Linux installs `/tmp` is tmpfs while the cache destination (`~/.cache/bwb/...`) is on the root filesystem — `os.Rename` then fails with `EXDEV` ("invalid cross-device link"). The cache is silently never persisted in production.

Two call sites are affected:
- line 83 for the JSONL temp
- line 121 for the manifest temp

## Instructions

1. Replace `os.CreateTemp("", ...)` with `os.CreateTemp(filepath.Dir(path), ...)` (and the same for `manifestPath` at line 121).
2. Add `path/filepath` import if not already present.
3. The existing `defer os.Remove(tmpPath)` cleanup paths remain correct.

## Files to Modify

- `internal/repository/filestorage/filestorage.go` (lines ~83, ~121)
- `internal/repository/filestorage/filestorage_test.go` (new test)

## Test

Add `TestSaveWithHash_CrossDeviceTempDir` (or expand an existing save test) that:
- creates the destination `path` inside a `t.TempDir()` (which is usually on the same FS as `/tmp` so does NOT reproduce the bug by itself — instead assert via inspection that the temp is created in `filepath.Dir(path)` not `os.TempDir()`).
- One robust strategy: monkey-patch nothing; instead inspect that no orphan temp file remains in `os.TempDir()` after a successful Save AND that an intermediate temp existed under `filepath.Dir(path)` during the call (use a hook or check via `os.ReadDir` before the rename completes — easier: just assert the call succeeds when `os.TempDir()` is overridden to a non-existent path via `TMPDIR=/nonexistent` env var if the test sandbox honors it).
- Simpler alternative: assert SaveWithHash succeeds when `os.TempDir()` returns a path on a different filesystem from `path`. Use the Linux-specific `unix.Statfs` to verify FS-ID equality of the temp with `filepath.Dir(path)` instead — write a small `testFSID(t, path) → uint64` helper.

If a fully deterministic test proves flaky, settle for a static-analysis-style test that reads the source for `os.CreateTemp("",` and fails if found — equivalent to a lint rule. Document the choice in the test body.
