## Description
Implement the minimal helper surface needed so the agent can run better ad-hoc real-app UI verification flows. Focus on practical scripts, not a general-purpose automation framework.

## Required Implementation Scope
### Priority order (MVP first)
1. `wait-for-text` / `wait-for-no-text` support in the PTY helper
2. optional checkpoints / intermediate snapshots
3. separate before/after state wrapper using `bd show --json`

### Extend `scripts/capture_bwb_screen.py`
Implement at least:
- send-key steps
- wait-for-text
- wait-for-no-text
- per-step timeout/error reporting
- wait-phase polling around 50ms
- optional checkpoints or intermediate screen snapshots
- structured JSON output that includes failure/checkpoint context
- guaranteed child cleanup on timeout, failure, crash, or interrupt paths

### Add one small wrapper script
Implement a separate tiny wrapper script that can:
- capture `bd show <id> --json` before
- run a PTY flow through the capture script
- capture `bd show <id> --json` after
- report field changes / no-change for a representative mutation flow

## Representative Flows To Support
- one read-only navigation flow
- one mutation save flow
- one mutation cancel/no-save flow

## Acceptance Criteria
- [ ] The PTY helper supports condition-based multi-step ad-hoc flows.
- [ ] `wait-for-text` / `wait-for-no-text` are implemented as the MVP capability for reliable multi-step flows.
- [ ] Failures identify which step timed out or failed.
- [ ] Wait-phase polling is responsive enough for ad-hoc use (target around 50ms).
- [ ] The wrapper proves a mutation flow with before/after `bd show --json` evidence.
- [ ] Child cleanup is guaranteed on timeout, crash, interrupt, and normal exit paths.
- [ ] Cleanup behavior is validated explicitly enough that orphaned PTY child processes are not left behind.
- [ ] The implementation remains small and script/tooling oriented.
