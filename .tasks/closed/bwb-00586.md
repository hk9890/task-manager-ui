## Description
Make message-driven selection updates the single source of truth inside `internal/app`. Remove polling-based selection syncing and keep `model.go` readable without widening the architecture.

## Instructions
1. Keep the message-based path ŌĆö `SelectionChangedMsg` emitted by board and search ŌĆö as the only selection synchronization mechanism driving detail loading.
2. Remove the polling-based shell selection path, including `syncSelectionFromControllers` and related controller `CurrentSelection()` polling.
3. If extraction materially shrinks `internal/app/model.go`, limit it to focused `internal/app` helpers for selection/detail coordination or shell-owned action dispatch. Do not create new packages, registries, or broad abstractions.
4. If new helper files are needed, keep them local and concrete (for example `internal/app/detail_selection.go` or `internal/app/shell_actions.go`) rather than introducing a new subsystem.
5. Preserve current user-facing behavior for help modal, action modal, detail loading, editor flow, launcher flow, and toast feedback.
6. Update scenario tests and golden files as needed.
