## Description
GitHub release `v0.1.0` exists without any attached binary assets, so `mise install github:hk9890/beads-workbench@0.1.0` fails with `No matching asset found for platform linux-x64`.

## Impact
- Release artifacts are unusable for binary installation.
- The documented release process creates tags/releases without shipping binaries.
- Current and future users cannot install `bwb` via mise GitHub backend.

## Steps to Reproduce
1. Run `mise install github:hk9890/beads-workbench@0.1.0` on Linux x64.
2. Observe failure because the release has no matching assets.

## Fix Requirements
1. Add release automation that builds archives for supported platforms and attaches them to GitHub releases.
2. Ensure archive naming is compatible with mise's GitHub backend platform matching.
3. Update release documentation to reflect the automated artifact publishing flow.
4. Backfill `v0.1.0` release assets so the existing release is installable.

## Acceptance Criteria
- [ ] Repository contains automated release packaging/upload configuration.
- [ ] Release documentation explains how binaries are produced and verified.
- [ ] GitHub release `v0.1.0` has installable assets for at least linux amd64 and darwin arm64/amd64.
- [ ] `mise install github:hk9890/beads-workbench@0.1.0` succeeds on supported platforms once assets are available.

## Files to Modify
- .github/workflows/ (add/update release workflow)
- release packaging config file (create)
- docs/RELEASING.md (update)
