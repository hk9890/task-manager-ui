The `Model.Update()` method in `internal/app/model.go` (lines 155-482) is a 330-line switch/case cascade. It handles:
- Window resize
- Detail loaded
- Edit issue result
- Launch action result
- Mutation catalogs loaded
- Mutation result
- Selection changed
- Action request
- Toast dismiss
- ~20 different key bindings

While it follows the Bubble Tea pattern, the key handling section alone (lines 327-478) is ~150 lines of sequential case matching. This is the largest method in the reviewed code and makes it hard to reason about which keys do what.

Suggested action: Extract key handling into a `handleShellKey(msg tea.KeyMsg) (Model, tea.Cmd)` method, and group the launcher hotkeys (nvim/opencode/shell-command) into a single `handleLauncherKey` since they share the same pattern (check detail mode, get issue context, launch).

This is a SUGGESTION, not a must-fix. The current code works correctly.
