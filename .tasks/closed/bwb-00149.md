## Description

The dashboard currently shows incorrect data in two ways:

1. **Wrong item limit** — each section is capped at a hardcoded 25 items regardless of terminal height. At a 24-line terminal a section can show ~21 rows; the limit should be derived from available height at query time, not baked in at definition time.
2. **Wrong section counts** — section headers either show no total or show the capped number (25) instead of the real count (e.g., Done has 353 issues but shows 25).
3. **Wrong sort field for Done** — the Done section sorts by `updated_at` instead of `closed_at`, so an issue that receives a comment after being closed incorrectly floats to the top.

## Success Criteria
- [ ] Done section sorts by closed_at descending
- [ ] Each section requests exactly as many items as fit in the visible terminal area (dynamic limit from height)
- [ ] Section headers show accurate total counts, not the display limit
- [ ] All child implementation tasks complete
- [ ] Acceptance review task complete
