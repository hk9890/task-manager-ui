## Outcome

bwb runs entirely on the new `Repository` interface. The old `BeadsGateway` package and `FakeBeadsGateway` are deleted. App startup accepts `--repo beads|memory` and `--repo-file <path>` flags so the in-memory backend with a seeded file can be used for fast end-to-end tests and demos.

## Motivation

Epic 1 ships the new layer as library code. This epic makes it real: the app uses it, the old code is gone, and the backend is selectable. Tests get faster (in-memory, no subprocess); the codebase loses ~3000 lines of dead intermediate gateway methods.

## Prerequisite

Depends on Epic [[hhho]] (E1).

## Scope -- 6 implementation subtasks + acceptance review

- 8pxi.1 (was .1+.2) -- ATOMIC migration: app/modes + 18 test files. Cannot ship as separate PRs (build goes red). Tranching is intra-PR.
- 8pxi.3 -- Delete `internal/gateway/beads/` public API + `FakeBeadsGateway`. **Argv runner cache STAYS** (see 8pxi.7).
- 8pxi.4 -- `--repo beads|memory` (default `beads`) and `--repo-file <path>` startup flags.
- 8pxi.5 -- Perf validation: re-run `scripts/analyze_dashboard_perf.py` and confirm post-refactor baseline matches pre-refactor (~5s cold, ~85% hit). Argv cache still present.
- 8pxi.7 -- Delete the argv-level runner cache; re-measure perf. Sequenced AFTER .5 so the baseline comparison is clean.
- 8pxi.6 -- Acceptance review.

Dependency chain: 8pxi.1 -> 8pxi.3 -> 8pxi.5 -> 8pxi.7 -> 8pxi.6 (and 8pxi.4 in parallel after 8pxi.1).

## Out of scope (this epic)

- Cross-session cache invalidation strategy (Epic 3 design)
- bd-show false-positive cache-bust fix -- this requires a different change-detection signal (not just per-ID typed cache; Dashboard is composite). Listed as open question in Epic [[kxci]].
