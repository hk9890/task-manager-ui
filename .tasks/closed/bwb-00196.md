In process_runner.go:19-20, when the launcher definition includes Env entries, `cmd.Env` is set to ONLY those entries. Per os/exec docs, setting Env to a non-nil slice means the child process gets ONLY those variables — no PATH, no HOME, no TERM, nothing from the parent.

This means any launcher definition that uses Env (like the built-in opencode definition with BWB_* vars) will produce a process that cannot find executables, has no home directory, and will likely fail or misbehave in confusing ways.

The fix is to prepend `os.Environ()` so launcher env entries are additive:
```go
if len(env) > 0 {
    cmd.Env = append(os.Environ(), env...)
}
```

File: internal/launcher/process_runner.go, lines 19-20.
