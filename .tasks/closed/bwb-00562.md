## Migration pattern (per file)
- Replace `fakes.NewFakeBeadsGateway()` with `memory.New()`
- Replace `fake.SeedIssue(detail)` with `memory.Seed(...)`
- Tests that used `fake.SetError(method, err)` must be rewritten: if they were testing app error-handling, drive them through a real failure path; if they were testing error formatting, move to a lower-level test against the bd-backed repository
- Tests that used `fake.Calls()` to assert call order need re-thinking -- observable state assertions are preferred over call counting where possible
- Tests that asserted on the specific 5 bd calls being fired should assert on `repo.Dashboard()` being called once (or just on observable result)

## Tranche order (per Epic 1 design)
1. Easy first: `render_regression_test.go` files (read-only, simple seed)
2. Then mode-level tests (board, search, details)
3. Then app-level orchestration tests
4. Last: scenario_workflow and mutation_errors (mutation chains)
