Add a `test:load` mise task that runs the load-test suite end-to-end without requiring the operator to remember env vars.

## Done when

- `mise tasks` shows `test:load`.
- Running it sets up the workload (parameters from env or task args), runs the measured operations, and emits the report.
- It does NOT silently skip — if `bd` is missing or a precondition fails, it errors out loudly with an actionable message.
- The task is documented in `docs/TESTING.md` (or a new `docs/LOAD_TESTING.md`).

## Why this exists

Today scale-fixture tests are gated behind `BWB_SCALE_FIXTURE=1` and silently skip without it. The load-test surface must be a positive affordance ('run this command') not a hidden opt-in.
