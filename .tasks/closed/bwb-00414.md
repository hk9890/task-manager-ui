## Description
Strengthen visual hierarchy so the shell header reads as product chrome rather than always-on help text.

## Instructions
1. Add or reuse styling helpers so the app title is emphasized, the active tab is highlighted, inactive tabs are dimmed, and right-side context/footer hints are visually muted.
2. Keep the style treatment readable in narrow widths and avoid making the footer louder than the board content.
3. Keep shell-specific styling localized to app/UI helpers rather than scattering raw Lip Gloss styling across unrelated call sites.
4. Update tests or goldens if the new styling changes expected rendering output.

- `internal/app/model.go` — apply shell chrome styles.
- `internal/ui/styles/colors.go` — add or reuse shell chrome color tokens.
- `internal/ui/styles/format.go` — add tiny formatting helpers if needed.
- `internal/app/model_test.go` — update expectations if rendering changes.

## Acceptance Criteria
- [ ] Active tab has a distinct highlighted treatment.
