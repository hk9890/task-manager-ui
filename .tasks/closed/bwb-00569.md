## Discovered during

[[8qw9.1]] (gateway contract audit, 2026-05-16).

## Expected vs actual

`bd show --json` omits the `description` field entirely (not `null`, not `""`) when an issue has no description set. The gateway calls `requiredString(primary.Description, "description")` (`internal/gateway/beads/read_gateway.go` ShowIssue, around line 252) which returns `ErrorCodeDecodeFailed` when the pointer is nil. Result: **ShowIssue returns a decode error for any issue created without a description** — a common state, especially for quick `bd create` invocations.

## Verification required

Must verify in the running app (`mise run bwb`) that opening the detail pane for any issue that has no description set shows an error state rather than the detail view. If confirmed visually, fix.

Orchestrator note: code-path verified by reading the decoder + showing detail in this repo via bd CLI; visual confirmation gated on user TTY.

## Suggested fix layer

Gateway (`internal/gateway/beads/read_gateway.go` ShowIssue): change `requiredString(primary.Description, "description")` to `optionalString(primary.Description)` (or equivalent) since an absent description is a valid state, not a decode error.

Add a unit test using a canned `bd show` response that omits `description` and assert ShowIssue returns success with `IssueDetail.Description == ""`.
