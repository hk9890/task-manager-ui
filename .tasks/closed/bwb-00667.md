## Description
Replace the gateway runner's free-form debug callback with central structured logging.

This task should ensure `bd` command execution traces and gateway failures are recorded in the central JSON log with useful machine-readable fields while preserving existing gateway error normalization behavior.

## Instructions
1. Update the gateway runner interface/config so it uses the central logging package rather than a raw `DebugLog func(string)` callback.
2. Emit structured events for `bd` executions that include at least operation, full argv, exit code, and duration; include failure details when available.
3. Keep gateway error normalization behavior intact so logging does not change returned domain errors.
4. Add or update focused tests in `internal/gateway/beads/runner_test.go` for successful execution traces, non-zero exits, and execution failures.

## Acceptance Criteria
- [ ] Gateway execution tracing uses `internal/logging`
- [ ] `bd` execution logs include operation, full argv, exit code, and duration
- [ ] Failure logging does not regress gateway error normalization
- [ ] Focused runner tests cover the new logging path
