## Summary

ReadyExplain audit (`beads-workbench-u53a`) identified 5 missing test/doc items. Consolidated into one ticket because all are small additions to the contractcheck validator + a few unit tests.

## Discovered during

`beads-workbench-u53a` Read-explain audit (parent epic: `beads-workbench-yspw`).

## Items

1. **TotalBlockedMatchesLenBlocked invariant** — `contractcheck.ValidateReadyExplain` checks `TotalReady == len(Ready)` but has no analogous check for TotalBlocked. Add invariant + wire in `RunReadContract/Invariants/ReadyExplain`.

2. **BlockedByEnriched invariant** — `RunReadContract/ReadyExplain` checks only `BlockedBy[i].ID`. Does not assert Title/Priority/Status are populated. A regression where decode returned bare-ID-only objects would pass. Add invariant: for each `BlockedBy` entry, assert Title and Status are non-empty.

3. **--limit doc clarification** — `interface.go:326-329` says limit's effect on blocked list is "unspecified by bd". Real-bd audit confirmed bd 1.0.4 IGNORES --limit in --explain mode for BOTH ready and blocked lists. Update clause to reflect this.

4. **CycleCount non-zero unit test** — pass-through field; no test exercises a payload with `cycle_count > 0`. Add canned fixture asserting `got.CycleCount == 3`.

5. **schema_version tolerance unit test** — decoder ignores the field; no test seeds a payload with `schema_version: 1` to confirm no decode error.

## Acceptance criteria

- [ ] BlockedByEnriched invariant added + wired.
- [ ] interface.go ReadyExplain --limit clause updated.

- [ ] schema_version tolerance unit test added.
