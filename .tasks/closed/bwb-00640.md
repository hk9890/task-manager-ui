## Description
Redesign the detail metadata rail as a compact grouped inspector instead of a loose flat key/value list. The goal is denser scanability and better visual hierarchy while staying within Beads Workbench styling.

## Decisions
- Rail width is fixed at 34 columns
- Divider lines separate metadata groups
- Labels render in their own section, one per line
- Close metadata appears only when available
- Counts remain in the right rail; navigable related issue lists stay out of scope for this task

## Instructions
1. Refactor metadata rendering to use grouped sections with divider lines.
2. Keep core fields near the top: Type, Priority, Status.
3. Add ownership, time, close metadata, labels, and counts sections when populated.
4. Render labels one per line with wrapping or truncation that respects the 34-column rail.
5. Preserve the existing two-column detail layout and single-column fallback.
6. Update tests and goldens to lock the new density and grouping behavior.
