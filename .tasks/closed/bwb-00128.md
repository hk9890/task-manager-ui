Discovered after vkmt+x46s pushed (commit 1e796b0). Windows CI shows ~30+ failing tests, grouped:

**Group A — Golden file comparisons (~25 tests)**: TestModelFixtureShapedBoardCaptureGolden, TestBoardModeAllGroupsPopulatedRendersGolden, TestBoardModeDashboardLayoutGoldensAcrossWidths/{w80,w120,w180}, TestRenderBoardColumnsGolden/*, TestRenderBoardResponsiveWideGolden, TestRenderCommentsGolden, TestRenderCompactGolden, TestRenderWideThreeColumnGolden, TestRenderDependencyRichGolden, TestRenderCompactClosedDurationGolden, TestRenderFallbackKeepsInlineRelatedWorkGolden, TestRenderMinimalGolden, TestRenderFullGolden, TestRenderGoldens/*, TestHarnessSupportsTeaTestGoldenVerification, TestModelBoardDetailBoardRoundTripPreservesLayoutAndFocus, TestBoardModeNavigationEmitsSelectionChangedAndActionRequest.

Root cause hypothesis: Windows git checks out .golden files with CRLF by default; tests compare LF-rendered output against CRLF goldens, causing diff. Fix: .gitattributes pinning .golden + .go to LF, OR normalize line endings in golden comparison helpers.

**Group B — Windows-specific platform tests (2 tests)**:
- TestLoad_DirectoryAtConfigPathReturnsError: MkdirAll fails because AppData\Roaming\bwb\ doesn't pre-exist on the runner. Needs MkdirAll for the parent dir before attempting to create the inverted directory-at-file scenario.
- TestRunCWDExecuteOnlyExitsWithCode1: Windows-specific cwd resolution. Needs inspection.
