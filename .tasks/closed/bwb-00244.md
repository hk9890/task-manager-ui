## Summary

The real `BeadsGateway.Query` rejects empty/whitespace `expr` with `ErrorCodeValidationFailed` before any bd call. `FakeBeadsGateway.Query` in `internal/testing/fakes/beads_gateway.go:348-369` does NOT validate the expression — it returns whatever is in `QueryResponse` or `QueryResponsesByExpr`. Tests using the fake with an empty Query expr would silently succeed instead of failing as the contract requires.

## Discovered during

`beads-workbench-oyb2` Read-lists audit (parent epic: `beads-workbench-yspw`).

## Fix

Add at the top of `FakeBeadsGateway.Query`:
```go
if strings.TrimSpace(expr) == "" {
    return nil, domain.GatewayError{
        Code:    domain.ErrorCodeValidationFailed,
        Message: "query expression must not be empty",
    }
}
```

## Acceptance criteria

- [ ] Fake Query returns ValidationFailed for empty/whitespace expr.
- [ ] Add contract sub-test asserting this behavior on both fake and real.
