## Contract restated (key clauses — see interface.go for full text)

### ListIssues
- Default closed-exclude semantics; `--status closed` or `--status all` to include
- `--status` accepts comma-separated values
- Sort direction translation: `SortDirectionAscending` → `--reverse`; `SortDirectionDescending` → no flag (bd default = DESC)
- `bd list --json` does NOT populate dependencies/dependents/related/description/notes/comments
- Labels may be nil (not empty slice)
- Assignee falls back to owner field when assignee absent

### ReadyIssues
- "Ready" = no unresolved dependency blockers (graph-defined, not stored status)
- Includes status=in_progress when no blockers; excludes closed unconditionally
- Result order is not guaranteed
- **KNOWN LIMITATION (faif.5 audit 2026-05-17):** silently capped at 100 items without `--limit` (bd default); not used by board hot path

### BlockedIssues
- "Blocked" here = dependency-blocked (NOT stored status=blocked)
- `BlockedBy` is []string of bare IDs from bd → gateway constructs `IssueReference{ID:id}` with empty Title/Priority/Status
- Use `ListIssues(Statuses:["blocked"])` to fetch stored-status=blocked instead

### Query
- Non-empty, non-whitespace `expr` required → `ErrorCodeValidationFailed` otherwise (no bd call)
- `IncludeClosed` adds `-a`; redundant but harmless for explicit `status=closed` exprs
- Same sort-direction translation as ListIssues
