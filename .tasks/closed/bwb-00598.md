Read methods are implemented on internal/gateway/beads.CLIGateway while write methods are implemented on internal/gateway/beads.Gateway. The shared interface in internal/gateway/beads/interface.go expects one BeadsGateway with both read and write methods, so Phase 1 consumers cannot construct a single CLI-backed implementation that satisfies the interface.
