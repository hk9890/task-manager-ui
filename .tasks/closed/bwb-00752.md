## Outcome

CachingRepository can hydrate its in-memory state from a per-session JSONL file at startup (using `filestorage.Load` from hhho.4), and saves its in-memory state back to that file periodically and at shutdown.

## File path
Per Q4 (settled at kickoff): `~/.cache/bwb/<project-hash>-<session-id>/repo.jsonl` + adjacent `.manifest.json`. Each bwb instance writes to its own path; no cross-process locking.

## Files (modified or new)
- `internal/repository/caching/caching.go` — add `Hydrate(path string) error` and modify the tick to also do periodic save (configurable interval; default 30s); add explicit `SaveNow() error` for shutdown
- `internal/repository/caching/options.go` — `WithSaveInterval(d time.Duration)`
- `internal/repository/caching/caching_test.go` — hydrate/save round-trip tests

## Hydrate behavior
- `Hydrate("")` → no-op; start with empty memory
- `Hydrate("/path")` and file does not exist → no-op, start empty
- `Hydrate("/path")` and schema mismatch (`filestorage.ErrSchemaMismatch`) → log warning, start empty (NOT an error — degrade to cold start)
- `Hydrate("/path")` and other read error → return the error (caller decides)
- Success → backing memory is the loaded memory; mark Dashboard dirty so first read triggers a backing fetch to verify staleness

## Save behavior
- Periodic save: tick body (kxci.3 already runs a goroutine; share it or add a separate timer) calls `filestorage.Save(memory, path)` if path != ""
- Save errors logged as warnings, do not crash or corrupt state
- `SaveNow()` is synchronous; called by app shutdown handler

## Out of scope for this task
- App wiring (path derivation, Start/Stop integration with cmd/bwb shutdown) → kxci.5

## Test coverage
- Hydrate then read: returns hydrated data without backing call (until invalidation)
- Hydrate missing file: starts empty, no error
- Hydrate schema mismatch: starts empty, logs warning
- Save then Hydrate (round-trip): same data
- Save during concurrent mutation: no corruption (test with -race)
- SaveNow returns error on disk full / permission denied (use a tmp dir + chmod to simulate)

## Acceptance criteria
- [ ] Hydrate handles all 4 cases (empty path, missing file, schema mismatch, success)
- [ ] Periodic save tick works (test with fake clock)
- [ ] SaveNow is synchronous and returns error on failure
- [ ] Round-trip lossless (all hydrated fields preserved after save+reload)
- [ ] -race clean
- [ ] mise run quality green
