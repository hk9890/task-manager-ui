## Description

Gateway test gaps identified in the 2026-05-16 audit that do not fit into a single implementation ticket:

1. **m6 — No test pins single-value JSON decode assumption.** `internal/gateway/beads/runner.go:150-166` rejects trailing JSON, which would break if `bd` ever switches a command to NDJSON. Add a regression-guard test and a doc note.
2. **m7 — `CreateIssue` parses stdout with `TrimSpace` then assumes single ID.** `internal/gateway/beads/writes.go:52`. If `bd create --silent` ever prints diagnostic chatter or trailing lines, the wrong token is lifted. Add a test asserting the issue-id shape (regex), and ideally migrate to `--json` decode.
3. **m11 — `errors_test.go` does not test `errors.Is/As` unwrap chains.** Add unwrap propagation test.

## Files to Modify

- `internal/gateway/beads/runner_test.go`
- `internal/gateway/beads/writes_test.go`
- `internal/domain/errors_test.go`

## Instructions

1. NDJSON guard test: feed `runner.DecodeJSONInto` a body of two concatenated JSON objects; assert the documented rejection. Add a code comment / doc note that NDJSON is unsupported.
2. CreateIssue stdout test: feed a stdout of `"bd-42\n\nwarning: foo\n"`; assert behavior (today: silently lifts `bd-42`). Either accept this with a test, or switch to `--json` decode. Pick the smaller change.
3. errors_test: add a test wrapping a sentinel via the gateway's error wrap function and asserting `errors.Is(wrapped, sentinel)` returns true.

## Acceptance Criteria

- [ ] NDJSON behavior pinned with a test + comment
- [ ] CreateIssue stdout behavior pinned (or migrated to `--json`)
