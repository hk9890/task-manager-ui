## Description
Add a small repository Makefile as a discoverability layer over the documented local workflow without creating a second workflow system.

## Why
This repo currently documents direct Go commands in README, docs, and CI, but has no single convenience entrypoint for common local checks. A thin Makefile should improve discoverability and reduce command memorization while leaving `docs/CODING.md` as the authoritative quality-gate definition.

## Scope
- Add a minimal `Makefile` with core convenience targets only
- Keep targets aligned to already-documented commands
- Add wrapper targets for lint, script validation, and hook installation only if those sibling tasks have already landed
- Update top-level docs so operators know the Makefile is a thin wrapper, not a second source of truth

## Proposed Core Targets
- `help`
- `build`
- `test`
- `vet`

## Optional Wrapper Targets (only if sibling tasks land first)
- `lint`
- `validate-scripts`
- `install-hooks`

## Explicit Non-Goals
- Do not add `quality`; it would duplicate the authoritative sequence in `docs/CODING.md`
- Do not add `runtime-ui`; that workflow includes manual/operator review and should stay documented in `docs/RUNTIME_UI_VERIFICATION.md`

## Files to Modify
- `Makefile`
- `README.md`
- `docs/CODING.md`
- `docs/TESTING.md`
- `docs/RUNTIME_UI_VERIFICATION.md`
- `docs/CHANGE-WORKFLOW.md` (only if the hook-install flow belongs there)
