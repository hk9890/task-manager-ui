## Description

Tracker-driven cleanup of test-coverage gaps identified in a coverage audit. The 69.6% per-package figure understated true coverage (78.2% with `-coverpkg=./...`), but the audit surfaced concrete, actionable items:

- Functions defined but never called (dead code).
- Production methods called only from tests (test-only accessors).
- Real-logic functions with no test (mostly layout math and small state transitions).

## Outcome

- 5 confirmed-dead functions deleted from production code.
- 6 test-only-caller accessors decided (remove or annotate).
- 7 small, fast test files added covering modal navigation, search metadata pane, details scroll math, responsive layout math, and gateway sort mapping.
- Coverage threshold raised from 69 to 75 in `.mise.toml`.

## Success Criteria

- [ ] All child implementation tasks closed
- [ ] `mise run test:coverage` passes with `COVERAGE_THRESHOLD=75`
- [ ] No grep hits for any of the deleted dead functions
- [ ] Acceptance review task closed

## Not in scope

- Cross-package coverage mode (`-coverpkg=./...`) — separate decision.
- Test infrastructure refactors beyond the four unused `ResetCalls` methods.
- `cmd/bwb-smoke` coverage (covered by smoke task, not unit tests).
