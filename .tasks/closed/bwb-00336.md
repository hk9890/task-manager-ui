## Description

`internal/repository/caching/caching.go:282` computes `precomputedDashboard, _ = loaded.Dashboard(context.Background())` — the error is silently discarded. If `memory.Repository.Dashboard` ever errors (corrupted snapshot, future invariant check, etc.), a zero-valued `DashboardData` is stored with `dashboardDirty=false` and served as fresh. The user sees a blank board with no warning.

Currently `memory.Dashboard` only errors on `ctx.Err()` so this is defensive — but the `context.Background()` use is also wrong (no cancellation), and the silent-discard masks any future error path.

## Instructions

1. Capture the Dashboard error: `precomputedDashboard, err := loaded.Dashboard(ctx)` (use Hydrate's incoming ctx, not Background()).
2. On error: do NOT precompute the dashboard cache. Instead leave `c.dashboardDirty=true` and let the first user-facing Dashboard() call fetch fresh from backing. Optionally log via the cacheLogger.
3. Keep the rest of the Hydrate logic.
