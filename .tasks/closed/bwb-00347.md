## Description
Reads through the bd gateway runner execute fully concurrently and uncapped (shared RLock in internal/gateway/beads/runner.go). A board auto-refresh fans out 4 bd subprocesses at once; each cold-opens the embedded Dolt DB and they contend super-linearly -- solo bd measured ~0.28s but contended calls were 1.25s median / 6.1s max across 2578 logged calls.

## Instructions
1. Add a bounded concurrency limiter to CommandRunner -- a buffered-channel semaphore (small constant capacity 2-3; document the choice in a comment).
2. Acquire a slot around the executor.Run call in CommandRunner.Run, released on completion (defer). Applies to all subprocess execs; the existing runMu RWMutex stays as-is.
3. Context cancellation/timeout while waiting for a slot must return promptly without executing -- select on ctx.Done() alongside the semaphore acquire, and return the normalized timeout/cancel error.

## Acceptance Criteria
- [ ] Concurrent bd subprocesses are bounded; a burst of N>cap reads never runs more than cap execs at once
- [ ] A runner test asserts the concurrency cap under a parallel burst
- [ ] Context cancellation while waiting for a slot returns promptly without executing
- [ ] go test ./internal/gateway/beads/... passes; quality:fast gate passes
