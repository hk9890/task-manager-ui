## Description
`internal/launcher/process_runner.go` sets `cmd.Env` to ONLY the launcher's env entries, which **replaces** the parent process's entire environment (PATH, HOME, SHELL, etc.). Any launcher definition with `Env` entries produces a broken child process that can't find executables.

This is a real runtime bug — launching `opencode` or any custom launcher with env vars will fail silently or produce confusing "command not found" errors.

## Instructions
1. Open `internal/launcher/process_runner.go`
2. Find the line where `cmd.Env` is set (around line 19-20)
3. Change it to append launcher env entries to the parent environment:
   ```go
   cmd.Env = append(os.Environ(), envPairs...)
   ```
4. Add `"os"` to imports if not already present
5. Update `internal/launcher/process_runner_test.go` to verify that parent env vars are preserved when launcher env is set

## Files to Modify
- `internal/launcher/process_runner.go` (fix env assignment)
- `internal/launcher/process_runner_test.go` (add test for env inheritance)
