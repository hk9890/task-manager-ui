## Description

Catch upstream CVEs in our Go dependencies without adding per-PR noise. dtctl runs `govulncheck` on every push AND on a daily cron — for bwb (smaller dep surface, fewer contributors) a **weekly cron** is the right cadence. Per-PR runs would mostly waste minutes since most PRs don't touch deps.

## Files to Modify

- New file: `.github/workflows/security.yml`

## Instructions

1. Create `.github/workflows/security.yml`:
   ```yaml
   name: Security

   on:
     workflow_dispatch:
     schedule:
       - cron: '0 6 * * 1'  # Mondays 06:00 UTC

   jobs:
     govulncheck:
       name: Vulnerability scan
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-go@v5
           with:
             go-version-file: go.mod
         - run: go install golang.org/x/vuln/cmd/govulncheck@latest
         - run: govulncheck ./...
   ```
2. Trigger once manually via `workflow_dispatch` (Actions tab → Run workflow) to confirm green on first run.
3. (Optional follow-up — out of scope here) wire failure notifications.

## Acceptance Criteria

- [ ] `.github/workflows/security.yml` exists with weekly cron
- [ ] Manual `workflow_dispatch` run succeeds
- [ ] No vulnerabilities currently reported (or, if any: file a follow-up task, link it from this issue)
