## Description
Extend the issue detail read model and gateway decoding so the metadata rail can display creator, closed timestamp, duration inputs, and close reason when `bd show --json` provides them.

## Decisions
- `owner` should surface as creator metadata, not as the assignee fallback when both fields exist
- Close metadata should be available to the UI when present
- Duration can be computed from created and closed timestamps in the UI if both are available

## Instructions
1. Extend the domain issue detail model with creator and close metadata fields needed by the detail rail.
2. Decode those fields from `bd show --json` without regressing existing assignee behavior.
3. Add tests proving creator and close metadata decode correctly when present and stay empty when absent.
4. Keep list/search summary behavior stable unless additional fields are explicitly needed there.

- internal/domain/issues.go
- internal/gateway/beads/read_gateway.go
- internal/gateway/beads/read_gateway_test.go
