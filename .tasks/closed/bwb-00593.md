## Description
Several test files have alibi coverage — they prove the function exists but don't test meaningful behavior. Additionally, `internal/ui/shared/renderhelpers/helpers.go` has no test file at all, and `internal/domain/issue_edit_document_test.go` is missing edge case coverage.

## Instructions

### 1. Improve toaster tests (`internal/ui/toaster/toaster_test.go`)
Currently 34 lines that don't test:
- Different toast styles (error vs warning vs info vs success) produce visually distinct output
- Toast placement in the overlay
- Toast expiry/dismissal behavior
- Multiple concurrent toasts

Add tests for at least: distinct style output and toast content rendering.

### 2. Improve loading tests (`internal/ui/loading/loading_test.go`)
Currently 27 lines that miss:
- Search scope rendering (search-specific loading state)
- Summary/detail text edge cases
- Empty/nil state handling

Add tests for scope variants and edge cases.

### 3. Create renderhelpers tests (`internal/ui/shared/renderhelpers/helpers_test.go`)
No test file exists. Add tests for:
- `NormalizeToken` (the canonical token normalizer)
- `TruncateWithEllipsis` or similar truncation helpers
- Any other exported functions in the file

### 4. Add editor opener fallback chain test (`internal/launcher/editor/opener_test.go`)
Currently only tests `splitEditorCommand`. The documented fallback chain (EditorCommand → $EDITOR → vi) has zero coverage. Add tests that verify:
- Explicit EditorCommand is used when set
- $EDITOR env var is used when EditorCommand is empty
- `vi` fallback when both are empty

### 5. Add issue edit document edge case tests (`internal/domain/issue_edit_document_test.go`)
Original review found missing edge cases:
- ClearLabels path (all labels removed)
- Priority boundary values (0, 4, and out-of-range values)
- "P2" prefix parsing (user writes "P2" instead of "2")
- Assignee clearing (set to empty)
- Multi-paragraph descriptions
- Partial marker corruption (missing BWB:FIELD markers)
