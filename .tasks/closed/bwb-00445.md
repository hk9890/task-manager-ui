## Description
Add targeted unit tests for two bd-1.0.4 workaround branches in `internal/repository/beads/lean_writes.go`. Neither is exercised by the contract test in `internal/repository/contract/parity_integration_test.go`. Grep for `leanIsCloseNotFound`, `--remove-label`, or "re-close" returns zero matches in real test code.

**Workaround 1 — `leanIsCloseNotFound` (lean_writes.go:107-149):** bd 1.0.4 emits `"issue not found"` when closing an already-closed issue within the same second (DATETIME resolution). The repo verifies via `bd show` that `status=closed` and swallows the error. Tracked upstream as gastownhall/beads#4025.

**Workaround 2 — `UpdateIssue` ClearLabels branch (lean_writes.go:81-95):** bd 1.0.4 silently ignores `--set-labels ""`. The repo fetches current labels via `Issue()` then emits `--remove-label <csv>` to actually clear them.

Both are exactly the conditional/recovery code where regressions hide silently when bd ships 1.0.5.

**Plan-review note (Q10):** the doc-comment at `lean_writes.go:84` says "enumerate each with --remove-label" but the code uses a single `--remove-label` flag with a CSV value. Fix the stale doc-comment as part of this task. Also: the CSV order depends on what `Issue()` returns — see step 4 below.
