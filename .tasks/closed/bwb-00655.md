## Outcome

`internal/gateway/beads/runner.go` (or a sibling file in the same package) exposes a helper that runs `bd vc status --json` (or parses the human-readable output if no `--json` flag exists) and returns the Dolt commit hash. CachingRepository uses this in its periodic invalidation tick (kxci.3).

## Investigation
- Run `bd vc status --json 2>&1` to see if a JSON form exists. If yes, parse JSON. If no, parse the human-readable `Commit: <hash>` line with a regex.
- Confirm the hash format is stable (alphanumeric, short hex-like).

## Files (new or modified)
- `internal/gateway/beads/vc_status.go` — `VCStatusHash(ctx context.Context, runner *CommandRunner) (string, error)` helper
- `internal/gateway/beads/vc_status_test.go` — unit tests with a recording executor

## Contract
```go
// VCStatusHash returns the current Dolt commit hash from `bd vc status`.
// Returns ("", err) on any failure; the caller treats error as "skip this tick".
// Two consecutive calls with no external bd writes return the same hash.
// Calls that succeed but cannot parse a hash return an error.
func VCStatusHash(ctx context.Context, runner *CommandRunner) (string, error)
```

## Acceptance criteria
- [ ] Helper compiles and exports as `beads.VCStatusHash`
- [ ] Unit test: success path returns the hash from a stubbed runner output
- [ ] Unit test: error path returns wrapped error (do not panic)
- [ ] Unit test: malformed output (no hash) returns a parse error
- [ ] `mise run quality` green
