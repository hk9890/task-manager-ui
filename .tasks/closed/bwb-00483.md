What I did: Launched bwb in an empty directory with no .beads/ repo (mktemp -d), captured the startup screen via scripts/capture_bwb_screen.py.

Expected (source: docs/user-guide/key-bindings.md:11 'ctrl+q — quit'; internal/config/keybindings.go:105 ShellActionQuit bound to ctrl+q): ctrl+q quits the app from any screen.

Actual: The fatal-error screen (shown for non-beads directories) hardcodes its quit keys at internal/app/model.go:339 as 'q' || 'ctrl+c'. It ignores ctrl+q entirely and instead accepts bare 'q', which is NOT a quit key anywhere else in the app. The on-screen hint (internal/ui/fatalerror/fatalerror.go:27) honestly reads 'Press q or ctrl+c to quit', so the screen is self-consistent — but it inverts the app's own keymap.

Severity: low. The screen self-documents its keys, so users are not stuck; the issue is a consistency papercut. A user with ctrl+q muscle memory hits a dead key on what may be the first screen they ever see. FEW HICCUPPS: Consistency / Familiarity.

Repro: bwb --cwd $(mktemp -d) ; press ctrl+q -> nothing happens; press q -> quits.

Suggested fix: also accept the configured ShellActionQuit binding (ctrl+q) on the fatal-error screen, and/or update the on-screen hint.
