## Description
The new dashboard fetch contract needs a single call that returns both ready issues and dependency-blocked issues (and their counts), because `bd query`/`bd list` do not expose per-issue dependency state. `bd ready --explain --json` provides exactly this: `{ready: [...], blocked: [...], summary: {total_ready, total_blocked, cycle_count}}`. Required by the dashboard rewrite (epic beads-workbench-au54).

## Instructions
1. In `internal/gateway/beads/interface.go`, extend `BeadsGateway` with:
   - `ReadyExplain(ctx context.Context, opts domain.ReadyExplainOptions) (domain.ReadyExplainResult, error)`
2. In `internal/domain/queries.go`, add:
   - `ReadyExplainOptions{Limit int}` (Limit 0 = uncapped; bd's `--limit`).
   - `ReadyExplainResult{Ready []IssueSummary; Blocked []BlockedIssueView; TotalReady int; TotalBlocked int; CycleCount int}` — reuse existing `BlockedIssueView` (defined in `domain/queries.go` already, see `internal/gateway/beads/read_gateway.go:117-122` for shape).
3. In `internal/gateway/beads/read_payloads.go`, add the explain envelope:
   ```go
   type bdReadyExplainPayload struct {
       Ready   []bdIssuePayload      `json:"ready"`
       Blocked []bdIssuePayload      `json:"blocked"`
       Summary bdReadyExplainSummary `json:"summary"`
   }
   type bdReadyExplainSummary struct {
       TotalReady   int `json:"total_ready"`
       TotalBlocked int `json:"total_blocked"`
       CycleCount   int `json:"cycle_count"`
   }
   ```
   Blocked items carry `blocked_by` (`[]string`) on the `bdIssuePayload` — already wired at `read_payloads.go:26`. Map each blocked item to a `BlockedIssueView{Issue, BlockedBy: refs}` mirroring the current `Gateway.BlockedIssues` implementation (`read_gateway.go:100-125`).
4. In `internal/gateway/beads/read_gateway.go`:
   - Add `operationReadyExplain = "ready explain"` constant.
   - Implement `ReadyExplain` using `RunJSON[bdReadyExplainPayload]` with argv `["ready", "--explain", "--json"]` + `--limit N` if `Limit > 0`.
5. Add a fake gateway counterpart in `internal/testing/fakes/`.
