## Description

`TestForcedRotationProducesRotatedOutput` in `internal/logging` fails intermittently under `mise run test` with `TempDir RemoveAll cleanup: unlinkat ... bwb: directory not empty`. Passes on retry and passes consistently when run directly via `go test ./internal/logging/...`.

Discovered during implementation of beads-workbench-i1th.4 (mise execution layer adoption). Not introduced by that ticket — pre-existing condition only surfaced because mise/gotestsum runs differ in process ordering vs bare `go test`.

## Root cause hypothesis

Race between the test's `t.TempDir` cleanup and a binary built into that directory not fully exiting. The test likely spawns a subprocess that holds a file handle past test return.

## Repro

```bash
# Run mise run test (sometimes fails on first run)
go clean -testcache && mise run test

# Run in isolation (passes reliably)
go test ./internal/logging/... -run TestForcedRotationProducesRotatedOutput -count=10
```

## Impact

`mise run test` non-deterministically fails. CI may report a false negative. No production code is affected.

## Acceptance criteria

- [ ] Test passes deterministically under `go test -count=10 ./internal/logging/`
- [ ] Test passes deterministically under `mise run test` across at least 5 consecutive runs
- [ ] Root cause documented in a comment on the test (if a subprocess race) or refactored to avoid the race entirely
