## Description
Add dedicated detail-mode keyboard navigation for the left rail and allow opening related issues directly from it.

## Instructions
1. Update the shell-owned detail presentation state and app integration so dedicated detail mode tracks pane focus across the left rail, content pane, and metadata rail.
2. Reuse existing detail scroll bindings as focus-conditional behavior: when the content pane is focused, up and down keep scrolling exactly as today; when the left rail is focused, up and down move the selected related issue row. Metadata remains non-interactive in v1.
3. Use left and right to move pane focus between left rail, content, and metadata.
4. When Enter is pressed on a focused left-rail item, emit a mode-level open-related-issue intent from `internal/mode/details/model.go`. Handle that intent in `internal/app/model.go` by loading the selected related issue and keeping dedicated detail mode active.
5. Preserve current content scrolling behavior and scroll restoration when moving between issues.
6. Keep search preview and compact detail rendering non-interactive.
7. Add or update tests for focus movement, related-issue open behavior, and existing detail-mode round-trip expectations.
