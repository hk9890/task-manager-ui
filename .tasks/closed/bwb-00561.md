## Files (new)
- `internal/repository/beads/repository.go` -- `beads.Repository` struct implementing the `Repository` interface
- `internal/repository/beads/repository_test.go` -- unit tests against a stub of the underlying BeadsGateway

## Architecture decision (locked for E1)

The translator **delegates to the existing `internal/gateway/beads/CLIGateway` (17-method `BeadsGateway` interface)** -- it does NOT skip CLIGateway to call `CommandRunner` directly.

Rationale:
- Smaller subtask (~150 LOC translator vs ~500 LOC reimplementation of fan-out + JSON decode)
- All existing bd-quirk handling stays in CLIGateway where it's already tested
- Accepted cost: CLIGateway survives through E2; deleted in [[8pxi.3]] AFTER [[8pxi.1]] migrates app callers to the new Repository interface

## Translation map

| Repository method | Underlying BeadsGateway calls |
|---|---|
| `Dashboard()` | fan out: `ReadyExplain` + `Query("status=in_progress")` + `Query("status=closed", sort=closed, limit=N)` + `CountIssues` + `Query("status=blocked")` -- in parallel via `errgroup.WithContext` |
| `Issue(id)` | `ShowIssue(domain.ShowIssueQuery{IssueID: id})` |
| `Search(q)` | `SearchIssues(q)` |
| `CreateIssue` / `UpdateIssue` / `CloseIssue` / `AddComment` | 1:1 pass-through |
| `HealthCheck` | `HealthCheck` |
| `Catalogs()` | fan out: `StatusCatalog` + `TypeCatalog` + `LabelCatalog` -- in parallel; combine into `Catalogs` struct |

## Concurrency

- `Dashboard()` and `Catalogs()` use `errgroup.WithContext` so:
  - Underlying calls run in parallel (respecting the existing semaphore cap in CommandRunner)
  - First error cancels remaining calls (returns error, NOT partial result -- per hhho.1 contract)
  - ctx cancellation propagates to all in-flight calls

## Acceptance criteria
- [ ] All 9 methods implemented as translator
- [ ] Dashboard and Catalogs fan-out runs in parallel internally (preserves existing perf characteristics)
- [ ] Partial-failure shape matches contract: any single underlying error fails the whole composite call
- [ ] Errors from underlying methods bubble up with original `domain.ErrorCode` (no double-wrap, no code translation)
- [ ] Unit tests pass with a stubbed underlying BeadsGateway (one stub configured per test scenario)
- [ ] No business logic in this layer -- pure translation; any bd-specific quirk handling stays in CLIGateway
- [ ] `mise run quality` green
