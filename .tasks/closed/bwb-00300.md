The 4 fake test files (beads_gateway_test.go, launcher_test.go, editor_test.go, process_runner_test.go) test that the fakes return what you configure them to return. This is tautological: you set a field, call a method, and assert the field came back. These tests verify the implementation of trivial struct access, not meaningful behavior. The compile-time interface assertions (var _ Interface = (*Fake)(nil)) already guarantee the fakes satisfy their contracts — that's the valuable check. The rest is noise that must be maintained when interfaces change.

Consider: if these tests were deleted, what bug could slip through? None — any interface mismatch is caught by the compile-time assertion, and any behavioral regression in the fake would surface in the 70+ production tests that use them. The fakes are too simple to have bugs.

Suggested action: delete the 4 fake test files. The compile-time interface assertions are sufficient. If a fake becomes complex enough to need its own tests (e.g., stateful behavior, conditional logic), add tests at that point.
