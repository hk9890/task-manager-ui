## Description
Redesign the Beads Workbench dashboard and browsing surface so opening the app immediately shows a proper triage dashboard comparable in usefulness to Perles. The board must become the primary high-value surface, use the full terminal effectively, and stop relying on a large selected-issue sidebar in the main screen.

## Goals
- Opening the app shows a proper 4-column triage dashboard immediately
- The dashboard uses the full terminal size effectively
- The main board screen is Perles-like in overview usefulness, density, and scanability
- Any selected summary shown on the main board screen is at most a short 2-3 line summary; full detail lives behind enter/detail mode
- Validation includes golden tests, representative-width layout tests, PTY/runtime checks, and board→detail→board UX verification

## Success Criteria
- [ ] Child tasks complete
- [ ] Existing UI bugs are resolved or folded into task outcomes
- [ ] Acceptance review passes with real PTY/runtime validation
- [ ] The user can open the app and immediately see a proper dashboard without a large detail sidebar consuming the board
