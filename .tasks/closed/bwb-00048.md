Discovered from live user review of detail metadata editing. Expected: once a metadata edit dialog is open, the user should be able to change the value and press Enter again to save it; pressing Escape should close the dialog without saving. Actual: user reports the dialog change flow feels broken and not like standard confirm/cancel behavior. Existing tests confirm status editing opens a modal, but live UX needs verification for keyboard submit/cancel behavior.

## Acceptance Criteria
- [ ] In a live-app metadata edit dialog, Enter submits the selected/edited value and saves it.
- [ ] Escape closes the dialog without saving.
- [ ] Verification covers both save and cancel paths.
- [ ] Enter on an unchanged value follows a defined no-op/close behavior without causing a broken save flow.
- [ ] If current modal behavior differs, the fix aligns metadata edit dialogs with standard confirm/cancel expectations.
