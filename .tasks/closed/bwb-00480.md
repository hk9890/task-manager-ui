What I did: Ran 'bwb --config <file> --check-config' against effectively-empty config files.

Expected: An effectively-empty config file should be treated as 'use all defaults' — consistent with a MISSING config file, which already does exactly that (internal/config/load.go:121 returns nil when requireExists is false -> defaults).

Actual: Every effectively-empty config file fails startup: exit 1, stderr 'error: failed to load config ... error=load config "...": decode yaml: EOF'. Verified by live repro (reviewer-confirmed):
  - 0-byte file -> exit 1, 'decode yaml: EOF'
  - whitespace-only file -> exit 1, 'decode yaml: EOF'
  - comment-only file (e.g. '# comment') -> exit 1, 'decode yaml: EOF'
  - a bare YAML document marker '---' -> exit 0, 'config OK' (the ONLY effectively-empty input that works)

Root cause: internal/config/load.go:141-143 — yaml.Decoder.Decode returns io.EOF for empty/whitespace/comment-only input (no document node is produced), and line 143 wraps it as an error BEFORE the empty-content fallback at line 145 ('if len(node.Content)==0 { return defaults }') can run. The :145 fallback is only reachable for inputs that decode to a node, such as '---'.

CORRECTION (post-review): an earlier version of this description incorrectly claimed whitespace/comment-only configs decode gracefully and only 0-byte files fail. That contrast is empirically false. The real defect is broader: ALL effectively-empty configs fail.

Severity: low-medium. Hard startup failure (exit 1) on realistic input — a 'touch'-ed config file, a truncated file, a comment-only file, or an editor that saved an empty/whitespace buffer. The message 'decode yaml: EOF' is also opaque — it never says 'empty' or names the file as the cause.

Repro:
  : > /tmp/empty.yaml ;           bwb --config /tmp/empty.yaml --check-config   # exit 1, 'decode yaml: EOF'
  printf '# c\n' > /tmp/c.yaml ;  bwb --config /tmp/c.yaml --check-config       # exit 1, 'decode yaml: EOF'

Suggested fix: in decodeOverride (load.go:142), treat io.EOF from Decode as empty input — 'if errors.Is(err, io.EOF) { return overrideModel{}, nil, nil }' — falling through to the same defaults path the len(node.Content)==0 check already provides. One change covers 0-byte, whitespace-only, and comment-only inputs.
