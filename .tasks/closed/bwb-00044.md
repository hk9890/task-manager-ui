## Description
Finish the last metadata-pane polish work for the issue detail screen. The current metadata rail still does not reserve a uniform selection gutter across all line types, so titles, editable fields, read-only fields, divider rows, and label rows do not share one clean left edge. This follow-up must align the metadata pane with the app-wide `› ` selection convention while keeping all rows horizontally stable.

## User Decision
- Keep the global app-wide blue `› ` selection marker.
- Do not invent a metadata-specific selection pattern.
- Reserve a permanent 2-character gutter for **all** metadata lines.
- Selected line uses `› `.
- Non-selected lines use `  `.
- The marker glyph/styling should be shared from one place so global selection-style changes propagate consistently.
