## Description

Mise becomes the single execution layer for build/test/lint/quality. **Create** `.mise.toml` (file does not yet exist in the repo) with both a `[tools]` section (pinning Go) and a `[tasks]` section covering all developer commands. Depends on i1th.2 (build tag exists), i1th.3 (gotestsum version pinned).

## Files to Modify

- `.mise.toml` (NEW — does not currently exist)

## Instructions

1. **Confirm Go version** from `go.mod` (the `go 1.X.Y` directive) and use that exact version in the new `[tools]` block. This makes mise the single source of truth for the toolchain. Once this lands, beads-workbench-i1th.7 will drop `actions/setup-go@v5` from CI.

2. **Create `.mise.toml`** with this structure:

```toml
[tools]
go = "<version-from-go.mod>"  # exact value; e.g. "1.25.6"

[env]
CGO_ENABLED = "0"

[tasks.build]
description = "Build the bwb binary"
run = "go build ./cmd/bwb"

[tasks.fmt]
description = "Format Go source"
run = "go fmt ./..."

[tasks.vet]
description = "Run go vet"
run = "go vet ./..."

[tasks.lint]
description = "Run golangci-lint at pinned version"
run = "go run github.com/golangci/golangci-lint/v2/cmd/golangci-lint@$(cat .golangci-version) run --timeout=5m"

[tasks.guardrails]
description = "Architecture guardrails (fast)"
run = "go test ./cmd/bwb -run TestArchitectureGuardrails"

[tasks.test]
description = "Unit tests (fast, no external deps)"
run = "go run gotest.tools/gotestsum@$(cat .gotestsum-version) --format pkgname-and-test-fails -- ./..."

[tasks."test:integration"]
description = "Integration tests (real bd binary, embedded fixture)"
run = "go run gotest.tools/gotestsum@$(cat .gotestsum-version) --format pkgname-and-test-fails -- -tags=integration ./..."

[tasks."test:all"]
description = "Run unit + integration tests"
depends = ["test", "test:integration"]

[tasks."test:verbose"]
description = "Unit tests with full output (for debugging a hang or noisy failure)"
run = "go run gotest.tools/gotestsum@$(cat .gotestsum-version) --format standard-verbose -- ./..."

[tasks.scripts]
description = "Validate script syntax (shell + python)"
dir = "{{config_root}}"
run = '''
bash -n internal/testing/e2e/embeddedfixture/setup.sh
find scripts -maxdepth 1 -name '*.py' -print0 | xargs -0 python3 -m py_compile
'''

[tasks."quality:fast"]
description = "Fast pre-commit gate (NO integration tests; ~15s)"
depends = ["vet", "lint", "guardrails", "test"]

[tasks.quality]
description = "Full pre-handoff gate (includes integration tests; ~2min)"
depends = ["vet", "lint", "guardrails", "test", "test:integration"]
```

3. **Note on invocation pattern.** All gotestsum and golangci-lint invocations use `$(cat .X-version)` directly — no shell variable indirection — because the version files contain the leading `v` (see beads-workbench-i1th.3 F10 decision).

4. **Verify** each task:
   ```bash
   mise tasks                       # lists every task with its description
   mise run vet                     # fast smoke
   mise run test                    # unit only — confirm <15s
   mise run quality:fast            # vet+lint+guardrails+test, no integration
   mise run quality                 # full gate
   mise run test:integration        # the 9 real-bd tests
   ```

5. **`quality` vs `quality:fast` semantics** (locked by user decision):
   - `quality:fast` is the in-flight / pre-commit gate — fast, omits the ~85s integration cost.
   - `quality` is the pre-handoff gate — includes integration; required before PR landing.
   - This distinction must be reflected in `docs/CHANGE-WORKFLOW.md` updates (owned by i1th.6).
