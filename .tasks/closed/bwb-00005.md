## Description
Before implementing the dashboard redesign, add the testing infrastructure needed to force real UX validation. The redesign tasks must not be able to pass on model-state tests alone; they must prove layout quality at representative terminal sizes and real runtime behavior.

## Instructions
1. Add parameterized dashboard rendering tests for representative terminal widths (at minimum 80, 120, and 180 columns).
2. Add helpers/goldens for full-board captures using realistic repo data and the embedded fixture when practical.
3. Add a board→detail→board round-trip test that verifies layout and focus behavior, not just state transitions.
4. Add simple density/layout assertions where appropriate (for example visible rows per column / max chrome overhead).
5. Document the expected UX verification workflow in tests or docs if needed.

## Files to Modify
- `internal/testing/ui/`
- `internal/app/*_test.go`
- `internal/mode/board/*_test.go`
- relevant golden files under `testdata/`
- `docs/TESTING.md` if test workflow needs documenting

## Acceptance Criteria
- [ ] Dashboard layout is golden-tested at 80, 120, and 180 columns
- [ ] There is at least one realistic full-board rendering capture test
- [ ] There is a board→detail→board round-trip layout test
- [ ] Tests enforce actual render/layout behavior, not only model state
