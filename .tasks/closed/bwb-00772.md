## Outcome

cmd/bwb wires the CachingRepository so that the default startup mode boots from per-session JSONL, serves reads from memory, polls bd for external changes, and saves on shutdown.

## Flag changes
- `--repo beads|memory|caching` (default changes from `beads` to `caching`)
- `--repo-file <path>` ‚Äî for `caching` mode, becomes the per-session cache file (default derived as `~/.cache/bwb/<project-hash>-<session-id>/repo.jsonl`); for `memory` mode, unchanged; for `beads` mode, unchanged (informational)
- Document in `--help` and `docs/CODING.md`

## Default repo-file path (caching mode)
- Reuse `defaultRepoFilePath(projectRoot)` from kxci.4 / 8pxi.4
- Extend it to take a session ID: `defaultRepoFilePath(projectRoot, sessionID string) string` returning `~/.cache/bwb/<hash>-<sessionID>/repo.jsonl`
- Session ID is the same `session_id` already used by `internal/logging` (find the helper in `internal/logging/` and reuse it; do NOT generate independently)

## Lifecycle wiring (caching mode only)
1. Construct the backing `beads.Repository` (as today)
2. Construct `caching.New(backing, ...)`
3. Call `cache.Hydrate(repoFilePath)` (logs warning + continues on error per kxci.4)
4. Call `cache.Start(ctx)` to launch the refresh + save tick
5. Register a signal handler / defer to call `cache.SaveNow()` + `cache.Stop()` on app exit

For `--repo beads` (legacy): construct beads.Repository directly, no caching
For `--repo memory`: unchanged from 8pxi.4

## Files (modified)
- `cmd/bwb/main.go` ‚Äî wire the new flag value + lifecycle
- `cmd/bwb/main_test.go` ‚Äî extend `TestParseCLIRepoFlags` to cover `--repo caching`; add `TestCachingModeStartupWiring` that boots with a tmp dir cache + a stub backing and asserts: (a) initial Dashboard call hits backing; (b) second Dashboard call hits cache (count backing calls); (c) shutdown saves the cache file
- `docs/CODING.md` ‚Äî update `--repo` documentation

## Migration concern
Default flag flip from `beads` ‚Üí `caching` is a user-visible behavior change. Document it clearly in the commit message and a docs note. Add a one-line print at startup ("Using caching repository backend; --repo beads disables") so users see what mode they're in.

## Out of scope for this task
- Per-ID Dashboard projection refinement (the current single-bool "dirty" Dashboard is fine for this epic; per-ID is a future optimization)
- Multi-process cache file consolidation (per Q4: each session has its own file; this is intentional)
