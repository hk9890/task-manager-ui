## Description

Today there is no test stratification — every test runs under `go test ./...` regardless of cost. Result: the default quality gate takes ~95s wall time, dominated by 9 real-bd integration tests that fork the `bd` binary and replay an embedded fixture (~10s each). There is also no consistent execution layer: callers invoke `go test` directly or via the existing Makefile with no shared task vocabulary.

This epic introduces a proper test structure with two categories (unit + integration — no e2e tier since none exist today) and adopts `mise` as the execution layer.

## Why

- **Fast inner loop**: developers should be able to run unit tests in <10s during implementation, opting into the ~85s integration run only when needed.
- **Clear vocabulary**: callers (humans + agents) should not need to memorize 5 different `go test` invocations to run "the right tests".
- **Better output**: default `go test ./...` produces a wall of text; agents and humans both lose signal.
- **Single execution surface**: mise tasks become the one place to look for "how do I run X".

## Reference model

ai-config-manager (`/home/hans/dev/github/ai-config-manager`):
- Build tags: `//go:build integration` (and `e2e` — not needed here)
- Default `go test ./...` runs unit only
- Makefile wraps the tag invocations
- `.mise.toml` for tooling (Go version), task layer to be added in this epic

## Success Criteria

- [ ] All child implementation tasks complete
- [ ] Acceptance review task complete
- [ ] `mise run test` runs unit tests in <15s
- [ ] `mise run test:integration` runs the integration suite (real-bd path)
- [ ] `mise run quality` is the single entry point for the pre-handoff gate
- [ ] CI uses `mise run` commands (single source of truth)
- [ ] Makefile removed; all docs reference mise commands
- [ ] No regressions in test coverage (every test that was running before still runs in one of the categories)

## Scope decisions (locked during planning)

- Migration is the **smallest move**: add `//go:build integration` ONLY to the 9 real-bd tests. Don't reorganize directories. Don't tag unit tests explicitly.
- Mise **replaces** the Makefile entirely (no backwards-compat shim).
- gotestsum is the chosen output formatter, invoked via `go run gotest.tools/gotestsum@latest` (no global install).
- No e2e category — only unit + integration today.
