## Description

Use the inventory from [[ppja.2]] and the consolidated `RecordingExecutor` from [[ppja.1]] to ensure every distinct bd argv shape bwb emits is pinned by an explicit test. Pattern matches `TestBoardInitRealGatewaySubprocessArgvCardinality`. This is the outgoing-contract analog of `RunReadContract` — both must hold.

## Instructions

1. Read [[ppja.2]]'s inventory and identify the gaps (argv shapes without existing pinning tests).
2. For each gap, add a test following the pattern of `TestBoardInitRealGatewaySubprocessArgvCardinality` (`internal/mode/board/model_test.go:988`):
   - Wire the consumer (mode model, app model, gateway method, etc.) against a `RecordingExecutor` with the expected argv pre-registered.
   - Drive the consumer to the point where the argv is emitted.
   - Assert the recorded calls match the expected argv shape exactly.
3. Group new tests by consumer in the appropriate `_test.go` file (e.g., search argv tests in `internal/mode/search/model_test.go`, detail argv tests in `internal/mode/details/...`, write argv tests in `internal/gateway/beads/writes_test.go`).
4. If during testing you discover an argv that's actually different from what [[ppja.2]] documented, fix the inventory (single source of truth).
5. **Dynamic flags get full boundary coverage (epic-review 2026-05-16 tightening):** where an argv contains flags that depend on runtime state (e.g., `--limit` based on terminal height, `--status` set based on user toggle, page size driven by window width), pin EVERY dynamic flag at:
   - Default value (the common case)
   - Maximum value (upper boundary — e.g. very tall terminal)
   - Minimum value (lower boundary — e.g. 1-line terminal)
   - One value just past the boundary (e.g. height = cap + 1)
   These are exactly where argv regressions silently hide. Punting on dynamic flags ("parametrize OR pin common cases") defeats the purpose of the epic.
6. After adding all tests, run `mise run quality` end-to-end.

## Acceptance Criteria

- [ ] Every argv shape in the [[ppja.2]] inventory has at least one pinning test
- [ ] Every dynamic flag (runtime-state-dependent) pinned at default + max + min + 1 boundary value
- [ ] All new tests use the consolidated `RecordingExecutor`
- [ ] `mise run quality` passes
- [ ] Argv inventory doc updated for any discrepancies found during testing
- [ ] **Deliberate-break**: pick one new test (preferably one covering a dynamic flag), change the consumer to emit a different argv; observe the test fail with a clear message; restore. Document in your closure comment.
