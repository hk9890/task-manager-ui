## Description

`internal/repository/beads/lean.go:132` shows `r.parentSiblingCache[parentID] = out` populated under Lock — but no write path or external invalidation ever clears the cache. Per-instance, once a parent's children are cached, that snapshot lives for the lifetime of the `*beads.Repository`. CachingRepository's hash-change `Reset` only clears its own `memory.Repository`; it cannot reach into the backing `*beads.Repository`'s `parentSiblingCache`.

User impact: external developer adds child c3 to parent P. bwb detects hash change and Resets its caching layer. User opens c1 again — backing reads stale `parentSiblingCache[P] = [c1, c2]` — ParentGroupBrowser never shows c3 until bwb restarts.

## Instructions

1. Add an `Invalidate()` method on `*beads.Repository` (or a more specific `InvalidateParentSiblingCache()`) that clears `parentSiblingCache` under its lock.
2. Wire `CachingRepository.RefreshIfChanged` to call this invalidation in the same place it calls `c.memory.Reset()`. The CachingRepository holds the backing as a `repository.Repository` interface — extend the interface narrowly with an optional `Resetter` capability OR type-assert in CachingRepository for backing implementations that support invalidation.
   - Type-assert is the simpler / less-intrusive choice: `if r, ok := c.backing.(interface{ Invalidate() }); ok { r.Invalidate() }`.

## Test

Two tests:

A. `TestBeadsRepository_Invalidate_ClearsParentSiblingCache` in `internal/repository/beads/`:
   - Skip if no bd binary (integration tier) OR use a `CommandRunner` mock to construct a beads.Repository with deterministic output.
   - Seed parentSiblingCache by calling Issue on a child.
   - Call Invalidate.
   - Assert next Issue call re-runs the bd subprocess (visible via the mock runner's call log).

B. `TestCachingRepository_RefreshIfChanged_InvalidatesBacking` in caching:
   - Build CachingRepository over a fake backing that implements Invalidate (tracked via a counter).
   - Trigger RefreshIfChanged with a hash change.
   - Assert backing.Invalidate counter incremented.
