## Description

Add `//go:build integration` to the 9 tests that fork the real `bd` binary against an embedded fixture. These are the slow tests (~10s each, ~85s total) that dominate the suite. After this change, default `go test ./...` should drop from ~95s to ~10s; the integration tests run only via `go test -tags=integration ./...`.

## Files to Modify

The 9 tests are in **5 files**:
- `internal/app/model_test.go` ŌĆö 7 tests prefixed `TestModelEmbeddedFixture*`
- `internal/mode/search/model_test.go` ŌĆö `TestSearchModeEmbeddedFixtureInitUsesEmptyQueryFallback`
- `internal/testing/e2e/embeddedfixture/setup_test.go` ŌĆö `TestSeedSkipsWhenToolsUnavailable` (file contains ONLY this test + a local `hasExecutable` helper; full-file rename suffices, no split needed)
- `internal/launcher/process_runner_contract_test.go` ŌĆö `TestExecProcessRunnerReapsChildren` and `TestExecProcessRunnerChildSurvivesParentCtxCancel` (these spawn real subprocesses; treat as integration)
- `internal/gateway/beads/runner_test.go` ŌĆö `TestMissingBDDatabaseDetectionSubstringPin` (runs real bd in empty dir)

## Instructions

1. **Split files where mixed.** Most affected files contain BOTH unit tests AND integration tests. Move the integration tests into a sibling `*_integration_test.go` file in the same package with `//go:build integration` at the top:
   - `internal/app/model_integration_test.go` ŌåÉ 7 EmbeddedFixture tests + their helpers (see step 2)
   - `internal/mode/search/model_integration_test.go` ŌåÉ 1 test
   - `internal/testing/e2e/embeddedfixture/setup_integration_test.go` ŌåÉ rename the existing `setup_test.go` (file already contains only the integration test + local `hasExecutable` helper)
   - `internal/launcher/process_runner_integration_test.go` ŌåÉ 2 contract tests
   - `internal/gateway/beads/runner_integration_test.go` ŌåÉ 1 test

2. **Helper movement (pre-resolved, do not re-litigate):**
   - `internal/app/model_test.go` lines 3183/3188 define `hasExecutable` and `runBDInRepo`. Both are used ONLY by the 7 integration tests being moved. **Move both helpers into `model_integration_test.go`.** Do not leave them in the unit file (would be unused ŌåÆ lint fail).
   - The duplicate `hasExecutable` in `internal/testing/e2e/embeddedfixture/setup_test.go` is local to that file; the file rename carries it.

3. **Verify launcher helper safety before moving the contract tests.** Run:
   ```bash
   grep -n "TestExecProcessRunnerReapsChildren\|TestExecProcessRunnerChildSurvivesParentCtxCancel\|hasExecutable" internal/launcher/process_runner_test.go internal/launcher/service_security_test.go internal/launcher/service_test.go
   ```
   If any sibling test file references a helper or constant defined in `process_runner_contract_test.go`, the move would break `-tags=integration` build. If a reference exists, surface it before moving and either duplicate the helper or move it into a tag-free helpers file.

4. **Build tag header**: each new file starts with:
   ```go
   //go:build integration

   package <pkg>
   ```

5. **Verify** both invocations:
   ```bash
   go test ./...                  # must NOT run any of the 9 tests; suite drops to ~10s
   go test -tags=integration ./... # must run the 9 tests; total time ~85-100s
   ```

6. **Coverage parity check.** Use the baseline captured in beads-workbench-i1th.8 (F29 ticket) ŌĆö total test count across both invocations must equal the pre-split count.
