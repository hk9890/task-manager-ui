## Description

PLACEHOLDER EPIC — captures scope so we don't lose context. Not actively planned yet.

`internal/testing/e2e/embeddedfixture/seed.json` has 3 issues. Every parity test, contract test, and integration test that uses the fixture inherits this scale ceiling. Real DBs have hundreds of issues. The bugs fixed in epic [[vr1d]] (ssom, kh54, o7tk) all needed real-data shapes to manifest — fixture-based tests passed while production failed.

## Probable scope (when activated)

Replace the seed with a designed scale fixture (~500-700 issues) that explicitly exercises:
- ≥ 75 closed issues (engages Done column 50-cap; produces "+" badge)
- ≥ 510 issues in one active group (engages 500 cardinality threshold)
- A block of 10+ closed issues sharing one `updated_at` (kh54 trigger)
- A block of P1 issues sharing one timestamp (sort tie-break)
- One 5-deep parent chain, one 3-wide siblings group
- Edge data: one emoji title, one shell-metacharacter title, one max-length title
- A shared keyword across 20+ issues for search relevance / order testing

Build with a recipe-driven generator (`internal/testing/e2e/embeddedfixture/scale/generator.go`) using a deterministic seed for reproducibility. Pair with a machine-readable invariant manifest (`fixture-invariants.yaml`) so the fixture's edge-case coverage doesn't silently erode as it's edited.

Keep the existing 3-issue seed as the "minimal anchor" fixture for tests that need exact ID assertions; new tests default to the scale fixture.

## Activation Trigger

Activate after [[8qw9]] (contract hardening) — the expanded contract test suite will benefit massively from running against the scale fixture. Doing both at once amplifies the value of each.

## Related

Complements [[8qw9]] (contract hardening). The contract gives us machine-checkable invariants; the realistic fixture gives those invariants real edge cases to exercise.
