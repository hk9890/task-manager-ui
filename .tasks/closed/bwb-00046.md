Discovered from ongoing live user review after group-boundary traversal was fixed. Expected: moving up/down in Dependencies should visit every visible issue row exactly once in render order. Actual: user still reports that sometimes selection appears to jump over an issue. Reviewer analysis indicates this is a confirmed real bug: the renderer can show the same issue ID in multiple visible groups, while the selectable dependency list de-duplicates by ID, making one visible row effectively unreachable/skipped.

## Root Cause
Selection is keyed by flattened unique issue IDs, but rendering still shows duplicate visible rows when the same issue appears in multiple groups. That makes some visible rows impossible to land on during one-row-at-a-time traversal.

## Acceptance Criteria
- [ ] One-row-at-a-time traversal can land on every visible dependency issue row in render order.
- [ ] If the same issue appears in multiple visible groups, either each visible row is separately selectable or duplicate visible rows are suppressed/de-emphasized so traversal matches what the user sees.
- [ ] Real-app verification confirms no visible issue row is skipped during stepwise traversal.
