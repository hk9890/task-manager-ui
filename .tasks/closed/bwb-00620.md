## Description

Enforce a coverage floor in CI so we don't silently regress. Initial threshold comes from `beads-workbench-ug82.4` (baseline measurement), set ~5pts below current. Plan is to ratchet up over time, like dtctl does (50 → 55 → 60 → 70).

**Scope:** gates on **unit+integration combined coverage** (per ug82.4 scope decision) — `-tags=integration` is included in the test invocation. This matches reality (integration tests cover the gateway↔bd boundary, a meaningful chunk).

**Blocked by `beads-workbench-ug82.4`** — do not set a numeric threshold until the baseline is measured. Issue is `blocked` + labeled `has:open-questions` until `.4` resolves the threshold value.

## Files to Modify

- `.mise.toml` — add `test:coverage` task (mirrors dtctl's `make test-coverage`, but includes integration)
- `.github/workflows/ci.yml` — add a coverage step after "Integration tests", uses `COVERAGE_THRESHOLD` env var

## Instructions

1. Read the baseline + recommended threshold from the resolved `.4` task.
2. Add to `.mise.toml`:
   ```toml
   [tasks."test:coverage"]
   description = "Run unit+integration tests and enforce coverage threshold"
   run = """
   go test -race -tags=integration -coverprofile=coverage.out -covermode=atomic ./...
   COVERAGE=$(go tool cover -func=coverage.out | grep total | awk '{print $3}' | sed 's/%//')
   COVERAGE_INT=${COVERAGE%.*}
   echo "Total coverage: ${COVERAGE}% (threshold: ${COVERAGE_THRESHOLD:-XX}%)"
   if [ "$COVERAGE_INT" -lt "${COVERAGE_THRESHOLD:-XX}" ]; then
     echo "FAIL: below threshold"; exit 1
   fi
   """
   ```
   (Replace `XX` with the value from `.4`.)
3. Edit `.github/workflows/ci.yml`:
   - Add `env: COVERAGE_THRESHOLD: XX` at the top of the workflow
   - Add a "Coverage gate" step after "Integration tests" that runs `mise run test:coverage`. (Alternatively, replace the existing unit+integration test steps with `test:coverage` since it supersedes them — your call based on signal clarity in the CI run summary.)
4. Run locally: `COVERAGE_THRESHOLD=XX mise run test:coverage` — must pass.
5. Push to a branch, watch the CI run.

## Acceptance Criteria

- [ ] `mise run test:coverage` enforces the threshold locally on unit+integration combined
- [ ] CI fails when combined coverage drops below threshold (verify by temporarily deleting a test, or by setting threshold artificially high in a draft PR — then restore)
- [ ] CI passes with coverage at or above threshold
- [ ] `COVERAGE_THRESHOLD` env var documented inline (comment) for future ratcheting
