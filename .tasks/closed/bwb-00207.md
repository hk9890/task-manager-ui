## Summary
During verification of beads-workbench-dml, beads-workbench-6j6, and beads-workbench-n5o, a real PTY run of the built bwb binary against the seeded embedded fixture did not load board data cleanly.

## Reproduction
1. go build -o /tmp/bwb-verify ./cmd/bwb
2. Seed fixture repo with internal/testing/e2e/embeddedfixture/setup.sh
3. Run the built binary in a PTY from the fixture repo with BD_NON_INTERACTIVE=1

## Observed
The UI renders board sections with error text such as:
- Error: blocked issues: command exited with code ...
- Error: ready issues: command exited with code 1: ...
- Error: list issues: command exited with code 1: ...
This prevents clean startup/full-app verification of user-facing workflows.

## Expected
The built app should load board data successfully in a real terminal session against the seeded fixture, matching the manual bd ready/bd blocked/bd list behavior and test expectations.

## Evidence
A PTY capture during verification showed board loading errors while the same fixture repo returned valid data for manual bd ready --json, bd blocked --json, and bd list --json commands.
