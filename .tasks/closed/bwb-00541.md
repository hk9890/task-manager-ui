## Description

Two test-side `exec.Command` swaps exist with overlapping purpose: `routingExecutor` (private, in `internal/gateway/beads/read_gateway_test.go`) and `RecordingExecutor` (public, in `internal/testing/fakes/recording_executor.go`). RecordingExecutor is more capable; consolidate.

## Instructions

1. Read `internal/testing/fakes/recording_executor.go` to confirm it supports route-by-argv (canned response per argv shape). If not, extend it — but I believe it already does (`rec.OnArgs(argv).Return(...)`).
2. Read `internal/gateway/beads/read_gateway_test.go` and identify every callsite of `routingExecutor` (likely the `newTestGateway` helper + the test cases via `routes` maps). Count them so the migration scope is clear up front.
3. Migrate one-by-one:
   - Replace `routingExecutor` construction with `RecordingExecutor`.
   - Replace `routes map[string]routeResponse{...}` with `rec.OnArgs(...).Return(...)` calls.
   - Keep test semantics identical — assertions should not change.
4. After all callers migrated, delete `routingExecutor` and its helper `argsKey` (if unused elsewhere).
5. Run `mise run quality` — must pass with the consolidated executor.
6. Bonus: extend `RecordingExecutor` doc to note it now subsumes the routing-only role.

## Acceptance Criteria
