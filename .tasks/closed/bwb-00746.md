Add a new section to `docs/RUNTIME_UI_VERIFICATION.md`. The verification target is the **/home/hans/dev/github/dtctl-test** repository (89 closed issues at the time of writing — enough depth to validate multiple load-more pages at small/medium terminal heights).

## Procedure

1. Build bwb: `mise run build` (or `go build -o /tmp/bwb ./cmd/bwb`).
2. Open a terminal at height ≤ 30 rows. Verify with `echo $LINES`.
3. Run against the dtctl-test repo:
   ```
   (cd /home/hans/dev/github/dtctl-test && BD_NON_INTERACTIVE=1 /tmp/bwb)
   ```
4. Observe the initial Done header — should read `N of 89` where N is roughly `height - 3` (e.g. `25 of 89` at height=28).
5. Press `RIGHT` three times to focus the Done column.
6. Press `j` repeatedly past the loaded slice end. Confirm:
   - Done header grows in N (e.g. `25 of 89 → 75 of 89`), M stays 89.
   - `›` chevron stays visible inside the column (this part depends on b38b.4).
   - No double-loads: enable `--debug` and confirm at most one `loadMoreClosed` event per threshold crossing in the `[bwb-debug]` trace.
7. Press `r` for manual reload. Confirm:
   - Header resets to the initial viewport-sized N.
   - Selection returns to the top of Done (per the vtvb.7 contract).
8. Reach the very last closed issue (`j` until N == 89). Confirm Done header reads `89 of 89` and `TotalIsExact` is honored (header format flips from "N of M" to plain "N" — see internal/ui/board/board.go header logic).

## Pass/Fail table

| Step | Pass | Fail signal |
|------|------|-------------|
| 4 | Initial header `≈25 of 89` | Header missing "of 89"; or N == 89 immediately (no pagination active) |
| 6 | N grows monotonically; `›` always visible; one load per crossing | N stays at initial value; chevron disappears; multiple loads per crossing |
| 7 | Header back to initial N; selection at top | Header keeps the deep value; selection lost |
| 8 | Final header `89 of 89` then flips to `89` | Final N < 89; or N never reaches 89 |

## Diagnostics on failure

- If `r` doesn't reset: check `internal/mode/board/model.go` for the doneLoadedCount reset path (vtvb.7).
- If N doesn't grow: check `loadMoreClosedCmd` dispatch threshold (vtvb.6) and the offset wiring in the chosen repo backend (vtvb.2/.3/.4).
- If double-loads: check the in-flight guard (vtvb.8).
- If chevron disappears: b38b.4 selection-following scroll has regressed.
