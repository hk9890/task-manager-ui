## Description
Create the shared automated testing harness that lets agents verify the Bubble Tea UI and full product workflows without relying on manual terminal testing. This task defines the TUI testing strategy, shared test doubles, embedded-mode fixture repos, and testing documentation used by all later UI and product verification tasks.

## Instructions
1. Establish `teatest` plus golden-output verification as the default Bubble Tea UI testing approach for this repo, combined with model/message-driven state-machine tests where behavior needs direct verification. If a surface needs a different technique, document the exception.
2. Create a shared `FakeBeadsGateway` or equivalent deterministic gateway test double for UI-level tests, including configurable per-method error injection for error-path testing.
3. Create deterministic fake seams for external editor launching and launcher process execution so tests never need to spawn real interactive tools.
4. Create a reproducible embedded-mode beads fixture repository with known issues, comments, labels, and dependencies for integration/e2e testing.
5. Define the test vocabulary for this repo: unit, integration, and end-to-end/smoke tests.
6. Create `docs/TESTING.md` and document:
   - test commands
   - test categories
   - Bubble Tea UI testing strategy
   - fake gateway/editor/launcher seams
   - embedded-mode fixture usage
7. Make it explicit that product verification must not require manual human UI inspection.

## Files to Modify
- shared UI test harness packages under `internal/` (create)
- fake gateway/editor/launcher test helpers (create)
- test fixture repo/data for embedded-mode beads testing (create)
- `docs/TESTING.md` (create)
- `docs/CODING.md` (update test vocabulary references if needed)
