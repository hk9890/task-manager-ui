internal/ui/overlay/overlay.go has 99 lines of ANSI-aware overlay logic (Place, calculatePosition) with no tests at all. This is the foundation for both toaster and modal overlay rendering. The position calculation has boundary clamping, padding math, and ANSI-width-aware string slicing that are all untested. A bug here silently corrupts both toaster and modal UI. Suggested action: add unit tests for Place() covering at minimum: center placement, bottom placement, foreground wider than background, negative offset clamping, and ANSI-styled foreground strings.
