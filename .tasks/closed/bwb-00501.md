## Contract restated (key clauses — see interface.go for full text)

**Preconditions:**
- ctx must be non-nil.
- `query.IssueID` must be non-empty, non-whitespace → otherwise `ErrorCodeValidationFailed` (no bd call).

**Postconditions:**
- `result.Summary` has non-empty ID/Title/Status/Type.
- `result.Description` may be "" when bd omits the description key.
- `result.ClosedAt` is zero time.Time when not closed.
- `result.CloseReason` is "" when not closed with a reason; "Closed" (literal) when closed with no `--reason`.
- `result.BlockedBy` is from `Dependencies` with `dependency_type=blocks` (or absent/other-non-related-non-parent-child).
- `result.Blocks` is from `Dependents` with `dependency_type=blocks`.
- `result.Related` is merged from related/relates-to in Dependencies AND Dependents AND top-level Related array.
- `result.ParentGroupBrowser` populated when parent-child dependency present (gateway issues second `bd show` for parent; cached per gateway instance across calls).
- `result.Comments` populated when issue has comments.

**Error semantics:**
- `ErrorCodeValidationFailed` on empty IssueID (no bd call).
- `ErrorCodeCommandFailed` for unknown IDs (bd exits 1; no `ErrorCodeNotFound` path in real gateway).
- `ErrorCodeDecodeFailed` if unparseable or required fields absent (id/title/status/issue_type/priority/created_at/updated_at).

**bd quirks observed:**
- 2026-05-16: `bd show` omits "description" key entirely when unset (not null, not ""). Fixed by 781a — decoder uses `optionalString(primary.Description)`.
- 2026-05-16: `bd show` exit 1 for unknown IDs (not a 404 equivalent).
- 2026-05-16: `bd show --json` wraps single issue in a JSON ARRAY (not bare object). Gateway reads `items[0]`.
