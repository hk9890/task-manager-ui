## Outcome

Sub-second first paint on warm-cache startup, satisfying the kxci.6 AR criterion. Currently `Hydrate` always sets `dashboardDirty=true` for safety, which forces a ~3s backing fan-out on first paint. With persisted+current hash comparison, an unchanged repo skips the fan-out entirely.

## Files (modify)
- `internal/repository/filestorage/filestorage.go` — `Save` accepts an optional `bdCommitHash string` (variadic option or second function); `Load` exposes the persisted hash to callers (either via a new return value, or via a struct return). Pick the least-invasive shape:
  - Option A (recommended): add `SaveWithHash(r *memory.Repository, path string, bdCommitHash string) error` and `LoadWithManifest(path string) (*memory.Repository, Manifest, error)`. Keep existing `Save(r, path)` and `Load(path)` as thin wrappers (Save → empty hash; Load → discard manifest). Tests for both signatures.
  - Option B: change `Save` signature with a variadic option; might break existing callers. AVOID unless A is awkward.
- `internal/repository/caching/caching.go`:
  - `SaveNow()` reads `vcStatusFunc(ctx)` for current hash, falls back to "" on error (still save); calls `filestorage.SaveWithHash` (or equivalent)
  - `Hydrate(path)` reads manifest + current hash; compares:
    - persisted hash == current hash AND non-empty → keep `dashboardDirty=false` (TRUST hydrated state)
    - persisted hash != current hash OR persisted empty (legacy file) OR vcStatusFunc unavailable → set `dashboardDirty=true` (current safe-default behavior)
- `internal/repository/caching/caching_test.go` — new tests

## Acceptance criteria
- [ ] `SaveWithHash` (or chosen API) writes the hash to manifest; round-trip via `LoadWithManifest` returns same hash
- [ ] `Hydrate` with persisted hash matching current `vcStatusFunc` result → `dashboardDirty` is false; first `Dashboard()` call hits memory only (zero backing.Dashboard calls)
- [ ] `Hydrate` with mismatched hash → `dashboardDirty` is true (current behavior preserved)
- [ ] `Hydrate` with empty persisted hash (e.g. file written by code before kxci.5a) → `dashboardDirty` is true (safe default)
- [ ] `Hydrate` when vcStatusFunc returns error → `dashboardDirty` is true (safe default)
- [ ] Existing `Save`/`Load` signatures preserved (no compile break)
- [ ] `mise run quality` green
- [ ] -race clean
