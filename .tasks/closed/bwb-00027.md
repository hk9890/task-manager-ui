## Summary

Clearing labels via the UI is **silently broken** in production. The ClearLabels workaround at `internal/gateway/beads/writes.go:122` emits `bd update <id> --remove-labels <csv>` (plural), but bd 1.0.4 does **not** have a `--remove-labels` flag. The correct flag is `--remove-label` (singular, takes a comma-separated value).

## Discovered during

`beads-workbench-ibif` Write gateway audit (parent epic: `beads-workbench-yspw`).

## Reproduction

In any bd 1.0.4 workspace:

```bash
TMPDIR=$(mktemp -d) && cd "$TMPDIR" && BD_NON_INTERACTIVE=1 bd init
ID=$(bd create --json --title "x" --labels "a,b,c" | python3 -c 'import json,sys;print(json.load(sys.stdin)["id"])')
bd update "$ID" --remove-labels "a,b,c"   # FAILS: "Error: unknown flag: --remove-labels"
bd update "$ID" --remove-label "a,b,c"    # WORKS: exits 0, labels cleared
```

## Production impact

Any edit-modal action that clears all labels on an issue:
- Sends `UpdateIssueInput{ClearLabels: true}` to the gateway
- Gateway calls ShowIssue to fetch current labels (correct)
- Gateway emits `bd update <id> --remove-labels <csv>` (BROKEN)
- bd 1.0.4 returns exit 1 with `Error: unknown flag: --remove-labels` on stderr
- Gateway surfaces this as `ErrorCodeCommandFailed`
- Labels are NOT cleared
- User sees a generic command-failed error in the UI

## Why this slipped through

The only unit test, `TestGatewayUpdateIssueClearLabelsEmitsRemoveLabels` (`writes_test.go:162,181`), uses the stub `RecordingExecutor` and asserts the **wrong** flag (`--remove-labels`). The stub doesn't know which flags bd accepts â€” it only records argv â€” so the test passes against a fiction. There is no integration test that exercises ClearLabels against real bd, so the bug shipped undetected.

## Fix

1. **`internal/gateway/beads/writes.go:122`** â€” change `"--remove-labels"` â†’ `"--remove-label"`.
2. **`internal/gateway/beads/writes_test.go:162,181`** â€” update test assertion to the corrected flag.
3. **`internal/gateway/beads/interface.go:82-89` and `:558-562`** â€” update the documented quirk text to reference the correct flag name.
4. **Add integration test** â€” see follow-up bug (filed separately) for adding a `RunWriteContract/UpdateIssue/ClearLabels` sub-test that exercises against real bd, so the stub-only test cannot mask this class of bug again.
