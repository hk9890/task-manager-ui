## Description
Create the new repository foundation for Beads Workbench and implement the first production-ready backend seam that talks only to official beads surfaces. This epic establishes the standalone codebase, the source-specific Beads Gateway boundary, a CLI-backed gateway implementation, and the test coverage needed before UI work begins.

## Goals
- Bootstrap the standalone Go module and binary entrypoint for `bwb`
- Establish the source-specific `BeadsGateway` seam for all issue reads and writes
- Implement a shared `bd` command runner and JSON decoding layer
- Implement core read and write gateway operations through official `bd` commands only
- Add focused automated tests for command construction, decoding, and failure handling

## Success Criteria
- [ ] The repository has the baseline module, binary entrypoint, and copied design docs
- [ ] A source-specific `BeadsGateway` interface exists in the new repo
- [ ] A CLI-backed gateway implementation performs core reads and writes through `bd`
- [ ] Gateway errors are normalized into UI-usable categories
- [ ] Tests cover command construction, output decoding, and failure behavior
- [ ] No direct SQL, BQL, or orchestration dependency exists in the active foundation path
- [ ] All child implementation tasks are closed
- [ ] Acceptance review task is closed
