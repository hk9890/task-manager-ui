## Outcome

bwb's app + test layers run on `repository.Repository` instead of `beads.BeadsGateway`. This subtask SUBSUMES both the original app-layer migration and the test-file migration -- they cannot ship as separate PRs (the moment `services.Gateway beads.BeadsGateway` flips to `services.Repo repository.Repository`, every test constructing `FakeBeadsGateway` stops compiling). Single atomic PR; tranching below is an intra-PR checklist.

## Files (app layer)
- `internal/app/services.go` -- swap `Gateway beads.BeadsGateway` for `Repo repository.Repository`
- `internal/app/model.go` -- every `services.Gateway.XXX` -> new repository method (`ShowIssue`->`Issue`, `StatusCatalog`+`TypeCatalog`+`LabelCatalog`->`Catalogs`, etc.)
- `internal/mode/board/model.go` -- replace 5-call fan-out with single `repo.Dashboard()` call; `dashboard.Compose` stays in board (raw materials in, columns out)
- `internal/mode/details/...` -- `ShowIssue` -> `Issue`
- `internal/mode/search/model.go` -- `SearchIssues` -> `Search`
- `internal/launcher/editor/service.go` -- `ShowIssue` -> `Issue`, `UpdateIssue` stays
- `cmd/bwb/main.go` -- construct `repository.Repository` (default `beads.New(...)`)

## Migration pattern (per file)
- Replace `fakes.NewFakeBeadsGateway()` with `memory.New()`
- Replace `fake.SeedIssue(detail)` with `memory.Seed(...)`
- Tests that used `fake.SetError(method, err)` wrap in `repository.NewErrorInjecting(memory.New(), ...)` -- new decorator delivered in `hhho.1`
- Tests that used `fake.Calls()` for call-count assertions: rewrite to assert on observable repository state where possible; for those that genuinely need call tracking, the `ErrorInjecting` decorator records calls too

## Mapping table
| Old gateway call | New repo call |
|---|---|
| `gateway.ReadyExplain + Query(in_progress/closed/blocked) + CountIssues` | `repo.Dashboard()` |
| `gateway.ShowIssue(id)` | `repo.Issue(id)` |
| `gateway.SearchIssues(q)` | `repo.Search(q)` |
| `gateway.CreateIssue/UpdateIssue/CloseIssue/AddComment` | 1:1 |
| `gateway.StatusCatalog + TypeCatalog + LabelCatalog` | `repo.Catalogs()` (fields used separately) |
| `gateway.HealthCheck` | `repo.HealthCheck` |
