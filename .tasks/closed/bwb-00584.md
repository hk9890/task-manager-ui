## Description
Replace the nested left related-issues widget model with a true left browser panel whose selection updates the adjacent detail workspace.

## Instructions
1. Implement the left side as a real browser panel in issue details view, not as a focusable sub-widget embedded inside detail content.
2. Build the browser panel on the normalized parent-group context from the backend task; do not use generic related/blocking relationships as the primary panel source in v1.
3. Show a stable group consisting of the parent issue plus sibling child issues.
4. Selecting a browser-panel item updates the center/right issue inspection workspace.
5. Browsing between items in the same parent group must not rebuild the left-panel contents on every selection.
6. If there is no parent-group context, omit or hide the special left browser panel for now.
7. Keep the center and right panes conceptually paired: center for issue text/content, right for metadata and lightweight actions.
8. Add or update rendering and app-integration tests to verify stable browser-panel behavior and browser-to-detail updates.
