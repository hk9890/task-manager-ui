## Description

Per `docs/RUNTIME_UI_VERIFICATION.md`: build bwb, run against the embedded fixture, trigger cold-start in each of the four loading surfaces, and visually verify the new skeleton appearance, pulse animation, layout stability, and theme rendering on both dark and light terminals.

## Why

T1–T3 change runtime UI behavior. Per docs/TESTING.md, automated tests verify code correctness but not feature correctness in a real terminal. The bug class being fixed (visual ugliness, layout shift, lack of motion feedback) is invisible to unit tests — needs operator verification. The light-theme adaptive color set in T3 is particularly hard to test without eyes-on.

## Verification setup

```bash
mise run build  # or: go build -o /tmp/bwb ./cmd/bwb

repoPath="$(mktemp -d)"
sh internal/testing/e2e/embeddedfixture/setup.sh "$repoPath" internal/testing/e2e/embeddedfixture/seed.json

(cd "$repoPath" && BD_NON_INTERACTIVE=1 /tmp/bwb --debug 2>/tmp/bwb-debug.log)
```

Run Checks 1–5 once on a **dark-theme terminal** and once on a **light-theme terminal** (toggle iTerm2 / Ghostty / your terminal theme between sessions, or use a different terminal app for the second pass).

## Checks

### Check 1 — Board cold-start
1. App starts on the board. Initial render shows skeleton rows in each of the 4 columns.
2. **Assert (visual):** Each skeleton row is shaped like `▓ ▓▓ ▓▓▓ ▓▓▓▓ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓` — 5 distinct ▓ blocks separated by single spaces (type / priority / state / id / title segments).
3. **Assert (visual):** Title segment widths vary across rows (not 6 identical lines).
4. **Assert (visual):** Skeletons pulse — color cycles smoothly over ~1.2 seconds; not strobing.
5. **Assert (visual):** When data lands, columns transition without layout shift — column borders don't move.

### Check 2 — Search results cold-start
1. Press Ctrl+Space to switch to search. Type a query, press Enter.
2. **Assert (visual):** Results pane briefly shows skeleton rows matching Check 1's shape and pulse.
3. **Assert (visual):** When results land, results-pane border doesn't shift.

### Check 3 — Search detail preview
1. With search results showing, select a result that has not been previewed before.
2. **Assert (visual):** Detail preview pane shows skeleton rows matching Check 1's shape — NO `⏳` hourglass, NO "Loading details for…" text.

### Check 4 — Details cold-start (full layout)
1. From board, open an issue's details (cold-start path — first time viewing).
2. **Assert (visual):** Detail pane immediately renders the full 3-pane layout (Dependencies | Content | Metadata) with skeleton rows in the Content pane. Pane boundaries are present from frame 1.
3. **Assert (visual):** When detail loads, pane boundaries do NOT shift.

### Check 5 — Settled state
1. Wait 2 seconds after all loads complete.
2. **Assert (visual):** No skeleton rows remain in any surface.
3. **Assert (visual):** Footer status line shows the string "Idle" (no spinner glyph, no "Loading: …" text).
4. **Assert (visual):** Header braille spinner glyph cell is two literal spaces (the fixed-width idle state per `headerSpinnerCell()`).

### Check 6 — Light theme rendering (NEW)
Repeat Checks 1–4 on a light-theme terminal.
- **Assert (visual):** Skeleton glyphs are visible against the panel background (not flat-on-flat).
- **Assert (visual):** Pulse animation is perceptible. If barely perceptible, note as a "tune follow-up" but PASS the check (per T3 design — light theme dynamic range may be narrower; we ship as designed and tune later if user-visible).
- **Assert (visual):** No theme-specific rendering artifacts (inverted contrast, missing chars).
