## Description

`internal/repository/memory/repository.go:711` (`Snapshot`) writes a `SnapshotIssue` containing only `DependsOn`. `BlocksIDs`, `Related`, `ParentID`, and `ChildrenIDs` are silently omitted. A `SaveNow → Hydrate` round-trip therefore drops parent/child relationships and related-issue references — the next session's cache hit returns an `IssueDetail` with empty `Related`, empty `BlockedBy`/`Blocks` reverse lookups, and a missing `ParentGroupBrowser` until a manual write triggers Forget and a backing refresh.

## Instructions

1. Inspect `SnapshotIssue` (top of `memory/repository.go` or in `memory/forget.go`). Add the missing fields with JSON tags: `BlocksIDs []string`, `Related []string`, `ParentID string`, `ChildrenIDs []string` (or whatever names match the in-memory `storedIssue` field set).
2. In `Snapshot()`, copy these fields from `storedIssue` into the emitted `SnapshotIssue` (deep-copy slices to avoid aliasing live state).
3. In `Seed()` / `LoadWithManifest`'s seed loop, restore the fields when constructing `memory.Issue` so subsequent `toDetailLocked` projections see them.
4. Verify `toDetailLocked` already reads these fields when projecting `IssueDetail.Related`, `IssueDetail.BlockedBy`, `IssueDetail.ParentGroupBrowser`, etc.; if not, route them.

## Files to Modify

- `internal/repository/memory/repository.go` (SnapshotIssue struct, Snapshot, Seed, toDetailLocked)
- `internal/repository/memory/forget.go` (if SnapshotIssue/SeedDetail live here)

## Test

Add `TestSnapshot_LosslessRoundTrip_FullDetail`:
1. Seed a `memory.Repository` with an issue whose `IssueDetail` has non-empty Blocks (reverse), Related, ParentID, ChildrenIDs.
2. Call `Snapshot()`, encode each `SnapshotIssue` as JSON, decode back into `SnapshotIssue`, feed back via `Seed()`.
3. Call `Issue(id)` on the reconstructed repo.
4. Assert deep-equal on the four fields.

Pair this with an existing-style `TestFilestorageSaveLoad_PreservesFullDetail` in `filestorage_test.go` if helpful.
