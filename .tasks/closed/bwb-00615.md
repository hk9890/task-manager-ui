## Description

Make the embedded fixture system support both:
- **Minimal anchor** (existing 3-issue seed.json) — for tests that need exact ID assertions (`bwf-1`, `bwf-2`, `bwf-3` etc.).
- **Scale** (new ~600-issue scale-seed.json from [[faif.2]]) — default for parity/integration tests that need realistic edge cases.

Audit existing consumers and route them appropriately.

## Instructions

1. Read `internal/testing/e2e/embeddedfixture/fixture.go` (and siblings) to understand the current API. Currently `SharedFixtureRepoPath(t)` returns a single fixture path.
2. Add a parallel `SharedScaleFixtureRepoPath(t)` that returns a path to the scale fixture (using a separate sync.Once / temp dir from the minimal one).
3. In `internal/testing/datasets/datasets.go`:
   - Keep `Fixture(t)` returning the minimal anchor (backward compat for existing tests).
   - Add `ScaleFixture(t)` returning the scale fixture as a Dataset.
   - Update `External(t)` doc to mention `ScaleFixture` for tests wanting realism without an external DB.
4. Audit every consumer of `Fixture(t)` and `SharedFixtureRepoPath(t)`:
   - For tests that assert specific IDs (`bwf-1` etc.) → keep on minimal anchor.
   - For tests that iterate "any issue" (parity tests, contract invariants, smoke checks) → consider migrating to ScaleFixture or running against both.
   - For tests that benefit from realistic data shapes (search relevance, sort tie-breaks) → migrate to scale.
5. Update the parity tests (`internal/dashboard/parity/`, `internal/mode/search/parity/`) to iterate over `Fixture`, `ScaleFixture`, `ThisRepo`, `External` — runs against all four when env-gates are set.
6. Update `bwb-smoke` to run against scale fixture as part of its check matrix (in addition to whatever `--dir` is passed).
7. Document the dual-fixture policy in `internal/testing/datasets/datasets.go` package doc:
   - "Use Fixture for tests that need exact ID assertions; use ScaleFixture for tests that need realistic edge cases."

## Acceptance Criteria

- [ ] `Fixture(t)` returns the minimal anchor; existing tests pass unchanged
- [ ] `ScaleFixture(t)` returns the scale fixture; new tests can opt in
- [ ] Parity tests iterate over both fixtures (where applicable)
- [ ] Smoke binary's check matrix includes scale fixture
- [ ] Dual-fixture policy documented in datasets package doc
- [ ] `mise run test:integration` passes — every existing test still works
- [ ] At least one new test in the parity suite exercises an edge case only present in the scale fixture (e.g., cap-engagement on Done column)
