issue_edit_document_test.go: The tests cover the happy-path round-trip and the no-change diff, but miss several important edge cases: (1) ClearLabels path in BuildIssueUpdateInput — when edited.Labels is empty but original has labels, ClearLabels should be true. This is untested. (2) Priority boundary values — only priority 2→1 is tested. Priority 0 and 4 parsing, and out-of-range values like 5 or -1, are not tested. (3) parseIssueEditPriority accepts 'P2' or 'p2' prefix syntax (line 275) but no test exercises this. (4) Empty assignee clearing — going from 'hans' to '' is untested in the diff. (5) Description with trailing/leading newlines — issueEditExtractField strips newlines (line 270) but description content newlines should be preserved. No test confirms multi-paragraph descriptions survive. Suggested action: add test cases for each of these 5 gaps.
