## Description
While implementing beads-workbench-hw1n the tasker ran `go test -race ./...` and observed 3 pre-existing failures unrelated to the hw1n work. Captured here so they don't get lost.

## Findings

### 1. `cmd/bwb` — global variable race
- Tests: `TestRun_VersionUsesFallback` cleanup races with `TestRun_CheckConfigPrintsWarningsAndSuccess`.
- Suspected cause: shared global `main.version` variable mutated by one test's cleanup while another reads it.
- File hint: `cmd/bwb/architecture_guardrails_test.go` is mentioned in the report; actual sites in the `main` package tests.

### 2. `internal/mode/search` — flaky timeout
- Test: `TestSearchModeEmbeddedFixtureInitUsesEmptyQueryFallback`.
- Fails intermittently with "condition not met after 1s" under load; passes 3/3 when run in isolation. Pre-existing fixture/teatest timing sensitivity.
- Both sl8c and tj4m taskers also saw this independently — same root cause.

### 3. `internal/ui/shared/markdown` — renderer global race
- Test file: `renderer_test.go`.
- Suspected cause: a shared global renderer state mutated concurrently across test cases.

## Instructions
- Investigate each independently. Likely fixes: extract globals into per-test state, or guard with mutex / `t.Setenv` style isolation.
- Each fix can ship as its own commit / small PR; no need to bundle.

## Acceptance Criteria
- [ ] All three test sites pass under `go test -race ./...` repeatedly (≥10 runs)
- [ ] No new globals introduced by the fix
- [ ] Full quality gate green
