Second wave after cqtk (commit 0f5d277). Golden tests fixed. Remaining failures:

**macOS (1 test)**:
- TestRun_CWDAndConfigResolutionAndStartOptions: 'expected config path resolved against start cwd, got /private/var/folders/...'. macOS t.TempDir returns /var/folders/... but path resolution canonicalizes via /private/var symlink. Fix: filepath.EvalSymlinks both sides before compare, OR set expected to the resolved path.

**Windows (multiple tests)**:
- TestLoad_DirectoryAtConfigPathReturnsError: still fails because testUserConfigDir(t) returns real APPDATA on Windows (UserConfigDir ignores HOME on darwin/windows). vkmt fix worked for macOS but Windows needs APPDATA env override too. OR skip this Unix-semantics test on Windows.
- TestLoad_UnreadableConfigReturnsError: 'permission bits differ on windows' — Windows ACLs ≠ Unix permission bits. Skip on Windows.
- TestRunCWDInaccessibleExitsWithCode1: same pattern as TestRunCWDExecuteOnlyExitsWithCode1 (which we already skipped). New name suggests recent rename — re-apply skip.
- TestRWMutexParallelReadOverlap: 'parallel read overlap too slow: 135ms >= 100ms (reads appear serialized)'. Windows scheduler overhead. Either skip on Windows or loosen threshold to 200ms.
- Cascading 'unknown' status tests after TestRunCWDInaccessibleExitsWithCode1 in cmd/bwb: typical of a single test panicking and killing the test process. Likely fixes by skipping the trigger.
