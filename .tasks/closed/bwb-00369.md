## Description

CLI startup edge cases identified as untested in the 2026-05-16 audit:

1. `--debug` combined with `--help`/`--version`: debug flag is silently ignored. Either document this is intended, or carry debug into the non-interactive path. Add a test asserting current behavior so future changes are intentional.
2. `--cwd` pointing at a directory the user cannot enter (EACCES — `--x` permission). Today this passes startup because `resolveAndValidateCWD` only checks IsDir. Test should land alongside the fix in beads-workbench-db0z.17.
3. `--config` with a relative path resolved against `os.Getwd()` while the current dir is not the process start dir. Currently tested only at the main level, not at the config-load level.

## Files to Modify

- `cmd/bwb/main_test.go`
- `internal/config/load_test.go` (for #3)

## Instructions

1. Add a `--debug --help` test: assert exit code 0 and that no `[bwb-debug]` lines appear on stderr (or they DO, depending on the decided contract). Document the decision.
2. Add an EACCES test: create a temp dir, chmod 0111 (or 0000 if reachable), pass as `--cwd`, assert exit code 1 and a clear error mentioning access.
3. Add a config relative-path test invoking `LoadWithOptions` from a different cwd than the process start dir.

## Acceptance Criteria

- [ ] All three coverage gaps covered with tests
- [ ] Decisions documented in CODING.md if they imply contracts (e.g., debug+help behavior)
- [ ] Full quality gate green
