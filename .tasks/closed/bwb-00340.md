## Interface

```go
type Repository interface {
    // Reads
    Dashboard(ctx context.Context) (DashboardData, error)
    Issue(ctx context.Context, id string) (domain.IssueDetail, error)
    Search(ctx context.Context, query domain.SearchIssuesQuery) (domain.SearchResultPage, error)

    // Writes
    CreateIssue(ctx context.Context, input domain.CreateIssueInput) (domain.CreateIssueResult, error)
    UpdateIssue(ctx context.Context, id string, input domain.UpdateIssueInput) error
    CloseIssue(ctx context.Context, id string, input domain.CloseIssueInput) error
    AddComment(ctx context.Context, id string, input domain.AddCommentInput) error

    // Setup
    HealthCheck(ctx context.Context) error
    Catalogs(ctx context.Context) (Catalogs, error)
}
```

## ErrorInjectingRepository (test-only decorator, per review C1)

Replaces the `FakeBeadsGateway.SetError(method, err)` pattern that ~13 tests rely on (concentrated in `internal/app/mutation_errors_test.go` and `internal/app/model_test.go`). Production `memory.Repository` is happy-path only -- this decorator wraps it for failure-path testing.

## Godoc contract requirements (per review smaller items)

Each method's godoc MUST document:

1. **Concurrency safety.** Spec: all methods are safe for concurrent use by multiple Bubble Tea goroutines. Reads may overlap with reads; writes serialize against everything.
2. **Cancellation semantics.** Spec: when ctx is cancelled during execution, the method returns `ctx.Err()` promptly. Dashboard fan-out cancels any in-flight underlying calls.
3. **Partial-failure shape for Dashboard.** Spec: if 2 of 5 underlying calls fail, Dashboard returns an error (no partial result) -- callers must treat Dashboard as atomic. Document this so callers don't expect "best-effort" partial composition.
4. **Error code stability.** Spec: unknown-ID returns wrapped `domain.GatewayError` with `ErrorCodeCommandFailed` to match bd's behavior; missing-DB returns `ErrorCodeNoDatabaseFound`; etc. (Concrete contract per method.)
5. **Catalog freshness.** Spec: `Catalogs()` results may be stale relative to externally-added labels mid-session. Documented limitation (per review smaller items; matches today's behavior). Caller should not call `Catalogs()` per-edit-form-open expecting fresh data.
