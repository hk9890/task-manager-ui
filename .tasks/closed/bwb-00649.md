## Description

Migrate every call site that today uses `repobeads.BeadsGateway` / `repobeads.NewCLIGateway` to construct and consume the lean `repository.Repository` from E4.1 instead. Switch production wiring in `cmd/bwb` and `cmd/bwb-smoke` so the lean Repository is what runs at startup, wrapped in `validatingRepository` (from E4.2).

## Sites to migrate

Production:
- `cmd/bwb/main.go` ŌĆö construct `repobeads.New(runner)` wrapped in `repobeads.NewValidating(...)`; drop any `BeadsGateway` import
- `cmd/bwb-smoke/main.go` ŌĆö same; refactor `runCountCheck`, `runSortCheck`, `runSearchCheck`, `runDashboardFetch` to take `repository.Repository` instead of `repobeads.BeadsGateway`

Tests:
- `internal/dashboard/parity/count_parity_integration_test.go` ŌĆö `runDashboardFetch` takes `Repository`
- `internal/testing/datasets/datasets.go` ŌĆö replace `NewGateway(t, ds) repobeads.BeadsGateway` with `NewRepository(t, ds) repository.Repository`
- `internal/mode/board/model_test.go` ŌĆö argv-cardinality assertions move to the `CommandRunner` seam (wrap the runner with the recording executor, assert recorded argvs there)
- `internal/mode/search/argv_cardinality_test.go` ŌĆö same
- `internal/mode/search/model_integration_test.go` ŌĆö Repository-based wiring
- `internal/repository/contract/parity_integration_test.go` ŌĆö remove `stubBeadsGateway`; stubbing where needed happens at the `CommandRunner` seam

## Instructions

1. For each production file: update imports, switch construction, run `go build`, fix any compile errors.
2. For each test file: switch the SUT (system-under-test) construction, update stub/recording wiring to the new seam, run the tests, fix any failures. Argv-cardinality assertions: wrap `*CommandRunner` with the recording executor instead of wrapping the gateway.
3. Once all sites are migrated, the only remaining references to `BeadsGateway` should be inside `internal/repository/beads/` itself (legacy code that E4.4 deletes).
4. Update consumers of `internal/testing/datasets/datasets.go` across the test tree where the helper is called.
