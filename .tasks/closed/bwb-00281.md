runner.go line 136: DecodeJSONInto calls decoder.DisallowUnknownFields(). This means if the bd CLI adds ANY new field to its JSON output, the gateway will return a decode error. For a TUI consuming an external CLI, this is backwards — you want to be tolerant of new fields and strict about fields you depend on. This will cause runtime breakage on bd version upgrades. Suggested action: remove DisallowUnknownFields() from DecodeJSONInto. The typed map[string]any decoding in read_gateway.go already handles field-by-field extraction safely.
