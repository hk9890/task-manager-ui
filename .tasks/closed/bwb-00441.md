At terminal heights where a board column or details pane has more rows than fit in the viewport, two coupled defects appear in every list rendered through `internal/ui/styles.FormSection`:

1. **Selection scrolls invisibly past the viewport.** Pressing `j` moves the model's selection index forward, but `FormSection` does a hard top-clip (`section.go:64-66`: `content[:innerHeight]`). The selected row leaves the visible window with no `›` chevron anywhere on screen; subsequent destructive actions (`x` close, `e` edit, `u` update) act on an issue the user cannot see.

2. **Headers do not signal truncation.** Done column shows `N of M` honestly (e.g. `20 of 89`) because the composer computes `TotalIsExact` from `ClosedTotal`. Ready / Not Ready / In Progress columns and the details panes show only `%d` even when the rendered window is smaller than `len(rows)`.

## Symptom (runtime evidence)

In `/home/hans/dev/github/dtctl-test` at terminal height=25:
- Ready column header shows `86` (no "of M") with ~19 rows visible.
- Press RIGHT to focus Ready, then `j` ×30 → status line: `Selected: dtt-2k5 → dtt-45v`. Visible rows unchanged; no chevron in the column.
- Done column correctly shows `20 of 89`.

## Affected sites (board)

- `internal/dashboard/composer.go:167-191` — hardcodes `TotalIsExact: true` and `Total = len(<col>Issues)` for NotReady / Ready / InProgress.
- `internal/mode/board/model.go` — selection movers update `Selected*` but no per-column scroll offset exists today.
- `internal/ui/board/board.go:91-96` — `topRight` chooses `"%d"` vs `"%d of %d"` based on `col.TotalIsExact`.
- `internal/ui/styles/section.go:64-66` — `FormSection` does the hard top-clip.

## Affected sites (details)

- `internal/mode/details/model.go:366 moveRelatedSelection(delta)` — updates `BrowserSelectedIndex`; does NOT touch `DependenciesScrollOffset`.
- `internal/mode/details/model.go:309 moveMetadataSelection(delta)` — updates `MetadataSelectedField`; does NOT touch `MetadataScrollOffset`.
- `internal/ui/details/details.go:242,303` — Dependencies pane uses `sliceWithOffset(deps, state.DependenciesScrollOffset, innerHeight, ...)`. Renderer is correct; the model never advances the offset.
- `internal/ui/details/details.go:393` — Metadata pane same shape.
- `internal/ui/details/details.go:246,310` — header is `fmt.Sprintf("%d", countDependencyReferences(detail))`, no "of M".

(Content pane is not affected — it has no selection, only explicit user scroll via `applyScrollAction`.)

## Out of scope

The Done column has the same visible-selection symptom but is structurally different: only viewport-sized rows are loaded from the repository (`opts.ClosedLimit`), so client-side scrolling cannot reveal hidden rows — there is nothing in memory to scroll into. Tracked separately as `beads-workbench-vtvb`.

## Fix

Shared helper:

```go
// internal/ui/scroll/window.go  (new package, or in ui/styles)
func EnsureVisible(offset, sel, window int) int {
    if window <= 0 { return 0 }
    if sel < offset { return sel }
    if sel >= offset + window { return sel - window + 1 }
    return offset
}
```

Call sites:

1. **Board** — introduce per-column scroll offsets in `internal/mode/board.Model` (one offset per section, mirroring `DependenciesScrollOffset` shape in details). Selection movers call `EnsureVisible` with the column's `innerHeight = sectionItemCapacity()`. Pass offset to renderer via a new `Column.ScrollOffset` field; renderer slices `rows[offset : offset+window]` instead of letting `FormSection` top-clip.

2. **Details Dependencies** — in `moveRelatedSelection`, after updating `BrowserSelectedIndex`, look up the selected ref's line index in the rendered dependency list (or precompute a parallel `lineIndex[i]` map), call `EnsureVisible(m.DependenciesScrollOffset, lineIndex, paneInnerHeight)`. The `paneInnerHeight` must be threaded from the renderer's layout into the model — either by emitting it after layout (preferred) or by mirroring the `height-2` arithmetic.

3. **Details Metadata** — in `moveMetadataSelection`, after updating `MetadataSelectedField`, look up the field's line index in the rendered metadata list, call `EnsureVisible` on `MetadataScrollOffset`.

4. **Header honesty** — in `internal/ui/board/board.go` `Render`, compute `visible := len(rows)` after the window slice; if `visible < col.Total`, render `"%d of %d", visible, col.Total` regardless of `col.TotalIsExact`. Apply the same in the Dependencies pane (`details.go:246,310`).
