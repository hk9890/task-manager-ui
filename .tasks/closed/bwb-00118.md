## Summary

`FakeBeadsGateway.CloseIssue` sets `Summary.Status = "closed"` but never sets `existing.CloseReason`. After `CloseIssue(id, input{Reason:"fixed"})` followed by `ShowIssue(id)`, the fake returns `CloseReason=""` but real bd returns `CloseReason="fixed"`. After `CloseIssue(id, input{Reason:""})`, fake returns `""` but real bd returns the literal `"Closed"` (bd's default when no reason is given).

## Discovered during

`beads-workbench-aww9` Read-detail audit (parent epic: `beads-workbench-yspw`).

## Why this masks behind tests

The lifecycle integration test runs against real bd so it would see the correct value. The fake-backed `RunWriteContract/CloseIssue/HappyPath` only asserts `Status="closed"` — it does not assert `CloseReason`. So the fake silently diverges.

## Fix

In `internal/testing/fakes/beads_gateway.go` CloseIssue method:

```go
existing.Summary.Status = "closed"
if input.Reason != "" {
    existing.CloseReason = input.Reason
} else {
    existing.CloseReason = "Closed"
}
```

Add `CloseIssue/CloseReasonStored` sub-test to the write contract that exercises both the explicit-reason and default-reason paths.
