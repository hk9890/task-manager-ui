The validating decorator logs 'repository contract violation' (rule=BlockedByNotNil, method=Issue) when Issue.BlockedBy is nil from the bd subprocess path. 312 occurrences. CORRECTION (review): this is NOT a post-v0.6.0 regression — the validating decorator and BlockedByNotNil rule did not exist at the v0.6.0 tag (added in commit 4bdbc0b). v0.6.0 sessions logged 0 violations because the DETECTOR was absent, not the condition; the nil-slice is almost certainly pre-existing and merely newly-detected. Root cause PINNED: leanDepsFromPayload (internal/repository/beads/lean_payloads.go:415) uses named return 'blockedBy' that is only ever append-ed, never initialized, so an issue with no depends-on deps returns nil. ('related' has the same latent gap but is not contract-checked.) SEVERITY: low — all consumers use len()/range/append, which are nil-safe in Go; no panic or user-visible effect. Decorator only logs. This is contract-purity / log-noise. Sample IDs: beads-workbench-99bs.4, czkq.4, znri.2.
