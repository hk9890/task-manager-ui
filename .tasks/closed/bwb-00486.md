## Description

`doc.go` exists in 13 of ~31 packages with no predictable rule — the split is
not even clean by tier (`internal/ui` has one, its leaves do not). Pick one
convention, document it, and make the tree conform. Cleanup-only — `doc.go`
files contain package doc comments, NO FUNCTIONAL CHANGE.

## Convention decision

Default: **only top-level subsystem packages get a `doc.go`** (least churn).
Leaf packages (`internal/ui/*` leaves, `internal/mode/{board,details,search}`,
`internal/version`, leaf `internal/testing/*`) do not. Record this decision as a
comment on this task before implementing; if the user picked the other rule
("every package gets one"), follow that instead.

## Instructions

1. Confirm the chosen rule (see above) and record it as a task comment.
2. Add a short "doc.go convention" subsection to `docs/CODING.md` stating the
   rule.
3. Make the package tree conform: under the subsystem-only rule, remove `doc.go`
   from leaf packages that have one OR (under the all-packages rule) add a
   minimal `doc.go` everywhere missing. Each `doc.go` is a package comment only.
4. Run `go vet ./...` to confirm nothing broke.

## Files to Modify

- `docs/CODING.md` (add convention subsection)
- `internal/**/doc.go` (add or remove to match the rule)

## Acceptance Criteria

- [ ] `docs/CODING.md` states the `doc.go` convention explicitly
- [ ] Every package's `doc.go` presence/absence matches the stated rule
- [ ] `mise run vet` and `mise run build` pass
- [ ] No functional change — package doc comments only
