## Outcome
Pressing `e` on an issue launches the external editor with full terminal control, and on save the edits round-trip back into `bd` with the detail view reloaded. The current garbled editor/TUI fight is eliminated.

## Root cause
`editIssueCmd` (internal/app/model.go) runs the interactive editor inside a `tea.Cmd` goroutine while Bubble Tea still owns the terminal (alt-screen + raw mode, with its own stdin reader). The editor and the TUI fight over the TTY, producing an unusable editor in any real terminal.

## Fix approach
Route the editor launch through `tea.Exec`, which Bubble Tea must process from `Update` (it cannot run from a goroutine). This requires splitting the monolithic `editor.Service.EditIssue` into a prepare phase and an apply phase so `tea.Exec` can sit between them.

## Success criteria
- `e` launches the configured editor with the TUI cleanly suspended and restored.
- Saved edits are parsed and applied via the gateway; no-change and error paths handled.
- teatest coverage for the full flow; one real-terminal verification run confirms the raw-mode handover.
