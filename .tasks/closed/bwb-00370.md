## Description

Two small launcher hygiene items from the 2026-05-16 audit.

1. **m6 — `splitEditorCommand` uses `strings.Fields`.** `internal/launcher/editor/opener.go:54-61`. An operator config like `editor.command: "code --wait --new-window 'path with space'"` silently breaks. Either use a POSIX-quote-aware splitter (e.g. small inline function or `mvdan.cc/sh/syntax`) or document that the editor command must be space-separated argv only.
2. **m7 — Editor temp file leak on SIGKILL/panic.** `internal/launcher/editor/service.go:64-70`. `defer os.Remove(path)` only runs on normal return. Best-effort cleanup at startup of `bwb-issue-*` patterns older than 24h would handle SIGKILL/panic leftovers.

## Files to Modify

- `internal/launcher/editor/opener.go`
- `internal/launcher/editor/service.go`
- new helper or extend existing startup path

## Instructions

1. For editor.command: pick the simpler option:
   - Quote-aware split (preferred). Add unit tests for: `code --wait`, `code --wait "with space"`, `code --wait 'single quoted'`, edge cases with embedded quotes.
   - Or document the restriction in `docs/CODING.md` and add a startup warning when `editor.command` contains a quote character.
2. For temp file cleanup: in `app.NewServices` (or wherever startup runs once), scan `os.TempDir()` for `bwb-issue-*.md` older than 24h and remove. Wrap in a goroutine so startup is not blocked. Log via the persistent log.

## Acceptance Criteria

- [ ] Editor command quoting either supported or documented + warned
- [ ] Stale `bwb-issue-*` temp files cleaned on startup
- [ ] Tests cover quote-aware split if implemented
- [ ] Full quality gate green
