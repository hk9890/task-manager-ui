Monitoring analysis found that internal/gateway/beads/runner.go logExecution always emits 'bd command finished' at INFO level, even when the bd subprocess exits non-zero or errors. Real failures (e.g. 14 'no beads database found' exit-1 results in the persistent log) are invisible to any level-based alerting or stderr mirroring.

Fix: in logExecution, emit at slog.Warn when exitCode != 0 (covers err != nil, since exitCode is set to -1 on err); keep INFO for success. Update runner_test.go execution-trace coverage and docs/MONITORING.md (the doc currently implies WARN/ERROR mirroring covers failures).
