## Description

Add supply-chain attestation to release artifacts: cosign keyless signing (no key management — uses GitHub OIDC) and syft-generated SBOMs. Lets downstream users verify "this binary really came from this commit in this repo," which becomes meaningful once bwb ships beyond the immediate team.

**Why now (this epic) vs. later**: setup is a 30-minute config change. The cost of doing it later (after broader distribution) is "we shipped N unsigned releases and now have to explain the cutover." Easier to bake in early.

## Files to Modify

- `.github/workflows/release.yml` — install cosign + syft, grant OIDC perms
- `.goreleaser.yaml` — add `signs:` + `sboms:` blocks (goreleaser drives both)
- `README.md` — add "Verifying releases" section

## Instructions

1. Edit `.github/workflows/release.yml`:
   - Update top-level `permissions:` (currently just `contents: write`):
     ```yaml
     permissions:
       contents: write
       id-token: write  # cosign keyless
     ```
   - Add steps before "Run GoReleaser":
     ```yaml
     - name: Install cosign
       uses: sigstore/cosign-installer@v3
     - name: Install syft
       uses: anchore/sbom-action/download-syft@v0
     ```
2. Edit `.goreleaser.yaml`, add (mirror `../dtctl/.goreleaser.yaml:49-65` exactly):
   ```yaml
   signs:
     - cmd: cosign
       artifacts: checksum
       args:
         - sign-blob
         - --yes
         - --output-signature=${signature}
         - --output-certificate=${certificate}
         - ${artifact}

   sboms:
     - artifacts: archive
       cmd: syft
       args:
         - "${artifact}"
         - "--output"
         - "spdx-json=${document}"
   ```
   Notes — do NOT drift from these:
   - Do **not** set `COSIGN_EXPERIMENTAL=1` — obsolete since cosign 2.0; keyless is the default mode and the env var may warn.
   - `--output-certificate=${certificate}` is **required** so downstream verifiers can pin to the OIDC identity. Without it, verification is awkward.
   - SBOM format is explicit `spdx-json` (don't rely on goreleaser defaults — they may shift across versions).
3. Verify on a snapshot run: `goreleaser release --snapshot --clean` should produce `*.sbom` (SPDX-JSON) files alongside archives and `*.sig`/`*.pem` for the checksum file. (Requires goreleaser, cosign, syft installed locally — or run via a draft tag in CI.)
4. Tag a test release (or use a throwaway tag like `v0.0.0-test1`) and verify:
   - `cosign verify-blob --signature <sig> --certificate <cert> <checksum-file>` works
   - SBOMs are uploaded to the GitHub Release page
5. Document the verification command in `README.md` under a "Verifying releases" section.
