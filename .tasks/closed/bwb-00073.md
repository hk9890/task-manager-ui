## Description
Add file-backed runtime configuration for Beads Workbench so operators can customize launchers and keybindings without rebuilding the binary.

## Goals
- Load an optional YAML config from the XDG config path for BWB.
- Keep safe built-in defaults and merge user overrides on top.
- Centralize keybinding definitions and make supported actions configurable.
- Surface non-fatal config warnings clearly while failing fast on truly invalid config.

## Decisions
- Config path: resolve with `os.UserConfigDir()` and load `bwb/config.yaml` (typically `~/.config/bwb/config.yaml`).
- Missing config file: start with defaults.
- Explicit config values override environment fallbacks such as `$EDITOR`; environment only fills omitted fields.
- Unknown YAML keys: ignore them but emit clear startup warnings/log output.
- Invalid YAML, unreadable existing config, invalid key names, and conflicting keybindings: fail startup with a clear error.
- Launcher overrides merge with built-in defaults by `action` instead of replacing the full launcher list.
- Keybinding work is split into a central keymap foundation task and a follow-on task for search/detail/modal integration.

## Success Criteria
- [ ] BWB loads an optional config from the XDG path for `bwb/config.yaml`.
- [ ] Editor and launcher settings can be overridden from YAML while built-in launcher actions remain available unless explicitly overridden.
- [ ] Shell, board, search, detail, and modal keybindings are centrally defined and configurable through per-action overrides.
- [ ] Unknown config keys produce startup warnings; invalid YAML and keybinding conflicts fail startup.
- [ ] User-facing docs describe the config path, merge semantics, and example schema.
- [ ] `go test ./...`, `go build ./cmd/bwb`, and `go vet ./...` pass.
