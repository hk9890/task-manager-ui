## Description
Add a small repository Makefile that wraps the existing documented build and test commands without introducing a separate workflow.

## Why
This repo currently documents direct Go commands in README, docs, and CI, but has no single convenience entrypoint for common local checks. A thin Makefile should improve discoverability and reduce command memorization while keeping docs and CI as the source of truth.

## Scope
- Add a minimal `Makefile` with convenience targets only
- Keep targets aligned to already-documented commands
- Update top-level docs so operators know the Makefile is a wrapper, not a second source of truth

## Proposed Targets
- `help`
- `build` -> `go build ./cmd/bwb`
- `test` -> `go test ./...`
- `quality` -> authoritative quality gate from `docs/CODING.md`
- `runtime-ui` -> focused runtime verification command from `docs/RUNTIME_UI_VERIFICATION.md`

## Files to Modify
- `Makefile`
- `README.md`
- `docs/CODING.md`
- `docs/TESTING.md`

## Acceptance Criteria
- [ ] `make help` lists the supported convenience targets
- [ ] `make build` builds `./cmd/bwb`
- [ ] `make test` runs `go test ./...`
- [ ] `make quality` runs the documented quality-gate sequence successfully
- [ ] `make runtime-ui` runs the focused runtime verification command from the docs
- [ ] README/docs explain that the Makefile is a thin wrapper around the documented commands
