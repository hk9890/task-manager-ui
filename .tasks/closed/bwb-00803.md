## Description
Add the initial built-in launcher actions that make Beads Workbench useful as a terminal tool hub for issue-driven work, while keeping v1 scope intentionally small and testable.

## Instructions
1. Implement built-in launcher definitions for:
   - `nvim`
   - `opencode`
   - a generic shell-command workflow
2. Defer `tmux` and terminal-tab specific launchers unless they can be added without platform-specific complexity and without weakening testability.
3. Ensure each launcher uses the structured issue-context interpolation from the launcher framework.
4. Add automated tests that verify constructed launcher commands via the launcher seam rather than spawning real tools.

## Files to Modify
- `internal/launcher/` (update built-in launcher definitions)
- `internal/app/` or `internal/mode/details/` (wire launcher actions into the UI)
- `internal/domain/` (update launcher models if needed)
- launcher-related `*_test.go` files (create or update)

## Acceptance Criteria
- [ ] Built-in launcher actions exist for `nvim`, `opencode`, and a generic shell-command workflow
- [ ] Additional platform-specific launchers are either deferred or explicitly justified
- [ ] Built-in launchers receive structured issue context
- [ ] Launcher integration is available from the app UI
- [ ] Automated tests verify launcher command construction without real interactive process execution
