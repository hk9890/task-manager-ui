## Description

Default `go test ./...` output is a wall of text — failures get lost in cached OKs, no summary at the end, no clear grouping. Adopt `gotestsum` as the standard output formatter. Invoke via `go run gotest.tools/gotestsum@$(cat .gotestsum-version)` so contributors do not need a separate global install (mirrors the existing `golangci-lint` pattern with `.golangci-version`).

## Files to Modify

- `.gotestsum-version` (new)

(No doc edits in this ticket — doc paragraph for the gotestsum pin policy is owned by beads-workbench-i1th.6, to avoid mid-epic merge conflicts.)

## Instructions

1. Pick a recent stable gotestsum release (check https://github.com/gotestyourself/gotestsum/releases). Pin its version in a new file `.gotestsum-version`.

2. **Version-pin convention** — match the existing repo convention. Verify the current state:
   ```bash
   cat .golangci-version
   ```
   The repo currently uses **leading-`v` convention** (file contents like `v2.1.6`). The new `.gotestsum-version` MUST follow the same convention: file contents start with `v` (e.g. `v1.12.0`). The invocation therefore reads:
   ```bash
   go run gotest.tools/gotestsum@$(cat .gotestsum-version) --format pkgname-and-test-fails -- ./...
   ```
   Note: NO literal `v` in the invocation — the prefix lives in the file. This matches how `tasks.lint` in beads-workbench-i1th.4 will read `.golangci-version`.

3. Sanity-check a focused run:
   ```bash
   go run gotest.tools/gotestsum@$(cat .gotestsum-version) --format pkgname-and-test-fails -- ./internal/launcher/...
   ```
   Confirm: produces per-package summary AND a final `DONE` line; exits non-zero on failure; lists each failed test with package qualifier.

4. Document the default `--format` for the mise tasks: `pkgname-and-test-fails` for normal runs. A `standard-verbose` variant for debug runs is wired in beads-workbench-i1th.4 as `test:verbose`.

## Acceptance Criteria

- [ ] `.gotestsum-version` file exists with a pinned version starting with `v` (matches `.golangci-version` convention)
- [ ] `go run gotest.tools/gotestsum@$(cat .gotestsum-version) --format pkgname-and-test-fails -- ./internal/launcher/...` exits 0 with: per-package PASS/FAIL token, final `DONE` line listing total + failures count, gotestsum process exit code propagates the underlying test exit code
- [ ] Inducing a failure (`-run NoSuchTest` is fine — empty package) produces exit code != 0 and a visible "Failures" listing in the summary
- [ ] No new entries in `go.mod`/`go.sum` (gotestsum is `go run`, not imported)
