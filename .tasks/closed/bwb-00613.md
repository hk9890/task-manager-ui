## Description

Mirror the read-side `RunReadContract` for write methods. Encode the contract from [[9x70.1]] as fixture-agnostic invariant assertions that both fake AND real gateway implementations must satisfy.

## Instructions

1. Read the updated write-contract sections in `internal/gateway/beads/interface.go` from [[9x70.1]] to extract invariants.
2. Create `RunWriteContract(t *testing.T, factory GatewayFactory)` in `internal/gateway/beads/contract/contract.go` (or a sibling file `contract_write.go` if separation aids readability).
3. Sub-tests per method with cross-method observability invariants:
   - **CreateIssue/HappyPath**: returns non-empty ID; subsequent `ShowIssue(returnedID)` returns the issue with the fields you supplied.
   - **CreateIssue/RequiredFields**: missing title returns the documented error code.
   - **UpdateIssue/HappyPath**: after Update, `ShowIssue` reflects the change.
   - **UpdateIssue/NonExistent**: returns the documented error code for unknown ID.
   - **CloseIssue/HappyPath**: after Close, `ShowIssue(id).Status == "closed"`.
   - **CloseIssue/Idempotency**: closing an already-closed issue — document the behavior (whether it errors or no-ops; the audit answers this).
   - **AddComment/HappyPath**: after AddComment, `ShowIssue(id)` includes the new comment.
4. Add cross-method invariants — these are the analog of the ssom invariant for writes:
   - **WriteVisibilityInvariant**: every successful write must be visible via subsequent read (Create→Show, Update→Show field match, Close→Show.Status=="closed", AddComment→Show.Comments contains the comment).
   - **CountIncrementInvariant**: after CreateIssue with status=X, `CountIssues(status=X)` total increments by 1.
5. Wire `RunWriteContract` into both `fake_contract_test.go` (new file `fake_write_contract_test.go` or extend existing) and `real_contract_integration_test.go` (or sibling).
6. The real-bd write tests need a fresh writable DB each run — use a per-test `mktemp -d` + `bd init`, NOT the shared fixture (which would mutate state across runs). The dataset adapter (`internal/testing/datasets/`) currently returns the shared fixture writable; consider whether to add a `WritableTempFixture(t)` helper that gives each test its own DB.
7. If any fake-side invariant fails, leave it visible — that surfaces work for [[9x70.3]] (fake fidelity).
