Discovered during acceptance review of beads-workbench-t7o.4. Expected: after a real blur->focus cycle, the active view refreshes immediately so off-terminal tracker changes are visible without waiting for the ~60s stale interval. Actual: quick focus regain does not refresh the active surface unless it is already dirty/stale. Reproduced on board and detail. Board repro: build /tmp/bwb, seed embedded fixture repo, start bwb, run bd update bwf-1 --title 'Immediate focus refresh ready', then send focus-reporting blur/focus within a few seconds; board still shows the old title. Detail repro: open detail for bwf-2, run bd update bwf-2 --title 'Detail focus refresh blocked', then blur/focus within a few seconds; detail still shows the old title. The refresh only happens once the stale interval elapses. Likely cause: focus-regain path in internal/app/model.go still gates refresh through shouldRefreshSurface().
