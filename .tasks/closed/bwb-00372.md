## Description

The reviewer flagged `internal/app/model.go:1071-1101` (`nextMode`, `prevMode`) as suspiciously identical. The hypothesis is that `prevMode` was copied from `nextMode` and never actually reversed — both functions appear to advance forward through the mode list. If true, `mode_cycle_prev` keybinding does nothing distinguishable from `mode_cycle_next`.

This is a low-priority but real correctness check from the 2026-05-16 audit (BS6 / Nit n8 in synthesis).

## Instructions

1. Read both functions and confirm or refute the hypothesis.
2. If they ARE identical, fix `prevMode` to actually move backward through the mode list (Detail → Search → Board → Detail).
3. Either way, add a unit test asserting `nextMode` and `prevMode` traverse the cycle in opposite directions for every starting mode.
4. If a fix is made, runtime UI verification: press the `mode_cycle_prev` hotkey and confirm it moves backward.
