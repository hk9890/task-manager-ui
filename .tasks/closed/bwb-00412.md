## Description
Make board issues easier to scan by adapting only the donor's compact colored metadata ideas to Beads Workbench's standalone renderer.

## Instructions
1. Review donor patterns in `/home/hans/dev/github/perles/internal/ui/shared/issuebadge/issuebadge.go` and `/home/hans/dev/github/perles/internal/ui/styles/styles.go`, but adapt only the minimal rendering ideas relevant to BWB.
2. Do **not** introduce a full donor-style `issuebadge` package, config struct, readiness glyph system, or multi-call-site abstraction. Keep the implementation small and local to BWB's existing board/renderhelpers flow.
3. Define BWB-owned issue colors in `internal/ui/styles/colors.go` for the currently used compact metadata set:
   - types: bug red, task blue, feature green, epic purple, chore gray
   - priorities: P0 red/bold, P1 orange, P2 yellow, P3/P4 gray

7. Update board renderer tests and goldens. Adjust golden normalization so ANSI color output does not make golden files unreadable.
