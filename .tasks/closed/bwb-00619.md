## Description

Create a `contract` sub-package under `internal/gateway/beads/` that exposes a single function `RunReadContract(t, factory)`. Wire it up twice: once with the fake gateway (runs in the unit tier), once with the real CLI gateway against the shared snapshot (runs under `-tags=integration`).

This is the load-bearing mechanism that lets us trust the fake at the unit tier — if the fake drifts from real bd behavior, this suite catches it.

## Why

Without parity enforcement, fakes silently develop blind spots ("real bd returns null, fake returns empty slice" etc.). One contract function exercised by both runs solves it.

## Instructions

1. Create `internal/gateway/beads/contract/contract.go`:
   - Define `type GatewayFactory func(t *testing.T) beads.BeadsGateway` (the factory returns a gateway *bound to a freshly-prepared fixture*).
   - Define `func RunReadContract(t *testing.T, factory GatewayFactory)` that runs one `t.Run` per read method. The method-specific assertions are stubbed out for now — they'll be filled in by the next ticket.
   - Method list to cover: `HealthCheck`, `ListIssues`, `ReadyIssues`, `BlockedIssues`, `ReadyExplain`, `ShowIssue`, `SearchIssues`, `Query`, `CountIssues`, `StatusCatalog`, `TypeCatalog`, `LabelCatalog`.
2. Create `internal/gateway/beads/contract/fake_contract_test.go` (no build tag):
   - `TestFakeGatewayReadContract` calls `RunReadContract` with a factory that returns a `fakes.FakeBeadsGateway` pre-seeded to match the fixture data.
   - The pre-seeding helper (`primeFakeFromFixtureSpec(t, fake)`) reads `embeddedfixture.ReadSeedSpec(tb)` and populates the fake's response fields accordingly. This guarantees the fake and real both "contain" the same data.
3. Create `internal/gateway/beads/contract/real_contract_integration_test.go` (`//go:build integration`):
   - `TestRealGatewayReadContract` calls `RunReadContract` with a factory that:
     - calls `embeddedfixture.SharedFixtureRepoPath(t)`,
     - wires `beads.NewCommandRunner` + `beads.NewCLIGateway`,
     - returns the real gateway.
4. For this ticket, leave the per-method assertions as `t.Skip("filled in by 163c-followup")`. The next ticket will implement them.
