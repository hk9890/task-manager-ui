## Description

`internal/config/load.go:244-252` documents (in CODING.md ~line 194) that launcher YAML overrides "match built-ins field-by-field" — but the actual contract is partial replacement: `nil` means keep base, non-nil replaces. YAML empty array `args: []` is non-nil so it should clear defaults. None of this is tested.

Identified as a coverage gap in the 2026-05-16 audit.

## Files to Modify

- `internal/config/load_test.go` (or `model_test.go` where launcher merges are exercised)

## Instructions

1. Add a test: override with `args: []` clears the built-in's args.
2. Add a test: override with no `args` key preserves the built-in's args.
3. Add a test: override that omits `command` preserves built-in command.
4. Add a test: new action name in override appends (does not replace existing).
5. If any of these reveals an ambiguity in the doc, update `docs/CODING.md`.

## Acceptance Criteria

- [ ] All four override-merge scenarios have explicit tests
- [ ] If any test reveals a contract ambiguity, docs updated
- [ ] Full quality gate green
