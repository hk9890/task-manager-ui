## Description

Machine-readable invariants that the scale fixture MUST satisfy. The verifier runs against the generated fixture; if any invariant fails, CI fails — preventing silent drift as the fixture is edited.

## Instructions

1. Create `internal/testing/e2e/embeddedfixture/scale/invariants.yaml` declaring the assertions that hold for any well-formed scale fixture. Each invariant has a name, predicate description, and threshold. Examples:
   ```yaml
   invariants:
     - name: closed_count_min
       description: Done column cap engagement requires >=75 closed issues
       predicate: bd count --status closed >= 75
       threshold: 75
     - name: cardinality_warning_threshold
       description: One active group must exceed 500 to engage composer cardinality warning
       predicate: max(bd count --status open, in_progress, blocked) > 500
       threshold: 501
     - name: bulk_closed_at_cluster
       description: At least 10 closed issues share one closed_at timestamp (kh54 regression guard)
       predicate: groupby(bd query 'status=closed', closed_at).max_size >= 10
       threshold: 10
     # ...one per recipe section
   ```
2. Build the verifier program (`internal/testing/e2e/embeddedfixture/scale/verifier.go` or similar). It:
   - Sets up the scale fixture in a temp DB (via setup.sh + scale-seed.json).
   - Runs each invariant predicate against the live DB.
   - Outputs per-invariant PASS/FAIL with the observed value.
   - Exit 0 if all pass, 1 if any fail.
3. Add `mise run fixture:verify`:
   ```toml
   [tasks."fixture:verify"]
   description = "Verify the scale fixture honors all declared invariants"
   dir = "{{config_root}}"
   run = "go run ./internal/testing/e2e/embeddedfixture/scale/cmd/verifier"
   ```
4. Wire `fixture:verify` into `mise run quality`'s dependency chain (or at least `quality:fast`) so CI catches drift.
5. Document the manifest as the "fixture contract" — analog of the gateway contract from 8qw9. Any change to recipe.yaml or generator.go must keep the manifest green.

## Acceptance Criteria

- [ ] `invariants.yaml` covers every recipe threshold and every bug class the recipe was designed to exercise
- [ ] Verifier runs against a freshly-generated scale fixture and reports per-invariant status
- [ ] `mise run fixture:verify` passes on the generated fixture
- [ ] CI quality chain includes `fixture:verify` (gate it as `quality` dependency)
- [ ] Deliberate-break: edit recipe.yaml to remove the bulk-close cluster; regenerate; verifier fails with a clear message; restore. Document the test in your closure comment.
