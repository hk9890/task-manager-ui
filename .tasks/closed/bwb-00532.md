Discovered during verification of beads-workbench-t7o.1. Expected: focus/tick refresh in search mode reloads current results without changing the query or depending on a typed rune. Actual: internal/app/model.go refreshActiveSurfaceCmd sends tea.KeyMsg{Type: tea.KeyRunes, Runes: []rune("r")}; internal/mode/search/model.go handles rune input first while query focus is active, so auto-refresh appends "r" to the query instead of issuing a pure reload. Repro: build /tmp/bwb, seed the embedded fixture repo, switch to Search, leave query focus active, then send blur/focus or wait for the refresh tick; the query becomes "r" and results change unexpectedly. Impact: auto-refresh corrupts active search state, and non-rune reload bindings are not honored by this path.
