## Files to change
- `cmd/bwb/main.go` -- add the flags, wire to repository construction

## Behavior
```
--repo beads|memory   default: beads
--repo-file <path>    default: ~/.cache/bwb/<project-hash>/repo.jsonl when --repo=beads
                      required when --repo=memory (the file is the source of truth)
```

When `--repo memory` is selected and `--repo-file` is provided:
- Load the file into `memory.Repository` via the Save/Load helpers from Epic 1
- Schema mismatch -> error out with a clear message (no bd fallback in memory mode)

When `--repo beads` is selected:
- Construct `beads.Repository` over the existing CommandRunner
- The `--repo-file` is currently informational (becomes the persisted-cache path in Epic 3)

## Acceptance criteria
- [ ] Flags parsed and documented in `bwb --help`
- [ ] `bwb --repo memory --repo-file fixture.jsonl` boots and renders dashboard from the file
- [ ] Invalid flag combination (`--repo memory` with no `--repo-file`) errors out at startup with a clear message
- [ ] Unit test for flag parsing
