## Description
`Gateway.HealthCheck` at `internal/gateway/beads/read_gateway.go:44-47` currently runs a full unbounded `bd list --json` (~680 ms here) just to detect bd availability and database presence. `bd ping --json` is the documented health probe and runs in ~210 ms. Independent quick win for epic beads-workbench-au54.

## Empirical behavior (verified)
- Inside a `.beads` dir: exits 0, prints `{"status":"ok","total_ms":...}` on stdout.
- Outside (no DB), invoked from cwd: exits 1, prints `Error: no beads database found` on stderr — same substring the runner's existing check at `runner.go:114-116` matches.
- `bd` not on PATH: shell exit 127 / `exec.ErrNotFound` (handled by existing `normalizeExecutionError` mapping).

## Instructions
1. In `internal/gateway/beads/read_gateway.go` `HealthCheck`, replace argv `["list", "--json"]` with `["ping", "--json"]`.
2. Update `read_gateway_test.go` cases:
   - argv asserted as `["ping", "--json"]`.
   - Happy-path: returns nil.
   - No-database: gateway returns `ErrorCodeNoDatabaseFound` (substring match in existing runner code; no changes to runner expected).
   - bd-not-found: returns `ErrorCodeCommandUnavailable`.
