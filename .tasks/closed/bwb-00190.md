In `internal/app/model.go`, `nextMode` (line 777-791) and `prevMode` (line 793-807) produce identical results because with only 2 browse modes (Board, Search), cycling forward and backward is the same operation — it just toggles between them.

This is confusing for readers who expect them to behave differently. Replace both with a single `toggleBrowseMode` function, or if Detail should be part of the cycle, fix the cycle to actually include it.

Current behavior for both functions:
- Board → Search
- Search → Board  
- Detail → (depends on lastBrowse, but same logic in both)
