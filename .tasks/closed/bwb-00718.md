## Description

Process scaffolding that closes the human gap left by contract enforcement. The contract suite catches divergences when they hit a documented invariant; this task ensures the discipline of finding-and-fixing divergences gets practiced consistently, with a permanent log of what was found and when.

## Instructions

1. Create `internal/testing/fakes/doc.go` with package documentation that establishes the fake as a contract-conforming peer of the real implementation. Cover:
   - The fake passes `RunReadContract` against the same fixture as the real gateway.
   - The fake is NOT a casual mock — when adding behavior, ensure it matches real bd's observable surface (consult [[8qw9.1]] interface contract).
   - When a bug's root cause involves fake-vs-real divergence, the fix MUST: (a) update the fake to match real bd, (b) add an assertion to `RunReadContract` (or its invariants) that would have caught the divergence.

2. Add a "Fake drift log" section at the bottom of `doc.go` with a chronological list:
   ```
   ## Fake drift log
   //
   // 2026-MM-DD: <one-line description of divergence and fix>
   //   - Symptom: <bug or test failure that revealed it>
   //   - Cause: <how fake's behavior diverged from real bd>
   //   - Fix: <commit or PR; updated assertion ref>
   ```
   Seed with the divergences discovered during [[8qw9.3]] (the fidelity audit).

3. Update `AGENTS.md` to include a pointer: "When fixing a bug whose root cause involves fake-vs-real divergence, follow the fake-update discipline in `internal/testing/fakes/doc.go`." Add as a one-line bullet in the appropriate section.

4. Optional: extend the existing `bd close` workflow note (if there's a project-specific closure template) to ask "fake-vs-real gap involved? if yes, fake updated?" — but only if a closure template exists. Don't invent one for this.

## Acceptance Criteria
