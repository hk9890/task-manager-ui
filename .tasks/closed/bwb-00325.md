## Description

`internal/repository/caching/caching.go:315` (`SaveNow`) captures the `*memory.Repository` pointer under `c.mu.RLock()` then RELEASES the lock before `filestorage.SaveWithHash` runs and calls `Snapshot()`. Concurrently, `RefreshIfChanged` (line 168) may acquire `c.mu.Lock()`, detect a bd hash change, and call `c.memory.Reset()` which replaces `r.issues` with an empty map. If the Reset interleaves between SaveNow's pointer capture and the inner `Snapshot()`, the persisted JSONL contains 0 issues — atomically renamed over the prior session's valid cache. Next bwb cold-starts despite having warmed the cache fully.

## Instructions

1. Take the snapshot *under* `c.mu.RLock` instead of just capturing a pointer. Either:
   - hold `c.mu.RLock` for the duration of `Snapshot()` (the snapshot itself takes its own `memory.mu.RLock`, so order is c.mu→memory.mu and that ordering already exists in other paths — verify no caller goes the opposite direction); OR
   - call a new `c.memory.SnapshotUnderLock()` helper while holding `c.mu.RLock`, then release before file IO; the captured slice is value-copied so the subsequent Reset cannot affect it.
2. Confirm `filestorage.SaveWithHash` no longer relies on calling `Snapshot()` itself if you pre-snapshot — refactor its signature to accept a `[]SnapshotIssue` (or a `Snapshotter` interface that's already been frozen) if needed.
3. Update the package doc comment around line 312 if it claims "no lock nesting" — make it accurate.

## Test

Add `TestSaveNow_RacesWithReset_PersistsNonEmpty`:
1. Build CachingRepository with a deterministic synchronization point — e.g. an injected `vcStatusFunc` that blocks on a channel until the test signals.
2. Seed memory with N issues.
3. In goroutine A: call `SaveNow()`.
4. In goroutine B: call `RefreshIfChanged()` arranged so its `memory.Reset()` is queued behind A's snapshot capture but ahead of A's file write.
5. Use a small `chan struct{}` and `sync.WaitGroup` to deterministically order the two operations.
6. After both complete, Load the JSONL and assert non-empty.

If a deterministic race test proves impossible without exposing internals, gate on the structural assertion that `Snapshot()` is called while holding `c.mu.RLock` (e.g. via an assertion-helper that records lock state from a test-only hook).
