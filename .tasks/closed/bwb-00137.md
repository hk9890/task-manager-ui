## Description
Discovered during beads-workbench-lgln implementation: `TestModelReusableDetailToolScenarioCoversEditorAndLaunchersWithFakes` can fail when run in parallel with tests that mutate the package-global `scheduleRefreshTickCmd` via `withRefreshTickScheduler`. Test passes reliably in isolation. Pre-existing issue not caused by lgln.

Root cause: `withRefreshTickScheduler` is a global-var test seam mutated by concurrent parallel tests without `t.Parallel()` isolation.

## Instructions
1. Find `withRefreshTickScheduler` in `internal/app/` and identify all call sites.
2. Either wrap mutations with a `sync.Mutex` (cheap fix) OR migrate to a per-test seam (e.g., constructor parameter, context value, or model field). Per-test seam is preferable.
3. Run `go test -count=10 ./internal/app/` to verify stability.
