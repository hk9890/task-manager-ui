## Description
The full Detail view fetches an epic's child tasks into `IssueDetail.Children` but never displays them. Add a "Children" group to the Dependencies pane, include children in the dependency count, and make them navigable like other related references (so the Enter-to-open work in the sibling task can navigate into them).

The repository layer already populates `IssueDetail.Children` (internal/repository/beads/lean_reads.go, lean_payloads.go ~442-458, dependency_type=parent-child) — no repository change is expected. This is established context, not a task step (plan-review decision Q7).

## Group ordering (plan-review decision Q3)
Render the relationship groups in this order, and use the SAME order everywhere groups are ordered: **Blocked by, Blocks, Related, Children, Structure**. Children (the issue's own children, downward) sit before Structure (parent + siblings, upward). A mid-level epic can show BOTH a Children group and a Structure group — every ordering must agree.

## Instructions
1. In `internal/ui/details/details.go`, add a "Children" group to `dependencyGroups()` (~line 501) sourced from `detail.Children`, placed per the ordering above. Render with the same `relationshipGroup`/`renderRelationshipGroups` machinery as the other groups.
2. Include `detail.Children` in `countDependencyReferences()` (~line 571) so the count shown matches reality.
3. Keep the THREE parallel orderings in lockstep (plan-review decision Q2). Children must be added consistently to all three:
   (a) render order in `dependencyGroups()` (details.go ~501),
   (b) navigation flat-list in `browserItemsFromDependencies()` (internal/mode/details/model.go ~569), so cursor navigation and selection highlighting cover children, and
   (c) the group-walk in `DependencyRefLineIndex()` (details.go ~583-615) that maps a selected index to a rendered line for scroll-into-view.
   If (a)/(b) gain Children but (c) does not, arrow navigation onto a child scrolls to the wrong line or fails to keep it visible — and goldens may still pass. Confirm the line numbers against the files; they are guidance, not exact.
4. Update affected golden files (board/details rendering goldens) per docs/TESTING.md.
