## Description
Implement the standalone search experience with text search and built-in structured filters. This task must not introduce BQL or bypass the gateway by shelling out directly from the UI layer.

## Instructions
1. Implement a search mode in the app shell.
2. Use only gateway-exposed search and catalog methods; do not shell out directly from the mode/controller.
3. Support text search UX and structured filters for status, type, priority, assignee, labels, ready, and blocked.
4. Use gateway-backed filtering first and keep any local narrowing limited to already loaded results.
5. Add automated tests covering each supported structured filter and representative search result states.
6. Do not introduce BQL parsing, SQL query generation, or generalized expression evaluation.

## Files to Modify
- `internal/mode/search/` (create or update)
- `internal/ui/search/` (create or update)
- `internal/domain/` (update search query/filter models if needed)
- search-related `*_test.go` files (create or update)

## Acceptance Criteria
- [ ] Search mode exists in the app shell
- [ ] Text search works through the gateway
- [ ] Structured filters for status, type, priority, assignee, labels, ready, and blocked are implemented
- [ ] Each supported structured filter has automated test coverage
- [ ] No BQL or custom query language is introduced
- [ ] No direct UI-layer shell-out bypasses the gateway
