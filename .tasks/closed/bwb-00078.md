Acceptance review found that the app-level edit action does not invoke the rich issue editor service. Evidence: internal/app/model.go handles key 'e' by calling launchActionCmd(m.services, "editor", issueContext), while editIssueCmd (the path that calls services.Editor.EditIssue) is only referenced from tests. The passing tests TestModelLaunchEditorActionUsesLauncherService and TestModelEditIssueActionUsesLauncherServiceFromDetailSelection confirm the current UI launches the generic launcher service instead of the rich issue-edit document flow. This blocks acceptance criterion: external editor handoff works for rich issue editing from the app.
