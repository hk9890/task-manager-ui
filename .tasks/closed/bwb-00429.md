## Description
`TestOsCommandExecutorSignalKillPreservesError` (`internal/bd/runner_test.go:617`) is NOT integration-tagged and has no `-short`/skip guard, yet it forks a real subprocess (`ex.Run(ctx, "sleep", []string{"5"}, ...)`, line 629) and depends on a 100ms wall-clock context timeout. It runs in the untagged unit build. This violates the TESTING.md tier rule ("if it forks a real subprocess ... it is integration") and adds a `sleep`-on-PATH portability dependency.

## Instructions (REVISED after plan review)
Use option (a): **retag the test as integration.** Do NOT rewrite it against `stubExecutor`.

Rationale: this test exercises the REAL executor converting SIGKILL into `(ExitCode -1, non-nil err)` — the d2oj.2 fix — which cannot be tested without forking a real process. The sibling `TestCommandRunnerRunLogsErrFieldOnSignalKill` feeds a stub `(ExitCode -1, err)` into `CommandRunner` and only covers the logging path; it never touches `osCommandExecutor`. A stub rewrite would assert "a stub returns what I set it" (testing no production code) and duplicate the sibling, destroying the only real-executor SIGKILL coverage.

1. Move `TestOsCommandExecutorSignalKillPreservesError` into the existing integration-tagged file `internal/bd/runner_integration_test.go` (which already carries `//go:build integration`).
2. Confirm no `sleep`-on-PATH dependency remains in the untagged unit tier.
3. Run `go test ./internal/bd/` (test no longer present in unit build) and `go test -tags integration ./internal/bd/` (test passes there).
