## Description
Implement the approved shell information architecture so the app chrome becomes one line, contextual guidance moves to the footer, and shell chrome styling is handled in the same task.

## Instructions
1. Update `internal/app/model.go` so the shell header renders a single-line app bar with `Beads Workbench`, two top-level tabs (`Board`, `Search`), and right-aligned context text.
2. Remove `Detail` from top-level tab rendering and mode cycling. `tab` / `shift+tab` should cycle only between `Board` and `Search`.
3. Re-map the `3` hotkey so it opens detail for the currently selected issue (same conceptual action as `enter`/`o`) rather than behaving like a third top-level tab switch.
4. When detail mode is active, keep `Board` and `Search` as the only tabs and use the header context area to show selected issue context (`Detail: <issue-id> · <status>` when available; otherwise `Detail`).
5. Replace the existing multi-line `headerHelpLines` behavior with a single-line footer that owns concise mode-specific help:
   - Board footer includes at least `h/l`, `j/k`, `enter`, `ctrl+space`, `?`, `q`
   - Search footer includes at least `type`, `/`, `tab`, `j/k`, `enter`, `esc`, `?`, `q`
   - Detail footer includes at least `j/k`, `pgup/pgdn`, `e`, `esc`, `?`, `q`
6. `UI.ShowModeSwitcherHelp` now controls footer-help visibility instead of the removed header-help lines; preserve the config flag semantics and update tests accordingly.
7. Move the existing selection/loading summary out of the footer and into the header context area. In browse modes show board/search context first and selected/loading summary as space allows; do not reintroduce a two-line header.
8. Apply shell chrome styling in the same change: app title emphasized, active tab highlighted, inactive tabs dimmed, right-side context muted, footer help dim.
9. Preserve readable narrow-width behavior by shortening context/help text rather than reintroducing a tall header.
10. Update shell tests and impacted goldens.

## Files to Modify
- `internal/app/model.go` — update header/footer rendering, key handling, mode cycling, and responsive text.
- `internal/ui/styles/colors.go` — add or reuse shell chrome color tokens used by the header/footer.
- `internal/ui/styles/format.go` — add tiny formatting helpers if needed.
- `internal/app/model_test.go` — update header/footer expectations and hotkey behavior.
- `internal/app/testdata/model_mode_switch_detail.golden` — update for the new shell chrome.
- `internal/app/testdata/model_embedded_board_w120.golden` — update for the new shell chrome.
- `internal/app/testdata/model_roundtrip_board_back_w120.golden` — update for the new shell chrome.

## Acceptance Criteria
- [ ] Header is one line at normal widths and renders only `Board` and `Search` as top-level tabs.
- [ ] `tab` / `shift+tab` cycle only between `Board` and `Search`.
- [ ] `3` opens detail for the selected issue instead of acting like a third tab switch.
- [ ] Detail mode does not render a `Detail` tab and shows selected issue context in the header.
- [ ] Footer shows mode-specific help containing the minimum key hints listed above, and `UI.ShowModeSwitcherHelp` now controls footer visibility.
- [ ] Active tab uses the existing highlighted shell treatment; inactive tabs, right-side context, and footer hints use muted/lower-emphasis styling.
- [ ] Existing shell tests and impacted goldens reflect the new layout.
