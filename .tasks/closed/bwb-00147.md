## Description

Section headers need accurate total counts independent of the display limit. There is currently no count operation in the gateway. `bd count --by-status --json` returns all non-zero status group counts in one fast call (~261ms) and is the correct backend.

## Instructions

1. In `internal/domain/queries.go` add new types:
   ```go
   // IssueCountQuery requests issue counts. Statuses/Types/Assignee/Labels narrow the
   // counted population. Empty fields = count all. Pass Statuses:[]string{"closed"} to
   // count only closed issues.
   type IssueCountQuery struct {
       Statuses  []string
       Types     []string
       Assignee  string
       Labels    []string
   }

   type IssueStatusCount struct {
       Status string
       Count  int
   }

   type IssueCountResult struct {
       Groups []IssueStatusCount
       Total  int
   }
   ```

2. In `internal/gateway/beads/interface.go`, add to `BeadsGateway`:
   ```go
   CountIssues(ctx context.Context, query domain.IssueCountQuery) (domain.IssueCountResult, error)
   ```

3. In `internal/gateway/beads/read_gateway.go`, implement `CountIssues`:
   - Add `operationCountIssues = "count issues"`
   - Build args: `["count", "--by-status", "--json"]` plus any filters from the query (use the same `buildFilterArgs` helper)
   - Parse JSON payload. **Important: groups with zero count are OMITTED by `bd count`.** Decoder must handle both single-group and multi-group output.
   - Multi-group sample (both open+closed non-zero): `{"groups":[{"count":5,"group":"open"},{"count":353,"group":"closed"}],"schema_version":1,"total":358}`
   - Empty population sample: `{"groups":[],"schema_version":1,"total":0}`
   - Single-group sample (only closed non-zero): `{"groups":[{"count":353,"group":"closed"}],"schema_version":1,"total":353}`
   - Add private payload structs `bdCountByStatusPayload` and `bdCountGroupPayload` for decoding.
   - Map to `domain.IssueCountResult`.

4. **All files that compile-assert `BeadsGateway`** must be updated. Enumerate:
   - `internal/gateway/beads/interface.go` — add method
   - `internal/gateway/beads/read_gateway.go` — implement
   - `internal/testing/fakes/beads_gateway.go` — add `CountIssuesResponse`, `MethodCountIssues` to `GatewayMethod` enum, and implementation recording `Calls`
   - Any `_test.go` files with `var _ beads.BeadsGateway = (*X)(nil)` assertions (grep to confirm)
