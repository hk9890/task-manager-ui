## Description
Define the concrete CLI/spec for the small ad-hoc verification toolkit. Keep this script-first and tightly scoped.

## Required Output of This Design Task
Produce a concise concrete design that defines:
1. exact PTY step syntax
2. exact PTY JSON output schema
3. exact per-step timeout behavior and failure reporting
4. polling behavior for wait phases (target around 50ms)
5. whether checkpoints are emitted inline or as named snapshots
6. the wrapper-script interface for before/after `bd show --json` checks
7. example commands for:
   - read-only navigation verification
   - mutation save verification
   - mutation cancel/no-save verification

## Constraints
- Keep the design small enough to implement directly in scripts.
- No framework-style branching or generalized test DSL.
- The design document/spec should stay short (roughly 30-50 lines or equivalent concise sections).

## Acceptance Criteria
- [ ] The PTY step syntax includes send-key and wait-for-text / wait-for-no-text behavior.
- [ ] The design includes per-step timeout semantics and step-specific error reporting.
- [ ] The design keeps the state-check wrapper separate from the PTY script.
- [ ] The design is specific enough that implementation can proceed without guessing.
