## Description
Update automated verification for the redesigned detail screen and run built-binary runtime verification against the embedded fixture so the final screen is checked in the real app, not only through unit/golden tests.

## Instructions
1. Update and/or add detail renderer tests and goldens to cover the new layout structure:
   - explicit height-aware rendering
   - full-height Dependencies box
   - dominant center Content pane
   - comments integrated into the center pane as a smaller lower subsection
   - right metadata rail with vertical divider and Quick actions
   - shared compact dependency row rendering
   - markdown-rendered details and comments.
2. Run the relevant automated test packages for the touched detail/app UI surfaces.
3. Build the real binary and verify the detail screen against the embedded fixture repo using the documented runtime verification workflow. The verification must include opening a seeded issue detail screen and checking that:
   - the screen uses the available height well
   - the center content area is dominant
   - comments are visually smaller and do not consume disproportionate vertical space
   - the implemented pane behavior matches the accepted 3-pane interaction model.
4. If runtime review reveals polish gaps, file follow-up beads work instead of silently ignoring them.

- [ ] Updated automated tests/goldens cover the redesigned dense full-height detail layout.

- [ ] Any newly discovered polish defects are captured as follow-up beads issues.
