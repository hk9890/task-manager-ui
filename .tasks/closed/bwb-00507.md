## Description

**Merged scope (epic-review 2026-05-16):** this task absorbs the original faif.2 (generator) and faif.3 (invariant manifest + verifier). The 3-layer abstraction was over-built for a stable fixture. This task delivers the scale fixture as a single committed JSON file plus a Go smoke test enforcing **product-behavior** invariants — not generator counts.

## Deliverables

1. **`internal/testing/e2e/embeddedfixture/scale-seed.json`** — committed artifact, ~550-700 issues. Author by hand or via a one-shot script (`scripts/gen-scale-seed.sh` or similar) — the script does NOT need to live in production code; the seed is the deliverable.

2. **`internal/testing/e2e/embeddedfixture/scale_fixture_smoke_test.go`** — Go test that loads the scale fixture and asserts product-behavior invariants (NOT generator counts).

## Scale fixture must contain

- ≥ 75 closed (engages Done column 50-cap; produces "+" badge — ssom regression guard)
- ≥ 510 issues in one active group (engages 500 cardinality threshold in composer)
- A block of 10+ closed issues sharing one `updated_at` (kh54 regression trigger)
- A block of 10+ closed issues sharing one `closed_at` (sort tie-break)
- A block of P1 issues sharing one `created_at` (active-column tie-break)
- At least one 5-deep parent chain; at least one parent with 3+ children
- At least 5 dependency-blocked issues; one block-cycle if bd supports it
- Edge text: one emoji title; one shell-metacharacter title (`;`, `'`, `"`, backtick); one max-length title; one multi-line description; one null/missing description
- A label shared across 20+ issues (for filter testing); plus rare labels
- ≥ 3 distinct keywords each appearing in 20+ issue titles/descriptions (search corpus)
- A few assignees with many issues; a few with one

## Product-behavior invariants (smoke test asserts these)

These are the **correct** invariants — they assert observable bwb behavior, not generator output:

- `composer.Compose(scaleFixture).Closed.Total >= bd count --status closed` (ssom regression guard)
- Done column row count ≤ 50 when total > 50 AND `+badge` is rendered
- Composer emits cardinality warning when one active group ≥ 500 (load it; assert log line)
- Sort order is stable when N closed issues share one `closed_at` (no kh54 false-positive warning fires; emit deterministic ordering)
- `dashboard.View()` against the scale fixture renders without doubled headers (o7tk regression guard)
- `gateway.SearchIssues("<shared-keyword>")` returns ≥ 20 results in deterministic order
- `gateway.ShowIssue` succeeds against the emoji-title, shell-metachar, and max-length issues
- `gateway.ShowIssue` succeeds against the null-description issue (781a regression guard)

## Instructions

1. Author the seed: decide between hand-edited JSON (~600 issues is tedious but auditable) vs. a one-shot generator script. Either works — the seed is the artifact.
2. If using a script, commit it to `scripts/` (not `internal/`) so it's clearly a build tool, not production code. The script need not be deterministic; the seed it produces does (commit the output).
3. Run `setup.sh` against the new seed and confirm it loads into a real bd DB without errors. If `setup.sh` needs extensions to support parents/dependencies/etc., extend it minimally.
4. Author `scale_fixture_smoke_test.go` with the product-behavior assertions above. Use `t.Run` sub-tests per invariant so failures are isolated.
5. Each invariant gets a one-line comment citing the regression class it guards (ssom / kh54 / o7tk / 781a / etc.).
