Build a seed generator that produces a beads repository on demand from a workload spec.

## Inputs

- Counts per status: open / in_progress / blocked / closed
- Average dependency density (edges per issue)
- Average comment depth
- Priority distribution (P0/P1/P2/P3 weights)
- Random seed (mandatory; for reproducibility)

## Output

- A seeded `.beads/` directory ready for bwb to open.
- A manifest JSON describing the actual generated shape (counts, density actually achieved, seed echo).

## Done when

- A small CLI or test helper accepts the spec and produces the seeded directory.
- Two runs with the same seed produce byte-identical `.beads/` content (modulo timestamps).
- Existing `scale-seed.json` workflow is untouched; this is a separate path.

## Why this exists

Today `scale-seed.json` is hand-edited; varying the workload means editing JSON. For load testing we need to sweep shapes programmatically.
