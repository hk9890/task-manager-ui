## Description
Prepare repository-visible and GitHub-visible release notes for v0.2.0 before tagging. This repository uses tag-triggered releases, so the primary versioned artifact in git is the release documentation rather than a standalone version file.

## Instructions
1. Review commits since `v0.1.0` with `git log v0.1.0..HEAD --oneline` and collect the user-facing changes that belong in the release notes.
2. Update `CHANGELOG.md` with a new `## [v0.2.0]` section using the existing Keep a Changelog style in this repo.
3. Review `README.md` and `docs/RELEASING.md` for any release-related text that would be inaccurate after shipping v0.2.0; update only if needed.
4. Draft the GitHub release notes from the finalized changelog and commit history so the publish task can apply them to the release.
5. Commit the documentation/release-note changes and push the release commit to origin before tagging.

## Acceptance Criteria
- [ ] `git log v0.1.0..HEAD --oneline` has been reviewed for release-note input
- [ ] `CHANGELOG.md` contains a v0.2.0 entry
- [ ] Release-related docs are reviewed and updated only where needed
- [ ] Release notes content is ready to publish to the GitHub release
- [ ] Documentation changes are committed and pushed before tagging

## Files to Modify
- `CHANGELOG.md`
- `README.md` (if needed)
- `docs/RELEASING.md` (if needed)
