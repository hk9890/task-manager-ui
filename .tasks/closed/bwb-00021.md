## Outcome

`internal/repository/beads/` contains exactly one slick `beads.Repository` type built directly on `*bdrunner.CommandRunner`. The 17-method `BeadsGateway` interface, its `Gateway` implementation in `read_gateway.go`, the `validatingGateway` wrapper, the `read_payloads.go` DTOs, all gateway-shape tests, and the `contractcheck` package (if orphaned after this work) are deleted. Contract validation is preserved as a `validatingRepository` decorator at the **Repository** boundary, operating on domain return types (`DashboardData`, `IssueDetail`, `SearchResultPage`, etc.). All call sites use `repository.Repository`; nothing references `BeadsGateway`. Docs reflect the lean structure.

## Motivation

E1 (epic [[hhho]]) built the `Repository` interface and `memory.Repository`. E2 (epic [[8pxi]]) wrapped `BeadsGateway` in a thin `beads.Repository` adapter and relocated the gateway types from `internal/gateway/beads/` into `internal/repository/beads/`. The 17-method gateway shape was not deleted — it survived the move and is still the inner layer of the bd-backed stack.

Runtime today: `App → repository.Repository (9 m.) → beads.Repository (adapter) → validatingGateway → BeadsGateway (17 m.) → Gateway impl → CommandRunner → bd` — six layers, two public surfaces.

Runtime after this epic: `App → repository.Repository → validatingRepository → beads.Repository (lean, direct) → CommandRunner → bd`.

## Reimplement, do not copy

The new `beads.Repository` is reimplemented from scratch. `read_gateway.go` is read as **specification** (which bd argv, which JSON fields) — its code is not the source. Anything that exists only to satisfy the 17-method shape (wrapping DTOs, methods that exist because the interface declared them, awkward caches) does not make the cut.

## Scope — 5 implementation tasks + acceptance review

- E4.1 — Reimplement `beads.Repository` directly over `CommandRunner` (parallel to legacy code; both build green)
- E4.2 — `validatingRepository` decorator at the Repository boundary (migrate rules from `validatingGateway` + `contractcheck`)
- E4.3 — Migrate all callers from `BeadsGateway` to `Repository`; switch app wiring
- E4.4 — Delete legacy gateway-shape code
- E4.5 — Docs sweep
- AR — Acceptance review

Dependency chain: E4.1 → E4.2 → E4.3 → E4.4 → E4.5 → AR.

## Out of scope

- The `CommandRunner` in `internal/gateway/beads/` (the bd-subprocess primitive) stays
- The `vc_status.go` work (E3 / kxci building block) is untouched
- `internal/repository/caching/` and `internal/repository/memory/forget.go` (in-flight E3 work) untouched

## Coordination with kxci

This epic blocks E3 [[kxci]]. kxci's `CachingRepository` decorator is designed to compose `Repository` implementations; doing it on top of the sandwich shape would force rework. Pause kxci execution until this epic closes.
