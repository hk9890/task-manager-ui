## Description
Detail mode currently renders issue content as a static block with no viewport/scroll support. If description, notes, comments, or dependency context exceed the available screen height, the user cannot scroll to inspect the rest of the content.

## Evidence
- User explicitly asked what happens when detail content does not fit on one page
- `internal/mode/details/model.go` is a thin render wrapper with no viewport state
- current codebase has no detail scroll/page navigation implementation

## Instructions
1. Add scrolling support for dedicated detail mode (and compact preview only if appropriate).
2. Ensure long descriptions/comments/dependencies remain readable in a constrained terminal.
3. Add tests for scroll behavior and rendered viewport state.
4. Keep `esc` back-to-browse behavior intact.

## Files to Inspect
- `internal/mode/details/`
- `internal/ui/details/`
- `internal/app/model.go`
- related detail/app tests

## Acceptance Criteria
- [ ] Long detail content can be scrolled in dedicated detail mode
- [ ] Users can reach content below the first screenful
- [ ] Navigation remains smooth and does not break `esc` back behavior
- [ ] Tests cover long-content scrolling behavior
