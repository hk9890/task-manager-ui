## Description
Create the foundational logging package that all BWB code should use.

This task establishes the central `internal/logging` package, session correlation, the default persistent JSON Lines log file, and fixed log rotation so follow-up integration work does not need to invent its own logging behavior.

## Instructions
1. Add `internal/logging` as the package that owns logger construction and component scoping.
2. Build it on top of Go `log/slog` and expose a small API that application code can use without touching handler setup directly.
3. Resolve the default log destination under XDG state (for example via `os.UserStateDir`) and ensure the BWB state directory exists before opening the log file.
4. Write diagnostics as JSON Lines records and include a per-process/session identifier on every record, following the decisions locked in `beads-workbench-oynk`.
5. Implement built-in rotation using `gopkg.in/natefinch/lumberjack.v2` with fixed defaults: max size 10 MB, max backups 5, max age 30 days, compress enabled.
6. Define and implement the fallback behavior when the XDG state directory or log file cannot be opened: emit a single stderr warning and continue with stderr-only logging.
7. Add focused unit tests for log-path resolution, record shape, forced rotation, session ID format/stability, and log-path failure fallback.

## Acceptance Criteria
- [ ] `internal/logging` provides a root logger plus component-scoped loggers/helpers
- [ ] JSON Lines records include timestamp, level, message, and `session_id`
- [ ] `session_id` uses a short random hex format and remains stable for one BWB run
- [ ] The default log file lives under XDG state in a BWB-owned directory
- [ ] Rotation uses the fixed defaults chosen for the epic
- [ ] A focused test forces rotation and proves rotated output is produced
- [ ] Behavior when the log path cannot be opened is covered by tests and emits exactly one stderr warning before continuing with stderr-only logging
- [ ] Focused unit tests cover package behavior
