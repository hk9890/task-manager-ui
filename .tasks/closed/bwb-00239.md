## Summary

When `LabelCatalogResponse` is the nil zero value, the fake's `append([]LabelOption(nil), nil...)` evaluates to nil. The real gateway uses `make([]LabelOption, 0, ...)` and always returns a non-nil slice. The contract spec says "returns empty slice (not an error)."

Additionally, the fake does NOT apply `strings.TrimSpace` or skip blank entries when reading `LabelCatalogResponse`. Real gateway always does. Currently no test exercises this divergence.

## Discovered during

`beads-workbench-r1m3` Read-catalogs audit (parent epic: `beads-workbench-yspw`).

## Fix

Initialize `LabelCatalogResponse: []domain.LabelOption{}` in `NewFakeBeadsGateway()` OR add a nil-check + make-empty fallback in the fake's `LabelCatalog` method. Add a contract test asserting non-nil empty slice when unconfigured.
