## Description

Production tripwire. Wraps the real `CLIGateway` so every response is validated against the contract invariants; violations emit `slog.Warn` (never fail the request). When bd's behavior drifts in production — schema change, edge case the test suite didn't anticipate, version skew — the warning surfaces in `~/.local/state/bwb/bwb.log` immediately, before users notice corrupted UI.

## Instructions

1. Create `internal/gateway/beads/validating_gateway.go`:
   ```go
   type validatingGateway struct {
       inner  BeadsGateway
       logger *slog.Logger
   }
   ```
   Implement every BeadsGateway method by delegating to `inner` and then running the invariants. Log with structured attrs: `method`, `violation`, `sample` (first 1-2 violating items), input summary.

2. The invariants must MIRROR those in `RunReadContract` (from [[8qw9.2]]) — same set of checks, same logic where possible. Factor a shared `internal/gateway/beads/contract/validators` package or keep validators inside `contract.go` exported and reuse them. The tradeoff: importing the contract package from the gateway might create cycles; if so, keep validators in a new neutral package (`internal/gateway/beads/contractcheck/`) consumed by both.

3. Wire `NewCLIGateway` to return `validatingGateway{inner: rawCLIGateway, logger: ...}` by default. Add a constructor variant `NewCLIGatewayRaw` for tests that intentionally probe invariant-violating responses (the contract tests themselves; ssom-style negative tests).

4. **High-cardinality degraded mode**: when a response has > 5000 items, skip full validation and instead spot-check the first 10 and last 10. The exact threshold is a per-method constant; defend the choice in a comment. Goal: bounded overhead even on very large DBs.

5. **Critical**: include the ssom invariant. Track CountIssues calls (by query.Statuses) within the gateway's lifetime; when a subsequent ListIssues with matching statuses returns N items at the limit, assert CountIssues(matching) ≥ N. The state is per-gateway-instance and ephemeral; don't try to persist it.

6. Add tests under `internal/gateway/beads/validating_gateway_test.go`:
   - Each invariant has a test feeding a deliberate violation (via a mocked `inner`) and asserting the warning fires with the expected structured attrs.
   - One end-to-end test wires `validatingGateway` over a real subprocess against the embedded fixture and asserts NO warnings fire (fixture is well-formed; if a warning fires here it's either a fake fidelity bug or a real bd quirk we should document).

## Acceptance Criteria

- [ ] `validatingGateway` implements every BeadsGateway method with delegate + validate pattern
- [ ] Invariant set is shared with `RunReadContract` (no duplicate definitions)
- [ ] `NewCLIGateway` returns the wrapped gateway by default; `NewCLIGatewayRaw` exists for opt-out
- [ ] High-cardinality fallback kicks in above 5000 items with a documented rationale
- [ ] ssom invariant included and tested with a deliberate-break unit test
- [ ] `validating_gateway_test.go` covers each invariant violation as a separate sub-test
- [ ] End-to-end embedded-fixture test produces zero warnings (or documented warnings are the fixture itself violating a contract — fix the fixture)
- [ ] `mise run quality` passes
- [ ] Overhead measured: invariants add < 5% wall time on a 1000-issue gateway call (capture in a benchmark or note as future work)
