## Description
Resolve the remaining design decisions that determine how the centralized logging implementation should behave.

The logging contract has now been decided and no longer requires follow-up discussion.

## Final Decisions
- [x] Human-facing warnings and errors are written to both `stderr` and the persistent JSON Lines log file
- [x] `--debug` enables DEBUG-level logging and mirrors debug diagnostics to `stderr`
- [x] The existing `[bwb-debug]` stderr prefix is preserved for compatibility
- [x] Log rotation uses `gopkg.in/natefinch/lumberjack.v2`
- [x] Fixed rotation defaults are: max size 10 MB, max backups 5, max age 30 days, compress enabled
- [x] If the XDG state directory or log file path is unavailable or unwritable, BWB emits a single `stderr` warning and continues with `stderr`-only logging
- [x] Session IDs use short random hex generated once per run, are included in every structured record, and are printed once to `stderr` during debug startup so operators can correlate runs
- [x] Gateway execution traces log full argv in local file logs

## Outcome
The remaining implementation tasks now have a concrete logging contract and can proceed without improvisation.
