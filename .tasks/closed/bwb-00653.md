## Outcome

Warm-cache hydration works in default operation. Each bwb still writes to its own per-session path (preserves Q4), but at startup the loader scans for the most-recent prior session's file (by manifest `synced_at`) for the same project and hydrates from that.

## Files (modify)
- `cmd/bwb/main.go` — change `constructRepository` to compute a separate `hydrateFromPath` (most-recent prior file) vs `writeToPath` (own session path); call `Hydrate(hydrateFromPath)` then `SetCacheFilePath(writeToPath)` (new method below)
- `internal/repository/caching/caching.go` — add `SetCacheFilePath(path string)` to separately set the write path post-Hydrate. Hydrate must NOT change the cache file path; or refactor Hydrate to take a `(loadPath, savePath)` pair. Pick the simpler shape.
- New helper at `cmd/bwb/cache_resolver.go` (or in main.go directly): `findLatestProjectCacheFile(cacheBaseDir, projectHash string) (string, error)`:
  - Glob `<cacheBaseDir>/<projectHash>-*/repo.jsonl.manifest.json`
  - Parse each manifest, pick the one with the latest `synced_at`
  - Skip unreadable/corrupt manifests with a warning
  - Returns the corresponding `repo.jsonl` path (NOT the manifest path)
  - Empty result → return ""

## Simpler shape (recommended): refactor Hydrate to take loadPath separate from writePath

```go
// Hydrate loads from loadPath (read source) and sets writePath for subsequent SaveNow.
// loadPath may be empty (cold start); writePath may be empty (no persistence).
func (c *CachingRepository) Hydrate(loadPath, writePath string) error
```

Update existing callers:
- `cmd/bwb/main.go`: pass (latestPath, ownSessionPath)
- Existing kxci.4 tests pass the same path for both (preserves current semantics for those tests)

Update existing test helpers to use the new 2-arg form.
