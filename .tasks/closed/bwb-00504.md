## Description

Verify that the `HealthCheck` method on `BeadsGateway` is in sync with real bd 1.0.4 behavior, the FakeBeadsGateway implementation, and the existing test coverage.

Part of epic `beads-workbench-yspw`.

## Methods in scope

- `BeadsGateway.HealthCheck(ctx context.Context) error` — `internal/gateway/beads/interface.go:190`
- Argv shape: `bd ping --json` (`ARGV_CONTRACT.md` row 1)

## Contract restated (from interface.go)

**Preconditions:**
- ctx must be non-nil.

**Postconditions:**
- Returns nil when bd is reachable and the DB is healthy.

**Error semantics:**
- `ErrorCodeCommandUnavailable`: bd binary not installed or not in PATH.
- `ErrorCodeNoDatabaseFound`: bd is reachable but no .beads DB in working directory tree.
- `ErrorCodeCommandFailed`: bd ping exited non-zero for any other reason.

**bd quirks observed (audit 2026-05-16):**
- `bd ping --json` emits `{"status":"ok","query_ms":N,...}` — only `status` is meaningful for health; gateway ignores payload, returns nil on zero exit. Disposition: ACCEPT.
