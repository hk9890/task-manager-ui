## Description

The parity tests and bwb-smoke shipped with hardcoded references to `ai-config-manager` (env var names, default path, test names, mise task). External DBs should be project-agnostic: one env var for the path, no separate enable flag, no hardcoded default.

## Instructions

1. `internal/testing/datasets/datasets.go`:
   - Rename `AIConfigManager(t)` → `External(t)`.
   - Consolidate env vars: drop `BWB_PARITY_AICM` and `BWB_PARITY_AICM_PATH`; introduce single `BWB_PARITY_EXTERNAL_PATH`. If unset → skip cleanly. If set but `.beads/` missing → skip with reason.
   - Remove `defaultAICMPath` constant.
   - Update Dataset.Name from "ai-config-manager" to "external".
2. Update callers in test files:
   - `internal/dashboard/parity/count_parity_integration_test.go`
   - `internal/dashboard/parity/sort_parity_integration_test.go`
   - `internal/mode/search/parity/search_parity_integration_test.go`
   - `internal/testing/datasets/datasets_test.go` — rename TestAICM* tests to TestExternal*.
3. `.mise.toml`: remove `[tasks."smoke:aicm"]`. Make `[tasks.smoke]` accept passthrough args so user can do `mise run smoke -- --dir /any/path --readonly`.
4. Docs:
   - `docs/RUNTIME_UI_VERIFICATION.md`: replace `BWB_PARITY_AICM=1 BWB_PARITY_AICM_PATH=...` with `BWB_PARITY_EXTERNAL_PATH=...`. Replace `mise run smoke:aicm` with `mise run smoke -- --dir /path/to/external/repo --readonly`.
   - `docs/RELEASING.md`: same replacements.
5. Update the AICM comment in `internal/gateway/beads/runner.go:268` to generic "external databases".
