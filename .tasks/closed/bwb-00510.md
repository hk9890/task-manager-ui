## Description

Extend `internal/testing/e2e/embeddedfixture/` to seed the bd fixture **once** per test process and hand each sub-test a fast filesystem copy. This is the foundation that makes the read-contract tier cheap.

## Why

Today every integration test calls `Seed()`, which forks `sh setup.sh` and pays ~10 `bd` subprocess startups (~10–12s). Reads don't mutate the repo, so they can share. Seeding once and copying the seeded `.beads/beads.db` per test brings per-test setup from ~10s to a few ms.

## Instructions

1. Add a new function `SharedFixtureRepoPath(tb testing.TB) string` to `internal/testing/e2e/embeddedfixture/fixture.go`.
   - First call per process: seed a fixture in a process-local cache dir (e.g. `os.TempDir()/bwb-embedded-fixture-cache`), guarded by `sync.Once`.
   - Each call: copy the seeded directory tree to a fresh `tb.TempDir()` sub-path and return that path.
   - The copy must include `.git/` and `.beads/` (SQLite db + any auxiliary files). Use `cp -a` via `os/exec` or `io/fs.WalkDir` + plain Go copy — pick the simplest that preserves perms.
2. Verify the snapshot/restore round-trips correctly by running `bd ready` against the cloned dir as a sanity check inside the helper (and `tb.Fatal` if it fails — defensive).
3. The existing `Seed(tb, repoPath)` function stays in place and remains the source of truth for "what is in the fixture". The shared helper internally calls `Seed` on the cache dir.
4. Add a unit test in `internal/testing/e2e/embeddedfixture/fixture_test.go` (or new file) that:
   - Skips if bd/jq/git missing.
   - Calls `SharedFixtureRepoPath` twice from two sub-tests and asserts:
     - both returned paths are distinct,
     - both contain a `.beads/beads.db`,
     - both successfully respond to a `bd query` call,
     - the second call is at least 10× faster than the first (rough perf guard).
5. The new test file should be gated behind `//go:build integration` (it forks bd).

## Files to Modify

- `internal/testing/e2e/embeddedfixture/fixture_integration_test.go` — new file with the round-trip test
