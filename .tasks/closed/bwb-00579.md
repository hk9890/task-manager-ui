## Description
Refactor search-mode state so it preserves the full gateway search page and distinguishes draft query input from the applied query, enabling honest completeness UI and reload behavior.

## Instructions
1. Update search mode to store draft query, applied query, current search page metadata, and loading/reload state instead of flattening the gateway response immediately to `[]IssueSummary` only.

4. Update search-mode and app-shell tests to cover the new session-state behavior.
