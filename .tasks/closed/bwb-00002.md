## Description
Tab key is handled by BOTH the active mode controller AND the shell in search mode, causing unpredictable behavior.

## Evidence
When tab is pressed in search mode:
1. The key is forwarded to search controller → calls cycleFocus(1) at internal/mode/search/model.go:132
2. The shell's tab handler at internal/app/model.go:258-270 also fires because CapturesShellKey does not capture tab
3. Result: single tab press both mutates search pane focus AND triggers mode switch

In board mode, tab is captured by the board controller (board/model.go:160) so the shell correctly short-circuits. Search mode does not have equivalent capture logic.

## Instructions
1. Have search mode's CapturesShellKey return true for tab key, matching board mode's behavior
2. Verify tab correctly cycles query→results→preview without switching modes
3. Add test coverage for this interaction

## Acceptance Criteria
- [ ] Tab in search mode cycles pane focus without switching modes
- [ ] Tab in board mode continues to work as before (column navigation)
- [ ] Tab behavior is consistent: captured by active mode when mode uses it, falls through to shell only when mode doesn't
