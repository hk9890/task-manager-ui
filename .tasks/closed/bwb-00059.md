## Description
Redesign the issue details experience around a full-height workspace with a dedicated left browser panel and a paired center/right inspection workspace, while also fixing the app-wide shell sizing inconsistencies that currently waste screen height and width.

## Goals
- Make issue details use a dedicated left browser panel for navigating connected issues in a stable grouping context
- Treat center and right as one inspection workspace: center for issue text/content, right for metadata and lightweight actions
- Use the full available terminal height and width consistently across the app shell
- Remove the current inconsistency where board and detail use the available height differently
- Make metadata directly actionable where appropriate, starting with status editing

## Product Direction
- The left browser panel is a special part of the issue details view, not a new universal panel that every mode must copy.
- Board and search keep their own mode-specific layouts, but they must share the same shell sizing contract and stop wasting vertical space differently from detail.
- The issue-details left browser should prefer stable parent-group browsing over general relation browsing.
- Selecting an item in the left browser updates the center/right detail workspace.
- The left browser contents should stay stable while browsing within the same group.

## Problems This Epic Must Address
- Current left related-issues area behaves like a nested detail widget instead of a true side browser
- Board and detail do not use full height consistently
- Layout under-uses screen width and feels boxed in
- Dashboard and detail use height differently, so the app is not even consistently wrong
- Metadata panel is mostly passive instead of being useful for quick actions

## Success Criteria
- [ ] Issue details has a true left browser panel with stable grouping behavior
- [ ] The center/right inspection workspace uses full available screen height and width consistently
- [ ] Board, search, and detail share a consistent shell sizing contract even though their panel content differs by mode
- [ ] Selecting an issue in the left browser updates adjacent detail without mutating the browser contents unnecessarily
- [ ] Metadata panel supports at least one meaningful direct action workflow
- [ ] Acceptance review confirms layout consistency, full-height usage, and UX coherence
