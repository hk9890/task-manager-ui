## Description
Create the central configurable keymap foundation for BWB, covering the shared keybinding model, startup validation, and the first controller migrations (shell and board mode).

## Instructions
1. Inventory the current hard-coded bindings in the shell and board mode and express them as named actions in a central keymap model.
2. Define default bindings in one place for all supported contexts so later tasks can consume the same resolved config structure.
3. Add YAML-backed per-action keybinding overrides that merge with defaults instead of replacing whole contexts.
4. Validate the resolved keymap at startup and fail with clear errors for invalid key names or duplicate bindings within the same context.
5. Refactor `internal/app/model.go` and `internal/mode/board/model.go` to use the central keymap model instead of scattered literal key strings.
6. Update shell and board help rendering so it reflects configured bindings where practical.
7. Add tests for default bindings, configured overrides, and conflict detection for the shell/board contexts.
8. Leave search, detail, and modal integration to the follow-on task; do not duplicate ad hoc key parsing there once the central model exists.

## Files to Modify
- `internal/config/*.go` — central keymap model, defaults, merge, and validation
- `internal/app/model.go` and tests — shell-level key handling and help text
- `internal/mode/board/model.go` and tests — board navigation bindings
- `docs/CODING.md` and/or `README.md` — document keybinding schema and examples for implemented contexts

## Acceptance Criteria
- [ ] Supported shell and board actions use centrally defined keybindings instead of scattered literal strings.
- [ ] YAML config can override shell and board bindings per action while keeping unspecified defaults.
- [ ] Startup fails with clear errors for invalid key names or conflicting bindings in shell/board contexts.
- [ ] Keyboard help for shell and board reflects configured bindings.
- [ ] Tests cover defaults, overrides, and conflict detection for the migrated contexts.
