## Parity suite design

Parameterized over implementations:
```go
func TestRepositoryContract(t *testing.T) {
    impls := map[string]func(t *testing.T) repository.Repository{
        "memory": func(t *testing.T) repository.Repository { return setupSeededMemory(t) },
        "beads":  func(t *testing.T) repository.Repository { return setupSeededBeads(t) }, // embedded fixture
    }
    for name, factory := range impls { t.Run(name, ...) }
}
```

## Pinned scenarios (numbered list, per review C2)

The suite MUST include these scenarios. Each runs against both impls and asserts identical observable results:

1. **Empty store** -- Dashboard, Issue("missing"), Search("anything") all return empty/notfound shapes
2. **Single open issue** -- Dashboard shows it in Ready; Issue(id) returns it
3. **Dependency chain (closed -> open)** -- Dashboard.ReadyExplain.Ready contains the open issue (its blocker is closed)
4. **Dependency chain (open -> open)** -- Dashboard.ReadyExplain.Blocked contains the dependent
5. **Stored status=blocked** -- Dashboard.Blocked contains it (regardless of dep state)
6. **Sort-direction parity** -- ListIssues-equivalent ordering for Dashboard.Closed (DESC by ClosedAt default; ASC if requested via Search). **Required** because `fakes/doc.go:137-142` documents this as a known gap in the OLD fake -- must NOT be inherited.
7. **Search hit shape** -- substring across Title + Description + Notes; case-insensitive; SearchResultPage.Completeness == "complete"
8. **Mutation effects** -- CreateIssue then Issue(id) returns it; UpdateIssue priority change reflected in next Dashboard; CloseIssue moves issue to Done column; AddComment appears in next Issue(id)
9. **Unknown ID error codes** -- UpdateIssue/CloseIssue/AddComment on missing ID returns wrapped error with matching code (both impls return same `domain.ErrorCode`)
10. **Partial failure of Dashboard** -- when 2 of 5 underlying calls fail (simulated by stubbing in beads impl; trivially N/A in memory impl since no failure path), Dashboard returns error (NOT partial result). Contract: Dashboard is atomic.
11. **Time-field semantic equivalence** -- NOT byte-equal (memory uses injectable clock, beads uses bd timestamps). Assert: Updated bumps on every mutation; Created < Updated; Closed only set when status=closed; relative ordering preserved across calls. Test uses a tolerance window.
12. **HealthCheck on empty store** -- returns nil for both (or matching error if neither has a DB initialized)
13. **Catalogs shape** -- both return the same `Catalogs` struct shape; values may differ (memory seeded with defaults, beads from bd config) but neither is nil

## CI tier

The parity suite runs in `mise run test:integration` (NOT in the fast `test` tier). Rationale: the bd-backed half requires the embedded fixture and a real bd binary -- too slow/flaky for hot-path test runs. CI runs both tiers; local devs run `mise run test` by default and `test:integration` on demand or before push.

## File format

`~/.cache/bwb/<project-hash>/repo.jsonl` for issues (one JSON object per line, keyed by ID).
`~/.cache/bwb/<project-hash>/manifest.json` for metadata:
```json
{"schema_version": 1, "synced_at": "2026-05-23T17:00:00Z", "bd_commit_hash": "abc..."}
```

On load: `schema_version` mismatch -> return `ErrSchemaMismatch` (typed, defined in `hhho.1`). Caller decides fallback policy.
