## Description
The dashboard rewrite (epic beads-workbench-au54) needs to assert "exactly N bd subprocesses on dashboard load" but the current test infrastructure cannot observe that: `FakeBeadsGateway` (`internal/testing/fakes/beads_gateway.go`) records gateway-method calls, not subprocess argv. Subprocess argv lives one layer down at `CommandExecutor` (`internal/gateway/beads/runner.go:42`) which board tests bypass entirely. This task adds a test seam at the `CommandRunner` / `CommandExecutor` boundary so subprocess-argv assertions are possible from any caller.

## Why now
beads-workbench-lgln's "exactly 3 bd subprocesses" AC and beads-workbench-6czx's "exactly 4 bd subprocesses" AC depend on this seam. Both have been temporarily reframed to the gateway-method-call layer; this ticket upgrades them.

## Instructions
1. Extend `internal/testing/fakes/` (or add a new sibling file under `internal/gateway/beads/testing/` if cleaner) with a `RecordingExecutor`:
   - Implements `CommandExecutor`.
   - Records each `Run` call as `{Command, Args, WorkDir, EnvLen, At time.Time}` in an internal slice protected by a mutex.
   - Exposes `Calls() []RecordedCall` for tests; safe to call concurrently.
   - Configurable per-call response: by argv match (e.g. `OnArgs([]string{"ready","--explain","--json"}).Return(ExecResult{...})`), or a default `ExecResult` if no match.
2. Update at least one existing runner test in `internal/gateway/beads/runner_test.go` to demonstrate the recorder.
3. Add a "scenario-style" test in `internal/mode/board/model_test.go` that wires the board model up against the REAL `*Gateway` + `CommandRunner` + `RecordingExecutor` (instead of the FakeBeadsGateway) and asserts:
   - On `Init`, the runner sees exactly 3 reads (ReadyExplain + Query in_progress + Query closed) with the expected argv shape.
   - No `count` / `list --status` argv ever appears.
4. Do NOT remove FakeBeadsGateway — it's still the right tool for tests that don't care about subprocess shape. The recorder is additive.

## Files to Modify
