In `internal/mode/details/model.go`, both `HandleKey` (line 90-97) and `ClampScroll` (line 61-68) call `uidetails.Render()` to compute total line count, even though the content hasn't changed. This means every single scroll key press re-renders the entire detail view just to count lines.

The fix: cache the rendered line count when the detail content changes (i.e. when Detail is set), and use the cached value in HandleKey and ClampScroll. The render is already being done in View() anyway.

This matters because detail content can be large (descriptions, comments, dependencies), and this is a hot path (every keystroke in detail mode).
