## Description

`internal/launcher/process_runner.go:17-25` calls `cmd.Start()` and returns immediately. There is no `cmd.Wait()` reaper goroutine and no `cmd.Process.Release()`. Two consequences:

1. Each successfully launched subprocess becomes a zombie until BWB exits. Long sessions launching `nvim`/`opencode`/`shell-command` repeatedly accumulate zombies in the process table and one Go goroutine each for context-cancel watching.
2. `exec.CommandContext` is used, so when the parent ctx is cancelled (typically at BWB exit), the subprocess receives SIGKILL. That contradicts the documented "fire-and-forget" contract in `docs/CODING.md` — launches should survive BWB exit.

Major finding L2 from the 2026-05-16 audit.

## Files to Modify

- `internal/launcher/process_runner.go`
- `internal/launcher/service.go` (review only — context source)
- `internal/testing/fakes/process_runner.go` (mirror new behavior if signature changes)
- `docs/CODING.md` (launcher contract section)

## Instructions

1. Decide and document: should launchers truly outlive BWB, or are they tied to BWB's process tree?
   - Recommended: truly fire-and-forget. Switch from `exec.CommandContext(ctx, ...)` to `exec.Command(...)`. Use `setsid`/`SysProcAttr.Setpgid: true` to detach from BWB's process group so SIGHUP/SIGINT to BWB do not propagate.
   - Update `docs/CODING.md` launcher contract section to state explicitly that launchers detach.
2. Reap the subprocess: after `cmd.Start()` succeeds, spawn `go func() { _ = cmd.Wait() }()` to claim the exit status (or call `cmd.Process.Release()` if reaping is intentionally skipped on this platform).
3. Add a test asserting no zombie remains after N (e.g. 5) sequential launches against a fast-exiting fake command (`true`). Acceptable proxy: count `cmd.ProcessState != nil` after a bounded wait.
4. Add a test (or doc note) asserting BWB exit does NOT send SIGKILL to launched subprocesses. This is hard to test directly; a comment in the runner referencing the design decision is acceptable here if the test would be too fragile.
