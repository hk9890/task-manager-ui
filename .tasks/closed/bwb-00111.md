The cold-start loading state renders every dashboard column as an identical block of 6 skeleton rows filled with solid '▓' glyphs, which reads as uniform noise rather than 'loading'.

Two UX improvements (chosen with the user):

1. Text placeholders: replace the solid title-slot block with a rotating dim word (loading…/fetching…/reading…) and switch the metadata-slot glyph from dark shade ▓ (U+2593) to light shade ░ (U+2591). Affects the shared issuerow.RenderCompactSkeleton, so board/search/details all get the lighter, text-bearing skeleton.

2. Per-column row count: the board's skeletonRows now draws its row count from a fixed per-column table (4/6/3/5) so adjacent columns differ in length and never form a uniform grid.

Scope: internal/ui/shared/issuerow/row.go, internal/ui/board/board.go, plus test + golden updates.
