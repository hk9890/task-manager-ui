## Description

Adopt the CI, build, and supply-chain patterns from the sister project `dtctl` (../dtctl) that make sense for bwb's stage and trajectory. Today bwb's CI is lean: ubuntu-only matrix, no race detector, no coverage gate, no release signing, no SBOMs, and `bwb --version` only carries the version string at release time (no Commit/Date, nothing for local builds).

This epic ships these as small independent child tasks, ordered by payoff vs. effort. Each child is scoped tightly enough to land in one PR.

## Motivation

1. **Reliability**: race detector catches concurrency bugs the current suite cannot see.
2. **Observability**: `bwb --version` should reveal commit + build date for any binary — local, CI, or release — so bug reports are debuggable.
3. **Quality ratchet**: a coverage floor (set just below the current baseline) prevents regressions without forcing a write-tests-under-duress sprint.
4. **Portability**: macOS users hit bwb today. A matrix prevents "works on my laptop" regressions, especially around TUI/terminal handling.
5. **Supply chain (when distributing more widely)**: cosign keyless signing + syft SBOMs make releases verifiable. Deferred until distribution broadens.

## Approach

Split into 8 implementation children + 1 acceptance review, roughly ordered by payoff and dependency:

1. **`.1` — Go 1.24.2 → 1.26.3** (foundation, tiny diff; blocks .2/.6/.8)
2. **`.2` — `-race` in test tasks** (foundation; depends on .1)
3. **`.3` — ldflags Version/Commit/Date** (build observability)
4. **`.4` — Measure coverage baseline** (spike — informs `.5`)
5. **`.5` — Coverage gate in CI** (blocked by `.4`; gates on unit+integration combined)
6. **`.6` — goimports check in CI** (closes pre-commit-hook gap for contributors without `hooks:install`; depends on .1)
7. **`.8` — Cross-OS test matrix (ubuntu + macos + windows)** (depends on .1)
8. **`.9` — Release signing: cosign keyless + syft SBOMs** (last; meaningful once distribution broadens)
9. **`.10` — Acceptance Review** (manual verification across all changes)

**Removed from scope:** `.7` (govulncheck weekly cron) — descoped at user request; can revisit when contributor base grows.

- goimports CI: `../dtctl/.github/workflows/lint.yml` lines 22-32
- Cross-OS matrix: `../dtctl/.github/workflows/test.yml` lines 18-21
- Cosign + syft: `../dtctl/.github/workflows/release.yml` lines 27-31, `../dtctl/.goreleaser.yaml` lines 49-65

## Out of Scope

Explicitly NOT adopting from dtctl:
- SDK-style dual-module split (bwb is single-module)
- Golden / snapshot tests (no stable formatted-output surface to snapshot)
- markdownlint Docker target (low value at current docs size)
- Homebrew tap publishing (no tap exists yet)
- govulncheck (descoped — was `.7`)
