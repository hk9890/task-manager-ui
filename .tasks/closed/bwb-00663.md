## Description

Operator needs a one-command release smoke check: point at a DB, get a pass/fail report comparing what bwb's data layer says vs what `bd` says. This is the tool the user explicitly asked for to avoid "obvious render bugs shipping again".

## Instructions

1. Create `cmd/bwb-smoke/main.go`.
2. Flags:
   - `--dir <path>` (required) — DB working directory; uses `bd --readonly` when `--readonly` (default true).
   - `--readonly=true|false` (default true) — never write to the target DB.
   - `--format text|json` (default text).
   - `--checks count,sort,search,render` (default all).
3. Checks (each prints PASS/FAIL with details):
   - **count**: dashboard column totals vs `bd count` per status. Reuses helper from [[55oi]].
   - **sort**: first 20 IDs of each column vs `bd list/ready/blocked` first 20 IDs. Reuses helper from [[izds]].
   - **search**: run 4 canonical queries, compare totals + first 10 IDs. Reuses helper from [[64y4]].
   - **render**: invoke board model headlessly through a small `WindowSizeMsg` sequence and count column-top borders. Reuses helper from [[xx8p]].
4. Exit code: 0 if all pass; 1 if any fail. Print summary table: `check | dataset | status | detail`.
5. Add `mise run smoke` task: `go run ./cmd/bwb-smoke --dir .` (this repo by default).
6. Add `mise run smoke:aicm` task pointed at the AI Config Manager path (env-gated so it's opt-in).

## Acceptance Criteria

- [ ] `go build ./cmd/bwb-smoke` succeeds
- [ ] `bwb-smoke --dir <fixture>` passes all checks
- [ ] `bwb-smoke --dir <this-repo>` currently FAILS count check (exposes [[bug-done-total-wrong]]) with a clear message — this is the desired pre-fix state
- [ ] `bwb-smoke --dir <ai-config-manager> --readonly` never writes to that directory's .beads (verify by mtime snapshot)
- [ ] `mise run smoke` works from a clean checkout
- [ ] JSON output format is machine-parsable for CI
