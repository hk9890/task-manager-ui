## Description
`internal/gateway/beads/runner.go` (around line 136) uses `json.Decoder.DisallowUnknownFields()` when decoding `bd` CLI JSON output. This means any new field that `bd` adds to its JSON output will cause a hard decode error, breaking the app on minor `bd` upgrades. The gateway already handles dynamic fields via `map[string]any` extraction, making strict decoding both redundant and harmful.

## Instructions
1. Open `internal/gateway/beads/runner.go`
2. Find the `DisallowUnknownFields()` call (around line 136)
3. Remove it — the decoder should tolerate unknown fields
4. Verify existing gateway tests still pass (they test the fields we DO parse)

## Files to Modify
- `internal/gateway/beads/runner.go` (remove DisallowUnknownFields call)

## Acceptance Criteria
- [ ] `DisallowUnknownFields()` removed from JSON decoder
