## Description
Enable shell-owned focus awareness and periodic refresh triggers because `internal/app` owns cross-mode reload decisions.

## Instructions
1. Update `cmd/bwb/main.go` to enable Bubble Tea focus reporting for the main program.
2. Add app-level message/state handling in `internal/app/model.go` for focus gain/loss and periodic auto-refresh ticks.
3. Route focus/tick events through a shell-owned refresh-active-surface path that can target detail, board, or search.
4. Skip auto-refresh when help/action modals are open or the target view is already loading.
5. Define the refresh cadence as a named constant that defaults to about 60 seconds, and make focus-triggered refresh degrade safely when the terminal does not report focus events.

## Acceptance Criteria
- [ ] Program startup enables focus reporting
- [ ] Shell handles focus regain and blur without crashing or spamming reloads
- [ ] Periodic tick can request refresh of only the active surface
- [ ] Duplicate reload commands are avoided while the target surface is already loading
- [ ] Tick interval is defined as a named constant and defaults to about 60 seconds
- [ ] If focus events are unavailable in the terminal, the app still behaves correctly and scheduled refresh remains the fallback freshness mechanism
