## Description

Section headers need to show accurate total counts (e.g., "Done (353)") independent of the display limit. The `TopRight` slot in `FormSection` already exists and currently shows `len(col.Rows)` — the loaded (display-limited) count, which is the lie this task fixes.

**Key constraints identified in review:**
- The existing `TopRight` count slot (`internal/ui/board/board.go:78`) must be replaced with the true total — golden files will need updating
- Ready/Not-Ready counts cannot use `len(results)` after j1m5 lands (that would be capped by the dynamic limit, recreating the same bug)
- `CountIssues` adds a 5th serial subprocess call — defer it to load after sections render to avoid a startup regression

## Prerequisites
- `CountIssues` gateway method must exist (beads-workbench-cwkb)
- Dynamic section limit must be wired (beads-workbench-j1m5)

## Instructions

1. **Replace the TopRight count in `internal/ui/board/board.go:78`** from `len(col.Rows)` to the true total count for that section. Pass the total count into the board renderer alongside the column rows.

2. **Store counts in the board model**: add `sectionCounts domain.IssueCountResult` to `internal/mode/board/model.go`. Initially zero-value (no count shown).

3. **Dispatch `countIssuesCmd` after section queries return**, not alongside them (to avoid adding to the startup critical path). Handle the `countIssuesLoadedMsg` and store result in model.

4. **Graceful loading state**: Before `sectionCounts` arrives, omit the count entirely from `TopRight` (empty string, not "0"). This avoids a "Done (0)" flash on startup.

5. **Count source per section:**
   - "In Progress" → `sectionCounts` group `"in_progress"` (or 0 if group absent)
   - "Done" → `sectionCounts` group `"closed"` (or 0 if group absent)
   - "Ready" → fire a separate **uncapped** `ReadyIssues` query with `Limit: 0` to get the true count, stored separately as `readyCount int`. Do NOT use `len(col.Rows)` — after j1m5 that is capped at the display limit.
   - "Not Ready" → same pattern: uncapped `BlockedIssues` query for count, stored as `blockedCount int`.

6. **Handle missing groups safely**: When reading `sectionCounts.Groups`, look up by `Status` field; return 0 if not found. Never assume a group is present.

7. **Update golden files**: `internal/ui/board/testdata/board_columns.golden` and related golden files will need updating to reflect the new TopRight content.

8. **Invalidate counts** on the same triggers that reload sections: auto-refresh tick, explicit reload keybind, and post-mutation reload. Do not let counts go stale after the user creates or closes an issue.

## Files to Modify
- `internal/mode/board/model.go`
- `internal/ui/board/board.go`
- `internal/ui/board/testdata/board_columns.golden` (and any sibling golden files)
