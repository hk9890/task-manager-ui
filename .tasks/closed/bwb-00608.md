## Description
`internal/gateway/beads/runner.go` introduced a `singleflight.Group` on top of the read cache. The board fan-out issues *different* argv per parallel call (ready/inprogress/closed/count/storedblocked), so singleflight cannot collapse those. The cache eliminates the 99%-byte-identical repeats by itself. Singleflight may only fire on the narrow miss-window race for the *same* argv issued in parallel — possibly zero hits in practice.

**Decision (locked):** Remove singleflight and drop the `golang.org/x/sync` dependency. No measurement step. Rationale: the board fan-out fires distinct argv per call so singleflight can only fire on the narrow miss-window race for identical argv — vanishingly rare in practice. Running an instrumentation session for a near-certain zero result is not worth the time.

Complexity review F3.

**Sequencing:** This task is **blocked by [[gx7n]]**. Do [[gx7n]] first so this task can also clean up the now-stale singleflight mentions in the `readCache` docblock in one pass.
