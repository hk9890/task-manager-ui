## Description
The codebase has 6+ helper functions written 2-3 times each. This creates maintenance risk (change one, forget the other) and adds unnecessary code volume.

## Instructions

### 1. Replace maxInt/minInt with Go 1.24 builtins
Go 1.24 (which this project uses per go.mod) has `min()` and `max()` builtins. Replace all hand-rolled versions:
- `internal/ui/modal/modal.go` (~line 412) ŌĆö delete `maxInt`, replace calls with `max()`
- `internal/ui/shared/issuerow/row.go` (~line 80) ŌĆö delete `maxInt`/`minInt`, replace calls with `max()`/`min()`
- `internal/mode/details/model.go` ŌĆö has `maxInt`, replace calls with `max()`
- `internal/app/model.go` ŌĆö has `maxInt`, replace calls with `max()`

### 2. Replace containsString/contains with slices.Contains
- `internal/gateway/beads/read_gateway.go` (~line 433) ŌĆö delete `containsString`, use `slices.Contains`
- `internal/ui/loading/loading.go` ŌĆö delete hand-rolled `contains`, use `slices.Contains`
- Add `"slices"` to imports where needed

### 3. Consolidate normalizeIssueToken / NormalizeToken
- `internal/ui/styles/colors.go` (~line 159) has `normalizeIssueToken`
- `internal/ui/shared/renderhelpers/helpers.go` (~line 121) has `NormalizeToken`
- These are identical. Keep one (in renderhelpers since it's the shared package), delete the other, and import it
- If `normalizeIssueToken` in colors.go is unexported and used locally, make it call the shared version or move the logic

### 4. Deduplicate MergeKeyBindings nil-check pattern
- `internal/config/keybindings.go` `MergeKeyBindings` repeats the same 3-line nil-check/range/clone pattern 5 times (once per context)
- Extract a helper: `mergeContextBindings(base, override map[string][]string) map[string][]string`

**NOTE:** Gateway filter arg-building dedup is owned by beads-workbench-ia6. Do NOT touch `read_gateway.go` filter assembly here ŌĆö only the `containsString` helper.

After each group of changes, run `go build ./cmd/bwb` to verify. Run `go test ./...` after all changes.
