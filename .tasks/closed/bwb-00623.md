## Description

Run unit and integration tests with `-race` so the data-race detector catches concurrency bugs the current suite cannot see. bwb has goroutine-driven loading, refresh schedulers, and UI ticks — exactly the surface where race issues hide.

## Files to Modify

- `.mise.toml` — add `-race` to the `test`, `test:integration`, `test:verbose`, and `test:integration:verbose` task `run` commands

## Instructions

1. For each test task in `.mise.toml`, insert `-race` into the `gotestsum -- ...` flags. Example for `test`:
   ```
   run = "go run gotest.tools/gotestsum@$(cat .gotestsum-version) --format pkgname-and-test-fails -- -race ./..."
   ```
2. Run `mise run test` and `mise run test:integration` locally. Confirm both still pass and the runtime increase is acceptable (likely <30s for the current suite).
3. If a real race surfaces, file a follow-up task instead of suppressing it.

## Acceptance Criteria

- [ ] All four `test*` tasks in `.mise.toml` pass `-race` through to `go test`
- [ ] `mise run test` passes locally
- [ ] `mise run test:integration` passes locally
- [ ] CI `ci.yml` "Unit tests" and "Integration tests" steps green
