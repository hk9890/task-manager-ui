## Description

After all 15 fix tasks land, run a manual end-to-end pass against a real `bd` project to confirm the fixes work in the live UI and to catch regressions automated tests cannot. This task is the human verification step before acceptance review.

## Pre-flight

- All of beads-workbench-kb8e.1 through beads-workbench-kb8e.15 are CLOSED.
- `mise run test` clean.
- `mise run quality` clean.
- Latest `bwb` binary built.

## Manual checklist

For each item, record OUTCOME (PASS / FAIL / NOT-OBSERVED) and short notes in a comment on this task.

### Persistence (F1, F2, F3, F4)

- [ ] **F1 cache persists**: launch `bwb` in a real beads-using project, browse a few issues, exit. Verify `~/.cache/bwb/<project-hash>/repo.jsonl` and `.manifest.json` exist and are non-empty.
- [ ] **F2 large-issue load**: add a beads issue with a >100 KB description, exit, relaunch. The previously-cached state loads without "bufio: token too long" or similar errors in the log.
- [ ] **F3 parent/related/children round-trip**: cache an epic + its children + a related-issue link. Exit, relaunch. Open the epic — ParentGroupBrowser shows the children, Related pane shows related issues.
- [ ] **F4 cache file integrity**: run two sessions close together with bd state changes between them. The cache file is never observed empty / 0-length / corrupted on disk during the runs.

### Cache coherency (F5, F6, F7)

- [ ] **F5 stale per-ID after external change**: in session A, view issue X (Title="old"). Exit. Externally `bd update X --title="new"`. Launch session B. Open X — Title shows "new", not "old".
- [ ] **F6 lost-write under Hydrate**: (depends on chosen approach; if option C the Hydrate-after-Start guard exists — verify by attempting it shows an error rather than silently losing state).
- [ ] **F7 Dashboard error during hydrate**: hard to reproduce without instrumentation; if a build flag or test hook exists, exercise it. Otherwise mark NOT-OBSERVED.

### Cache-hit projection (F8, F9, F10, F11, F12)

- [ ] **F8 CreateIssue post-create detail**: create a new issue from the create form. Immediately open its detail view. Status/Priority/Type/CreatedAt match what `bd show <new-id>` returns from CLI (not input-defaults).
- [ ] **F9 cross-ref Title on cache hit**: create issue A blocking issue B. Open A — BlockedBy lists B with title. Navigate to dashboard then back to A — BlockedBy still shows title, not bare ID.
- [ ] **F10 catalogs after new-label create**: open the create-issue form, note the available labels. Create an issue with a brand-new label `manual-test-<date>`. Open the create form again. Verify the new label appears in autocomplete.
- [ ] **F11 dashboard refresh after comment**: note an issue's position in InProgress column. Add a comment to it. Refresh dashboard (or wait <1s if reactive). Verify the issue's UpdatedAt-based sort position reflects the comment.
- [ ] **F12 Creator on second view**: open issue X, note Creator in metadata pane. Navigate away. Open X again. Creator is still shown.

### Beads adapter (F13, F14, F15)

- [ ] **F13 epic children not shown as Blocks**: open an epic with multiple children. The Blocks panel does not list the children; the ParentGroupBrowser (or children pane) does.
- [ ] **F14 new child visible after external add**: open epic P. Externally run `bd dep add <new-child> <P> --type=parent-child`. Force a refresh (or wait for tick + invalidation). Open P again — new child appears in the children pane without restarting bwb.
- [ ] **F15 search completeness affordance**: search with no limit (or default that maps to Limit=0). The result list does NOT show the "load more / partial" affordance when the result set is exhaustive.

## Cross-cutting checks

- [ ] `bd quality` / `mise quality` clean on a fresh checkout.
