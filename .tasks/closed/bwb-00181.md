## Description
Refine the Beads Workbench detail metadata rail so it feels denser, more structured, and more informative. This follow-up replaces the first flat key/value rail with a compact fixed-width inspector closer to the donor Perles feel without copying it verbatim.

## Decisions
- Right rail uses a fixed width of 34 columns on wide layouts
- Labels render as a dedicated one-per-line section, not a single inline comma-separated value
- Close metadata should be shown when available: closed timestamp, duration, and close reason
- Creator metadata should also be surfaced when available

## Target Layout
- Core section: Type, Priority, Status
- Ownership section: Assignee, Creator
- Time section: Created, Updated, Closed, Duration
- Close section: Close Reason
- Labels section: one label per line
- Counts section: Comments, Blocked by, Blocks, Related

## Success Criteria
- [ ] Metadata rail uses a fixed 34-column layout
- [ ] Rail is grouped with divider lines between sections
- [ ] Labels render as a dedicated section
- [ ] Close metadata and creator render when available
- [ ] Narrow layouts still degrade cleanly
