## Description

Go program that consumes the recipe from [[faif.1]] and produces a deterministic `bd init` + create-issues sequence (or a single seed JSON file) representing the scale fixture. Fixed RNG seed for reproducibility. Add a `mise run fixture:generate` task.

## Instructions

1. Read [[faif.1]] recipe schema before designing.
2. Create `internal/testing/e2e/embeddedfixture/scale/generator.go` (or `cmd/fixture-gen/main.go` if a standalone binary reads better — pick after weighing repo conventions).
3. The generator:
   - Loads `recipe.yaml`.
   - Uses a fixed-seed RNG (e.g., `rand.New(rand.NewSource(42))` or seed from recipe).
   - Produces output in one of two forms — pick the simpler:
     - **Form A**: a single `scale-seed.json` file matching the format of the existing `seed.json` (which can be replayed by `setup.sh`). Pro: same setup pipeline; con: needs the setup.sh to support all the things in the recipe (parents, dependencies, etc.).
     - **Form B**: a shell-script of `bd create/update/close/dep add` commands that build the fixture. Pro: most direct; con: slower per-test setup.
   - I lean **Form A** — extend setup.sh capability as needed (smaller change than building a new pipeline).
4. The generator must be idempotent — running it twice produces byte-identical output.
5. Add `mise run fixture:generate`:
   ```toml
   [tasks."fixture:generate"]
   description = "Generate the scale fixture seed from the recipe"
   dir = "{{config_root}}"
   run = "go run ./internal/testing/e2e/embeddedfixture/scale/cmd/generator > internal/testing/e2e/embeddedfixture/scale-seed.json"
   ```
6. Document the generated `scale-seed.json` as a build artifact (commit it OR re-generate on every test run — pick after measuring generation time; if < 1s, regenerate; if longer, commit).
7. Add a unit test that runs the generator and asserts the output is byte-identical to a committed reference (catches accidental nondeterminism).

## Acceptance Criteria

- [ ] Generator compiles and runs from `mise run fixture:generate`
- [ ] Output is byte-identical across runs (determinism test passes)
- [ ] Generated fixture honors every count/threshold from the recipe
- [ ] Generator completes in < 5 seconds (target < 1s; budget for soft cap)
- [ ] If output is a JSON seed: setup.sh works against it unchanged or with documented extensions
- [ ] `mise run quality` passes

- possibly commit `internal/testing/e2e/embeddedfixture/scale-seed.json` (decision per step 6)
