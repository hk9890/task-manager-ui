## Description
The first pass of the right metadata rail is functionally correct, but the current UX is too loose and text-heavy compared with the donor Perles detail view. The next iteration should make the rail denser, more structured, and more informative without copying Perles verbatim.

## Discussion Outcome
- Use a fixed metadata rail width of 34 columns
- Render labels as a dedicated one-per-line section
- Add support for creator, closed timestamp, duration, and close reason
- Keep dependency information as compact counts in the right rail
- Use grouped metadata sections with divider lines instead of one flat field list

## Agreed Target Sections
- Core: Type, Priority, Status
- Ownership: Assignee, Creator
- Time: Created, Updated, Closed, Duration
- Close: Close Reason
- Labels: one label per line
- Counts: Comments, Blocked by, Blocks, Related
