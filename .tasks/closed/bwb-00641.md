## Scope

Runtime UI verification per `docs/RUNTIME_UI_VERIFICATION.md` for all three refresh surfaces, on both dark and light themes.

Mirrors the h2xu.4 + h2xu.5 runtime-verification pattern in a single task because the visual change here is smaller (dim foreground only, not full skeleton replacement).

## Procedure

1. Read `docs/RUNTIME_UI_VERIFICATION.md` for the canonical PTY-snapshot procedure.
2. For each surface (board, search, details), in each theme (dark, light):
   a. Launch bwb against a real test fixture.
   b. Trigger a refresh (manual reload, search re-execute, detail re-fetch).
   c. Capture PTY snapshots across multiple animation ticks during the refresh window.
   d. Verify (i) the rows render with a dim foreground that cycles through `styles.SkeletonShades`, (ii) the stale text content is still readable, (iii) selected row keeps its selection background highlight.
3. Persist verification snapshots (as h2xu.4 did) under whatever conventional location the project uses (search for `docs/RUNTIME_VERIFICATION` artifacts to locate).
