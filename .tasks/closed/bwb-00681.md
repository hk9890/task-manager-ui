## Description
Extend the central keymap system to the remaining interaction-heavy contexts so search, detail, and modal flows also use configurable bindings without regressing typing or navigation behavior.

## Instructions
1. Refactor `internal/mode/search/model.go` to use the central keymap for supported search actions while preserving text entry semantics in the query field.
2. Update `CapturesShellKey` so it derives from configured search bindings and still allows normal query typing for non-action keys.
3. Refactor `internal/mode/details/model.go` to use central keybindings for scroll/navigation actions.
4. Refactor `internal/ui/modal/modal.go` so modal navigation bindings come from the resolved keymap instead of package-level hardcoded defaults.
5. Update help text and any related rendering/tests so configured bindings are reflected for search, detail, and modal workflows.
6. Add tests for default behavior, configured overrides, and conflict-sensitive behavior in search/detail/modal flows.

## Files to Modify
- `internal/mode/search/model.go` and tests — search bindings and shell-capture behavior
- `internal/mode/details/model.go` and tests — detail scroll bindings
- `internal/ui/modal/modal.go` and tests — modal navigation bindings
- `internal/app/model.go` and tests — help text updates for search/detail/modal bindings
- `internal/config/*.go` — extend keymap usage/tests as needed
- `docs/CODING.md` and/or `README.md` — document search/detail/modal keybinding examples

## Acceptance Criteria
- [ ] Search actions use configurable bindings without breaking free-form query typing.
- [ ] `CapturesShellKey` respects configured bindings and still allows text entry for non-action keys.
- [ ] Detail scroll/navigation bindings are configurable through the central keymap.
- [ ] Modal navigation bindings are configurable through the same config surface.
- [ ] Tests cover defaults, overrides, and interaction-sensitive behavior for search/detail/modal flows.
