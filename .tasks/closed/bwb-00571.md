## Description

Encode the contract from [[8qw9.1]] as generic, fixture-agnostic assertions in `internal/gateway/beads/contract/contract.go`. Today RunReadContract pins specific fixture IDs (bwf-1, bwf-2, bwf-3); we add a complementary layer of assertions that hold for ANY input and catch contract drift independent of fixture content.

## Instructions

1. Read the updated `interface.go` from [[8qw9.1]] to extract the per-method invariants.
2. Add new sub-tests to `RunReadContract`, one per invariant cluster per method:
   - **ListIssues**: every returned summary has non-empty ID; if SortBy was set, output is sorted in that order; if Statuses filter was set, every result has a matching status.
   - **ReadyIssues / BlockedIssues**: every result has the expected work-state; non-empty ID; no closed issues.
   - **ShowIssue**: returned detail's ID matches input; comments and dependencies are non-nil slices (empty allowed, nil not).
   - **SearchIssues**: returned page's result-list length matches the page's reported count; every result has non-empty ID.
   - **CountIssues**: total equals sum of groups; group names are non-empty.
   - **Query**: returned issues match the requested expression (e.g., if expr was "status=closed", every result has status=closed).
   - **StatusCatalog / TypeCatalog / LabelCatalog**: non-empty; every entry has non-empty name.
3. Add the **ssom invariant** as a CROSS-method assertion: call CountIssues(closed) and ListIssues(status=closed, limit=N); when the slice is at the limit, CountIssues total must be >= len(slice). This is the assertion that would have caught ssom at contract-test time.
4. Run `mise run test:integration` — both fake and real (via embedded fixture) must pass.
5. If the fake fails an assertion that the real passes, leave the fake failure visible — it surfaces a [[8qw9.3]] fake-fidelity issue rather than masking it.

## Acceptance Criteria

- [ ] At least one generic invariant assertion exists for every method on BeadsGateway
- [ ] The ssom cross-method invariant (`CountIssues ≥ len(ListIssues)` at saturation) is encoded and named explicitly
- [ ] `mise run test:integration` passes for the real implementation (against embedded fixture)
- [ ] Any fake failures are clearly attributed to fake gaps, not contract bugs — surfaces work for [[8qw9.3]]
- [ ] Generic assertions do NOT reference specific fixture IDs (bwf-*)
