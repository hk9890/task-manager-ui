Discovered while implementing beads-workbench-ikoe (concurrent integration tests under -race). SharedFixtureRepoPath used a fixed cache dir (os.TempDir()/bwb-embedded-fixture-cache) seeded once per process via sync.Once, with no cross-process lock or completion marker (unlike the scale cache). A broken or half-seeded cache from a prior run, or a concurrent test-binary process, could poison every subsequent seed: setup.sh skips 'bd init' when .beads already exists, so a stale .beads later fails with 'issue_prefix config is missing'.

Fix applied: cache dir is now process-unique (bwb-embedded-fixture-cache-<pid>) and RemoveAll'd before MkdirAll, isolating each test binary. internal/testing/e2e/embeddedfixture/fixture.go.
