## Description

Add targeted low-overhead tests that pin the bugs found in epic `beads-workbench-znri` so they can be detected automatically. Tests are written to FAIL today (the bugs are real) and PASS once the bugs are fixed.

This task also hardens `scripts/capture_bwb_screen.py` so future exploration sessions do not generate PTY-artifact false positives like znri.2/.3/.4/.6.

## Instructions

1. **Fix `scripts/capture_bwb_screen.py` silent fall-through on unknown key names** (~3 lines).
   - Today the script silently treats unknown key-name strings as a literal send. Change it to raise an error unless the input is a single literal character or a documented named key (Enter/Esc/Tab/etc).

2. **Add an `internal/app` keybinding contract test** asserting documented global keys hold from each context.
   - Pins znri.4 ('?' toggles help) and znri.7 (ctrl+q quits on fatal screen).
   - The test should reference each documented binding in `docs/user-guide/key-bindings.md` and assert the runtime model honors it.

3. **Add a caching-repository cold-start integration test** — empty `t.TempDir()` cache + real bd → non-empty Dashboard.
   - Pins znri.2. Expected to PASS today (the bug was a harness artifact, not real) but guards against any future regression that would actually break cold-start.

4. **Add help-modal golden tests at 80x24 and 120x34**.
   - Pins znri.5 (bottom-border missing). Today these goldens cannot match because the bottom border is absent — `TestHelpModal_RenderedFrameHasClosingBottomBorder` should be marked `t.Skip` referencing znri.5 until that bug is fixed.
