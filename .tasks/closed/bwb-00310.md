errors.go line 44: GatewayError.Unwrap returns e.Cause directly. This is correct for errors.Is/errors.As unwrapping. However, GatewayError is a value type (not pointer receiver), which means errors.As will fail if callers check for *GatewayError instead of GatewayError. The runner_test.go assertGatewayErrorCode helper uses errors.As(err, &gatewayErr) with a value receiver which works, but this is a subtle Go gotcha that could bite future callers. Also, Error() drops the Cause message entirely — if Cause is set, the user-facing error loses the underlying detail. Suggested action: (1) consider including Cause in Error() output (e.g. 'operation: message: cause'), (2) add a comment on GatewayError documenting the value-receiver Unwrap behavior.
