## Description

Apply the non-blocking loading pattern to the Detail surface. The detail pane is a single monolithic gateway call (`Gateway.ShowIssue()`), so per-region loading inside detail isn't meaningful — it's all-or-nothing per issue.

Today `mode/details/model.go:67` triggers a `blockingLoad` whenever `m.Loading && !m.isPreviewingTarget()`, and `details.go:98-104` returns a full-screen loading takeover. Extend the existing preview pattern (`mode/details/model.go:54-57, 362-369, 408-422 loadingPreviewDetail`) to ALL refreshes so that:

- Cold start (no prior detail loaded — `Detail.Summary.ID == ""`): render skeleton/placeholder content.
- Refresh (same issue OR a different but previously-loaded issue): keep stale Summary/Description/Comments visible; header spinner from `beads-workbench-0x36.6` is the only signal.

## CRITICAL constraints (DO NOT VIOLATE)

- **DO NOT remove the `m.detail.Loading = true` assignment at `internal/app/model.go:771`.** `loadingStates()` (`internal/app/model.go:1267-1269`) depends on it to drive the header spinner. Only the View() RENDERING changes here, not the loading-state plumbing.
- **DO NOT regress `beads-workbench-db0z.7`.** When the TARGET ISSUE CHANGES (not refresh of the same issue), scroll offsets MUST reset. Call `ApplyLoadedDetail(newID, ...)` synchronously on selection-change BEFORE issuing the new gateway call so scroll resets immediately — don't wait for the gateway response.

Depends on `beads-workbench-0x36.1` (SkeletonRow helper).

## Files to Modify

- `internal/mode/details/model.go` — change the `blockingLoad` predicate at line 67 so the full-screen takeover only triggers when `Detail.Summary.ID == ""` (no content). Rename/export `loadingPreviewDetail` (lines 408-422) to `placeholderDetail(id, parentRef)` so both cold-start of the main path and preview can share it.
- `internal/ui/details/details.go` — when `state.Loading && state.Detail.Summary.ID == ""`, render skeleton placeholder for description/metadata/dependencies via `loading.SkeletonRow`. When `state.Loading && state.Detail.Summary.ID != ""`, render the existing detail layout unchanged. Replace the current preview-path `Description: loading.View(...)` (around line 411) with skeleton via `loading.SkeletonRow`.
- `internal/app/model.go` — at the selection-change entry point (around line 770, `selectDetailForCurrentMode`), on issue-id change, synchronously call `ApplyLoadedDetail` with a placeholder BEFORE the new `loadDetailCmd`, so scroll offsets reset immediately. Add an inline comment at the `m.detail.Loading = true` line warning future readers not to remove it.
- `internal/mode/details/model_test.go` — cover the three rendering cases (cold-start skeleton, refresh keeps stale, idle) AND add an explicit regression test that scroll offsets reset on issue-switch (no-regression for `beads-workbench-db0z.7`).
